<KeyTakeaways>

- Quilt to widelec społecznościowy z 2021 roku, będący rozgałęzieniem [Fabric](https://fabricmc.net/), który uruchamia mody Fabric bez zmian.
- Fabric to oryginalny loader z o wiele większym ekosystemem modów i bazą użytkowników.
- Kompatybilność działa tylko w jedną stronę: Quilt ładuje mody Fabric, ale Fabric nie może ładować modów dostępnych tylko dla Quilt.
- Dla większości graczy Fabric jest bezpiecznym wyborem domyślnym; Quilt odpowiada osobom, które chcą korzystać z jego dodatkowych API.

</KeyTakeaways>

Quilt to społeczna wersja rozwidlenia [Fabric](https://fabricmc.net/), która zachowuje kompatybilność binarną z modami dla Fabric, jednocześnie dodając własne biblioteki. Fabric to oryginalny ładowacz i posiada znacznie większy ekosystem modów oraz bazę użytkowników. Dla większości graczy Fabric jest bezpiecznym wyborem domyślnym. Quilt odpowiada osobom, które chcą korzystać z dodatkowych API lub wspierają rozwidlenie z zasady.

<Callout variant="info" title="Gdzie pasuje Opal">
Opal to mod Fabric, więc Fabric jest loaderem, którego potrzebujesz do niego. Build Fabric również działa na Quilt, jeśli wolisz ten fork. Zobacz [przewodnik instalacji](/pl/docs/setup).
</Callout>

## Co czym się charakteryzuje

Fabric to lekki, modułowy loader modów utrzymywany przez FabricMC. Łączy w sobie małe jądro z oddzielnie wersjonowanym [Fabric API](/pl/learn/what-is-the-fabric-api) i wykorzystuje model łatania oparty na mixinach. Szybko aktualizuje się do nowych wydań Minecrafta i posiada największy katalog modów wydajnościowych i użytkowych.

Quilt to fork Fabric, który powstał w 2021 roku po sporach o zarządzanie w FabricMC. Uruchamia mody Fabric bez zmian, dostarcza własny Quilted Fabric API jako warstwę kompatybilności z Fabric API i dodaje QSL, Quilt Standard Libraries, dla własnych API. Jego adopcja jest niewielka w porównaniu z Fabric.

## Jak się różnią

Najważniejszy praktyczny punkt to kierunek kompatybilności. Quilt może ładować mody Fabric, ale Fabric nie może ładować modów, które zależą od funkcji QSL specyficznych dla Quilt. Zatem mod napisany dla czystego Fabric dociera do obu odbiorców, podczas gdy mod zależny od QSL dociera tylko do użytkowników Quilt.

| | Fabric | Quilt |
| :--- | :--- | :--- |
| Posiadanie | Oryginalny loader od FabricMC | Fork Fabric z 2021 roku |
| Ekosystem modów | Największy w nowoczesnym Minecraft | Uruchamia mody Fabric i mody QSL |
| Własne API | Fabric API | Quilted Fabric API (kompatybilność) + QSL |
| Kompatybilność | Nie może ładować modów specyficznych dla Quilt | Ładuje mody Fabric bez zmian |
| Adopcja | Bardzo duża | Mała w porównaniu z Fabric |
| Model łatania | Mixiny | Mixiny (odziedziczone z Fabric) |

## FAQ

<FAQList>

<FAQItem question="Czy Quilt jest kompatybilny z modami Fabric?">
Tak. Quilt zachowuje kompatybilność binarną i ładuje mody Fabric za pomocą warstwy Quilted Fabric API. Odwrotna sytuacja nie ma miejsca: Fabric nie może uruchamiać modów, które wymagają wyłącznie API QSL od Quilt.
</FAQItem>

<FAQItem question="Dlaczego Quilt istnieje, skoro Fabric już działa?">
Quilt oddzielił się od FabricMC w 2021 roku z powodu sporów dotyczących zarządzania. Ma na celu zachowanie kompatybilności z Fabric, oferując jednocześnie własny projekt API poprzez QSL.
</FAQItem>

<FAQItem question="Czy klient narzędzia powinien celować w Fabric czy Quilt?">
Zazwyczaj celują w Fabric, ponieważ baza użytkowników i wsparcie wersji są większe, a build Fabric również działa na Quilt. Budowanie wyłącznie dla Quilt w większości przypadków zmniejszyłoby odbiorców bez żadnych korzyści.
</FAQItem>

<FAQItem question="Czy jeden jest szybszy od drugiego?">
Nie ma znaczącej różnicy. Quilt dziedziczy model mixinów z Fabric, więc wydajność zależy od zainstalowanych modyfikacji, a nie od loadera.
</FAQItem>

</FAQList>
