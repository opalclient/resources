GraalVM JavaScript to szybki, zgodny ze standardami silnik JavaScript, który działa na JVM, będący częścią projektu [GraalVM](https://www.graalvm.org/). Aplikacje Java go osadzają, aby użytkownicy mogli pisać kod JavaScript, który wywołuje funkcje programu hosta i przeładowuje go w locie. W aplikacji Java oznacza to dodawanie lub zmienianie zachowania w JavaScript bez rekompilacji kodu Java.

## GraalVM i GraalVM JavaScript

GraalVM to wysokowydajny środowisko uruchomieniowe, które może wykonywać wiele języków na JVM. Jego silnik JavaScript jest wystarczająco nowoczesny, aby obsługiwać najnowsze funkcje ECMAScript. Aplikacja dostarcza silnik, przekazuje skryptom zestaw obiektów do pracy i je uruchamia.

## Dlaczego aplikacje go osadzają

- **Gorące przeładowanie**: zmień skrypt i zobacz efekt bez konieczności ponownego uruchamiania lub przebudowywania.
- **Bezpieczna powierzchnia**: host decyduje dokładnie, które obiekty i klasy Java skrypt może dotykać.
- **Interoperacyjność z Javą**: skrypty mogą wywoływać własny kod aplikacji poprzez obiekty, które ona udostępnia.

## Jak pojawia się w kliencie

Program osadzający silnik zazwyczaj udostępnia skryptom punkt wejścia do rejestrowania się, a następnie mały zestaw „obiektów proxy”, które łączą JavaScript z wnętrznościami programu. Użytkownicy piszą skrypt, wrzucają go do folderu ze skryptami, a silnik go ładuje. W ten sposób klient może pozwolić użytkownikom na dodawanie funkcji w JavaScript zamiast kompilowania moda.

## FAQ

<FAQList>

<FAQItem question="Czy GraalVM JavaScript to to samo co Node.js?">
Nie. Oba uruchamiają JavaScript, ale silnik GraalVM ma być osadzony w aplikacji JVM, podczas gdy Node.js to autonomiczny środowisko uruchomieniowe zbudowany na V8.
</FAQItem>

<FAQItem question="Czy muszę znać Javę, żeby napisać skrypt?">
Zazwyczaj nie. Piszesz JavaScript w oparciu o obiekty, które udostępnia aplikacja hostująca. Znajomość tych obiektów pomaga przy bardziej zaawansowanym użytkowaniu.
</FAQItem>

<FAQItem question="Dlaczego używać skryptów zamiast pełnej modyfikacji?">
Szybkość iteracji. Skrypt przeładowuje się natychmiast, podczas gdy skompilowaną modyfikację trzeba ponownie zbudować i uruchomić.
</FAQItem>

</FAQList>
