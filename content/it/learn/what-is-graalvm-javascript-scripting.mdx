GraalVM JavaScript è un motore JavaScript veloce e conforme agli standard che funziona sulla JVM, parte del progetto [GraalVM](https://www.graalvm.org/). Le applicazioni Java lo integrano, consentendo agli utenti di scrivere JavaScript che richiama il programma host e si ricarica dinamicamente. In un'applicazione Java, ciò significa aggiungere o modificare il comportamento in JavaScript senza ricompilare il codice Java.

## GraalVM e GraalVM JavaScript

GraalVM è un runtime ad alte prestazioni in grado di eseguire diversi linguaggi sulla JVM. Il suo motore JavaScript è sufficientemente moderno da supportare le funzionalità ECMAScript più recenti. Un'applicazione include il motore, fornisce agli script un insieme di oggetti con cui lavorare ed esegue gli script.

## Perché le applicazioni lo integrano

- **Hot reload**: modifica uno script e osserva l'effetto senza riavviare o ricostruire.
- **Superficie sicura**: l'host decide esattamente quali oggetti e classi Java uno script può toccare.
- **Interoperabilità Java**: gli script possono chiamare il codice dell'applicazione stessa tramite gli oggetti che espone.

## Come si presenta in un client

Un programma che integra il motore di solito fornisce agli script un punto di ingresso per registrarsi, quindi un piccolo insieme di "proxy" oggetti che collegano JavaScript agli interni del programma. Gli utenti scrivono uno script, lo inseriscono in una cartella degli script e il motore lo carica. Questo è il modo in cui un client può consentire agli utenti di aggiungere funzionalità in JavaScript invece di compilare una mod.

## FAQ

<FAQList>

<FAQItem question="GraalVM JavaScript è la stessa cosa di Node.js?">
No. Entrambi eseguono JavaScript, ma il motore di GraalVM è progettato per essere incorporato all'interno di un'applicazione JVM, mentre Node.js è un runtime autonomo basato su V8.
</FAQItem>

<FAQItem question="È necessario conoscere Java per scrivere uno script?">
Di solito no. Si scrive JavaScript interagendo con gli oggetti esposti dall'applicazione host. Conoscere a cosa corrispondono questi oggetti aiuta nell'uso avanzato.
</FAQItem>

<FAQItem question="Perché usare gli script invece di una mod completa?">
Velocità di iterazione. Uno script si ricarica istantaneamente, mentre una mod compilata deve essere ricostruita e rilanciata.
</FAQItem>

</FAQList>
