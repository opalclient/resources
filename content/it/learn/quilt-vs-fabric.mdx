## Cosa sono l'uno e l'altro

Fabric è un mod loader modulare e leggero, mantenuto da FabricMC. Abbina un core piccolo con il [Fabric API](/it/learn/what-is-the-fabric-api), versionato separatamente, e utilizza un modello di patching basato su mixin. Si aggiorna rapidamente alle nuove versioni di Minecraft e possiede il catalogo più ampio di mod per prestazioni e utilità.

Quilt è un fork di Fabric nato nel 2021 a seguito di dispute di governance all'interno di FabricMC. Esegue le mod Fabric senza modifiche, include la propria Quilted Fabric API come livello di compatibilità con il Fabric API, e aggiunge QSL, le Quilt Standard Libraries, per le proprie API. La sua adozione è limitata rispetto a Fabric.

## Come differiscono

Il punto pratico principale è la direzione della compatibilità. Quilt può caricare le mod Fabric, ma Fabric non può caricare le mod che dipendono dalle funzionalità QSL esclusive di Quilt. Quindi, una mod scritta per Fabric semplice raggiunge entrambi i pubblici, mentre una mod dipendente da QSL raggiunge solo gli utenti Quilt.

| | Fabric | Quilt |
| :--- | :--- | :--- |
| Origine | Mod loader originale di FabricMC | Fork della community di Fabric del 2021 |
| Ecosistema di mod | Il più grande su Minecraft moderno | Esegue mod Fabric più mod QSL |
| API propria | Fabric API | Quilted Fabric API (compatibilità) + QSL |
| Compatibilità | Non può caricare mod solo Quilt | Carica mod Fabric senza modifiche |
| Adozione | Molto ampia | Piccola rispetto a Fabric |
| Modello di patching | Mixin | Mixin (ereditati da Fabric) |

## Quale scegliere

<Checklist>
<ChecklistItem>Scegli Fabric se desideri la più ampia selezione di mod, il supporto più veloce per le nuove versioni e il loader a cui la maggior parte dei client e delle mod si rivolgono per impostazione predefinita.</ChecklistItem>
<ChecklistItem>Scegli Quilt se desideri le sue funzionalità QSL, eseguire una specifica mod solo per Quilt, o preferisci la governance della fork. Le tue mod Fabric esistenti funzionano comunque.</ChecklistItem>
<ChecklistItem>Non puoi eseguirle entrambe contemporaneamente. Scegli un loader per profilo, e dato che Quilt legge le mod Fabric, il passaggio da una all'altra costa poco, in entrambi i casi.</ChecklistItem>
</Checklist>

## DOMANDE FREQUENTI

<FAQList>

<FAQItem question="Quilt è compatibile con le mod di Fabric?">
Sì. Quilt mantiene la compatibilità binaria e carica le mod di Fabric tramite il suo livello API Quilted Fabric. Il contrario non è vero: Fabric non può eseguire mod che necessitano delle API QSL esclusive di Quilt.
</FAQItem>

<FAQItem question="Perché esiste Quilt se Fabric funziona già?">
Quilt si è separato da FabricMC nel 2021 a causa di disaccordi sulla governance. Mira a rimanere compatibile con Fabric offrendo, al contempo, un proprio design API tramite QSL.
</FAQItem>

<FAQItem question="Un client di utility dovrebbe puntare a Fabric o a Quilt?">
La maggior parte punta a Fabric perché la sua base di utenti e il supporto delle versioni sono più ampi, e una build di Fabric funziona anche su Quilt. Costruire esclusivamente per Quilt ridurrebbe il pubblico senza alcun vantaggio nella maggior parte dei casi.
</FAQItem>

<FAQItem question="Uno è più veloce dell'altro?">
Non c'è una differenza significativa. Quilt eredita il modello di mixin di Fabric, quindi le prestazioni dipendono dai mod che installi, non dal loader.
</FAQItem>

</FAQList>
