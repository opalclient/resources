## 自由开源与$8.99及成品

应有尽有：Wurst 免费且源代码就在那里，遵循 GPLv3 许可。你可以阅读每一行代码，进行分支，发布你自己的版本，无需支付一分钱。自 2014 年起，它一直处于持续的公开发布状态，比这个领域中几乎所有其他项目都更长久。这并非营销手段，而是事实。

Opal 售价 $8.99。购买的是别人的成品，而不是一个空白工具包：一个基于 Blaze3D 效果管线的定制玻璃 UI 引擎，70 个已经调优好的模块，而不是等你来调优，以及一个处理登录和更新的启动器，确保你的首次会话不是一个设置会话。Wurst 给你零件。Opal 给你组装好的成品。

## 无政府工具包与迷你游戏战斗：不同的工作

Wurst 自己的模块列表清楚地告诉你它的用途：Tunneller 用于条矿，Tree-bot 用于自主采伐木材，AutoFish，复制漏洞支持，BoatFly。这是一款严肃的、专为无政府和原版生存而设计的工具包，在像 2b2t 这样的服务器上，它正在完成它被构建的任务。当然也有战斗模块，但它们不是重中之重.

Opal 的模块集指向另一个方向：一套战斗装备，包含自身的瞄准和辅助瞄准子包，桥梁辅助模块，移动工具，以及围绕快节奏战斗构建的视觉效果。 该套装的响亮部分针对BedWars和SkyWars 风格的小游戏战斗进行了优化，而另一套则旨在在大多数其他服务器上也能稳定运行。 两个客户端都不试图赢得对方的游戏。

如果你想要在混乱服务器上使用复制查找器和自动建造者，Wurst 是合适的工具，Opal 不属于这个讨论范畴。 如果你在小游戏大厅中战斗，反过来考虑这句话。

## 手工构建的菜单 vs 脚本引擎和 UI 运行时

Wurst 的 ClickGUI 和 Navigator 是手工构建的 OpenGL，直接覆盖在Minecraft自身的 GuiGraphics 之上。 没有嵌入式浏览器，没有第三方 UI 库，无需加载任何内容。 这是一个有意识的选择，它体积小且可预测，并且在过去十年里一直保持稳定，无需重建。 如果“自己阅读每一行代码”对你很重要，那么这种简单性值得真正的东西.

Opal 进一步深入堆栈：一个嵌入式的GraalJS脚本引擎允许用户在运行时加载和卸载 JavaScript，并与客户端的内部 API 交互；而用户界面本身就是一个托管在同一GraalJS运行时中的 React 风格层，使用 Yoga Flexbox 排列在自定义的 Blaze3D 效果管道之上。它是一个更重、更动画化的界面，注重精良的打磨，但代价是 Wurst 默认提供的简单、可审计的透明度。选择你的权衡.
