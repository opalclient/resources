## Fabric 模组 vs. 独立客户端：每个客户端实际上构建在什么基础上

Feather 独立客户端：没有 Fabric，没有 Forge，它根本不通过模组加载器注入。这是一种真正的架构选择，而不是脚注。独立客户端不受模组加载器跟上新版 Minecraft 发布速度的限制，这是一种值得称赞的真正优势。

Opal 采取相反的路径。它作为一个 Fabric 模组（Fabric Loader 0.19.3，Fabric API 0.150.0+26.2）发布，并包含在自己的启动器中，因此您永远不会直接接触 Fabric，但底层它依赖于 Fabric 生态系统： Mixins、访问扩展器、整个工具链。这使得 Opal 能够快速将 Minecraft 26.2 部署到 Java 25 上，因为它建立在积极维护的加载器之上，而不是从头开始重新实现客户端。 权衡是双向的：Opal 的更新速度快，因为它依赖 Fabric 来完成繁重的工作，并且它也受到 Fabric 发布节奏的限制。
