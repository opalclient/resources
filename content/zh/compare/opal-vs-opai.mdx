## 两种不同的选择

Opai 和 Opal 实际上并没有在同一条线上竞争。Opai 押注于广度和开放性：一个客户端覆盖了广泛的Minecraft版本，拥有封闭的核心以及围绕其开放扩展 SDK 和 GitHub 主题生态系统构建的社区插件。 Opal 押注于精选：更少的组件，更多由其决定，并被一个看起来真正设计过的界面所包裹。

## Opai 做得对的地方

功劳要归功于功劳。一个构建在封闭核心之上的开放 SDK，加上一个真正的 GitHub 主题插件市场，在这一领域中并不常见，而覆盖Minecraft历史更长时间的版本范围，远超大多数付费客户端所能达到的，这确实是一大优势。如果你想要一个能够跟随你跨版本的客户端，并且喜欢社区构建的插件，那么 Opai 正在做一件值得尊敬的事情。

## Opal 采取了另一种方式

Opal 采取相反的方式：一个精选的 70 个功能集，在您触碰任何设置菜单之前就已经过调优并启用，并通过一个自定义的玻璃 UI 引擎呈现。没有需要学习的 SDK，也没有需要组装的插件列表。您登录后，它已经成为了您想要的客户端，而不是您最终要构建的客户端的启动包.

## 一位客户，两份工作

Opal 是一种混合体。吵闹的配置，飞行、脚手架、激烈的战斗，专为 BedWars 和 SkyWars 风格的战斗而设计。安静的配置，微妙的瞄准、射程、速度，旨在在大多数其他服务器上保持冷静。你切换取决于你正在玩什么服务器，而不是你记得先安装哪个扩展。
