## 十年之差的 Minecraft 版本

Fan 运行在 MC 1.8.9 上。Opal v2 运行在 MC 26.2 上。这两个 Minecraft 版本相隔大约十年，为其中一个版本构建的客户端无法在期望运行另一个版本服务器上运行。这与其说是 UI 与 UI 的比较，或是功能与功能的比较，不如说是来自同一场景的不同时代的两个客户端。如果你因为某种原因仍然使用 1.8.9，那么 Opal 对你来说根本不是一个选项。

## 其中一个仍在更新

Fan 自己的记录将其列为已废弃，在 intent.store 市场上的活跃期大约是 2020 年至 2021 年。记录中的任何内容都没有表明在那个时间段之后还有持续的开发。Opal v2 正好相反：它是当前的、正在维护的 Opal 客户端，跟踪 MC 26.2，今年记录了更新。停止接收更新的客户端将无法跟上服务器端接下来所做的任何更改。这适用于任何客户端，而不仅仅是这对客户端。

## 模块数量和设计理念

范氏发布了 55 个模块，分布在七个类别下，使用单个作者的 `cedo/` 命名空间，主要归功于 Cedo。其中一些设计选择确实很巧妙：一个专用的“隐藏”类别，可以将某些模块从客户端数组列表 HUD 中移除；以及并排发布的双 Scaffold / BadScaffold 对，用于 A/B 测试绕过。它还同时运行 Click GUI 和 TabGUI，让用户可以选择自己的导航方式。这完全是一位开发者的设计思路，并非偶然。

Opal v2 包含 70 个模块，分为一组显眼的类别（飞行、脚手架、针对 BedWars/SkyWars 风格小游戏战斗进行优化的激进战斗）和一组构建用于在大多数服务器上运行的类别，以及一个嵌入的 GraalJS 脚本运行时，供任何想要扩展其功能的人使用。它是一个更大、更现代的功能集，围绕单个自定义玻璃 UI 引擎构建，而不是两个竞争的菜单，如果你不想自己选择的话。

## 你实际为金钱得到的

Opal v2 的价格为 $8.99，并通过启动器提供：登录并开始游戏，无需手动设置。范氏的记录中没有列出价格，只有提到它是在 intent.store 上售出的，而分发的源代码中没有任何构建配置，只有提取的快照和一个手写的 `Start.java` 入口点。对于任何研究客户端历史的人来说，这很有意思。如果你只是想今晚加载游戏，那就没那么有趣了.
