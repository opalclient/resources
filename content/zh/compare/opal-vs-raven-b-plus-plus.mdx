## 自由开源 vs $8.99 和闭源

Raven B++ 免费。它采用 GPL-3.0 许可，完整的源代码在 GitHub 上公开，作者为 K-ov，LycanLD 贡献了代码，您可以自由阅读、构建或分叉它。如果您想在运行模块之前准确了解其功能，或者仅仅不想为客户端付费，这绝对是一个优势。 功劳归于原作者：这是一种合法的运行客户端的方式，而不是妥协。

Opal 售价 $8.99 且为闭源。购买的是一个维护良好的启动器产品，而不是您需要自行编译的仓库：登录、启动、游玩。您为成品构建和背后的工作付出了费用，而不是为了阅读代码的能力。

## 1.8.9 模组 vs 为当前版本构建的客户端

Raven B++ 仅适用于 Forge 的 MC 1.8.9 版本。甚至在它的 build.gradle 文件中留下了注释掉的 1.8.8 构建路径，这表明该项目已经永久停留在那个时代。如果您在仍然运行 1.8.9 风格战斗的服务器上玩游戏，尤其是在具有 Hypixel 风格的排名 PvP 模式的服务器上，那么这个版本锁定是客户端的重点，而不是弱点。但这意味着 Raven B++ 无法在 MC 26.2 或为当前版本构建的任何其他版本上运行。

Opal 是为 MC 26.2 的 Fabric 版本构建的。如果您的服务器运行当前版本的游戏内容，那么 Raven B++ 根本不是一个选项：只有在您知道自己在哪个时代玩游戏时，这个比较才有意义.

## 更多原始模块与为特定目的构建的模块

Raven B++ 提供了大约 110 个模块类，涵盖战斗、移动、玩家、渲染以及 Hypixel 小游戏辅助等类别：例如，床战争叠加层、决斗统计、相扑围栏等。 在纸面上，这比 Opal 的 70 个模块多。 更多的模块听起来似乎意味着更多的价值，直到你询问每个模块实际上在做什么。

它的源代码注释诚实地回答了这个问题：客户端“不作任何特定的绕过声明”，不包含任何混淆，并且其 Disabler 模块存在，但没有任何声明的防作弊规避措施。 这不是什么贬义，而只是一个开放、未混淆的 1.8.9 模组的特性。 Opal 的 70 个模块主要围绕服务器端的防作弊机制构建，具有专门的禁用器和脚手架规避模式，并由一个封闭、积极维护的构建支持，而不是任何人都可以检查和反击的开放的 jar 文件。

## 自己安装模组与使用启动器

Raven B++ 要求你从 GitHub 构建或下载 Forge jar 文件，将其加载到 Forge 1.8.9-11.15.1.2318 中，并在运行时安装 Essential 模组。 这是标准的开源模组路径：获取 jar 文件，加载加载器，开始。 如果你以前做过，这只是一个五分钟的小任务。 如果你没有做过，可能需要一个晚上。

Opal 通过其自己的启动器分发：你登录，它会处理其余的事情。 这是购买 $8.99 的主要优势，相比于从 GitHub 拉取免费仓库.
