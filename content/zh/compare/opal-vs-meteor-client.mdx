## 自由开源与$8.99及闭源

Meteor Client 没有任何费用。它采用 GPL-3.0 许可协议，完整的源代码托管在 GitHub 上，用户名是 MeteorDevelopment，项目依靠捐赠而非销售来维持运营。如果你想阅读代码、进行分支，或者仅仅不想为一款 Minecraft 模组付费，这都是合理的理由，我们不会假装另有其意。

Opal 售价为 $8.99，源代码未公开。你所购买的是一个围绕启动器构建的成品：登录、启动、游玩，无需考虑 Fabric 安装或映射。你为组装和维护这些工作付出的费用，而不是为了获取查看源代码的选项。

## 针对不同场景：小游戏与混乱服务器

这是真正的分歧，值得直言不讳，而不是虚构一场不存在的竞争。Meteor Client 专为混乱服务器设计，例如 2b2t，这些服务器实际上没有反作弊机制，模块集侧重于自动化、基地寻找以及水晶或锚点战斗，而不是规避。根据其自身的范围，它不是排名的 PvP 或小游戏作弊工具，并且根据其自身的文档，Hypixel 风格服务器实际运行的服务器端反作弊系统可以轻易检测到它，因为那场战斗从未是重点.

Opal 的设计目标是完全相反的环境：BedWars/SkyWars 风格的小游戏战斗，它拥有为这种战斗而设计的丰富功能集，以及一套独立的、安静的功能集，旨在适应大多数服务器。这两个客户端并没有试图完成对方的任务，如果假装它们在做，那就低估了它们各自的能力。

## 模块数量：70 vs 约 150

Meteor Client 提供了约 150 个模块，涵盖战斗、移动、玩家、渲染和世界等类别，其中一部分数量专门用于混乱模式的自动化，而 Opal 根本不尝试这些功能：例如，道路建造者、基地定位的储藏箱查找器、多实例机器人协调。它还提供了一个记录完善的插件 API，因此，一旦社区插件生态系统被计算在内，您实际可以运行的模块数量上限会高于基础数量。值得称赞。

Opal 提供了 70 个模块，涵盖战斗、移动、视觉、世界和实用工具，设计上更加精简，专注于小游戏战斗、移动和安静功能，而不是混乱模式的自动化。模块更少，但每个模块都指向相同的目标.

## UI 界面与设置

Opal 运行着自定义界面：一个基于 Minecraft 的 Blaze3D 渲染管线（模糊、菲涅尔、折射、阴影效果）构建的自定义玻璃 UI 引擎，以及驱动动态岛屿叠加层等功能的 GraalJS/Yoga UI 运行时。它通过启动器到达，因此无需手动安装模组即可正常运行。

Meteor Client 的突出 UI 特色是其由 Starscript 驱动的 HUD，它允许您将实时值直接脚本到 HUD 文本中，而不是从固定的预设中选择。这确实是巧妙的设计。虽然它是一个 Fabric 模组，但设置意味着安装 Fabric Loader 并自行放置 jar 文件，这对于免费和开源来说是值得的，但 Opal 完全跳过了这一步。
