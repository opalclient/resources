Opal 提供了 GraalVM JavaScript 引擎，这是一个完整的 JS 运行时，运行在 Minecraft 客户端内部。您可以在游戏运行期间编写脚本，与 Opal 的内部 API 进行交互。这既不是宏录制器，也不是按键绑定系统。它是一个真正的编程环境，可以访问客户端的事件系统、模块配置和游戏状态.

<KeyTakeaways>

- 一个完整的 GraalVM JavaScript 运行时，而不是一个宏记录器。
- 响应游戏事件，更改模块设置，并绘制你自己的 HUD。
- 脚本在运行时加载，无需重启，并以沙盒模式运行在 Opal 的 API 中。
- 大多数其他客户端仅限于基本的宏.

</KeyTakeaways>

## 为什么选择 GraalVM？
GraalVM 是 Oracle 提供的多语言运行时。我们选择它的原因很具体：

- 速度：JIT 编译保持脚本足够快，可以在每个时钟周期运行。
- JavaScript：大多数人已经熟悉它，因此学习曲线很短。
- 线程安全：JS 到 Java 的绑定能够正确处理并发，而不是让您在其中绊倒。
- 熟悉模式：标准的 JavaScript API 表现方式符合您的预期
## 你能建造的东西

### 定制自动化
响应游戏内事件：

```javascript
// Auto-respond to certain chat messages
on('chat', (event) => {
  if (event.message.includes('1v1?')) {
    sendChat('/duel accept');
  }
});
```
### 动态模块配置
根据游戏状态更改模块设置：

```javascript
// Increase reach when health is low
on('tick', () => {
  const health = player.getHealth();
  if (health < 8) {
    modules.get('Reach').setSetting('distance', 3.3);
  } else {
    modules.get('Reach').setSetting('distance', 3.1);
  }
});
```
### 自定义 HUD 元素
绘制您自己的覆盖层：

```javascript
// Display a combo counter on screen
let combo = 0;
on('attackEntity', () => { combo++; });
on('damage', () => { combo = 0; });
on('render2d', (ctx) => {
  ctx.drawText(`Combo: ${combo}`, 10, 10, 0xFFFFFF);
});
```
### 服务器特定配置
加入不同的服务器时，自动切换配置：

```javascript
on('serverJoin', (event) => {
  if (event.ip.includes('pvp.example.com')) {
    loadConfig('server-bypass');
  } else {
    loadConfig('ghost-default');
  }
});
```
