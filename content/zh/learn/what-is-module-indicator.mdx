<KeyTakeaways>

- 模块指示器是一个紧凑的屏幕列表，显示在 Minecraft 实用工具客户端中当前启用的模块。
- 它实时更新：启用模块，模块名称就会出现；禁用模块，模块名称就会消失。
- 它完全在客户端渲染，从不与服务器联系，因此不存在防作弊风险。
- 它执行与数组列表相同的功能，只是作为一个更简单的模块单独构建，而不是共享代码。
- 它会出现在像 Fan 这样的客户端中.

</KeyTakeaways>

模块指示器是指通常停留在屏幕角落的简短文本列表，它会告诉你 Minecraft 工具客户端当前正在运行哪些模块。开启 Killaura 时，它的名称就会显示出来；关闭它，名称就会消失。这就是该功能的全部内容：一个显示你客户端状态的实时读数，仅此而已。它扮演着数组列表的角色，在那些同时提供这两者的客户端中，这两个通常被编写为独立的、独立编码的功能，而不是一个带有两种外观的列表。你会在像 Fan 这样的客户端中找到它，它会做与其名称相符的事情。

## 工作原理

每帧，客户端都会检查哪些模块当前被标记为启用，并绘制它们的名称，通常以简短列表的形式，固定在屏幕的角落。启用一个模块，就会添加一行；禁用它，该行会立即消失。这个机制就仅此而已。

由于该列表仅读取本地客户端状态并在游戏视图上绘制文本，因此它从不与服务器通信。没有数据包发送出去，没有游戏逻辑被触及，只是在你的现有操作之上添加了一个绘制调用。它属于与坐标 HUD 或 FPS 计数器相同的类别：完全本地化、完全装饰性，并且对任何没有查看你自己的屏幕的人来说都是不可见的。

## 模块指示器与数组列表

这里重叠并非偶然。这两个特性都旨在快速回答同一个问题：现在有什么？它们的区别在于范围，而非目的。

| | 模块指示器 | 数组列表 |
|---|---|---|
| 目的 | 列出当前活动的模块 | 相同：列出当前活动的模块 |
| 特性深度 | 仅显示名称的精简列表 | 通常更丰富，包含排序或样式选项等附加功能 |
| 构建方式 | 独立的简单模块，即使客户端同时发货这两个模块也单独编码 | 经常是更成熟的，“主要”版本 |
| 服务器交互 | 无 | 无 |
| 反作弊风险 | 无 | 无 |

客户端可以选择发货一个、另一个或两者并排。你最终看到的是哪个，或者是否将两者堆叠在角落里，取决于开发团队如何构建他们的 HUD，而不是你实际看到的功能差异。如果你想要更深入地了解功能更丰富的兄弟版本，请访问 [数组列表](/zh/learn/what-is-array-list) 页面.

<Callout variant="tip" title="Opal 在这方面的立场">
Opal 并没有提供名为“模块指示器”的模块。对于实用型客户端来说，一个基本的“当前显示模块”的读取信息是标配，而不是卖点，因此该模块是否存在并不重要，重要的是它实际显示的内容。把注意力放在模块本身上。
</Callout>

## 常见问题

<FAQList>

<FAQItem question="模块指示器会影响游戏体验吗？">
不会。它只在你的屏幕上显示文字，从不影响移动、战斗或其他任何游戏逻辑，因此对你的实际游戏体验没有任何影响。
</FAQItem>

<FAQItem question="反作弊系统能检测到模块指示器吗？">
不能。它从不向服务器发送任何信息，因此服务器端的反作弊系统根本无从观察或标记。
</FAQItem>

<FAQItem question="模块指示器和数组列表有什么区别？">
它们的作用相同，都是列出当前激活的模块，但模块指示器通常是更简单、独立编码的版本，而不是与数组列表共享实现。
</FAQItem>

<FAQItem question="客户为何会同时发送模块指示器和数组列表？">
通常是因为这两个功能在不同的时间点独立构建的，而不是合并成一个。这只是一个开发细节，不会影响您在屏幕上看到的内容。
</FAQItem>

<FAQItem question="模块指示器是否可见给其他玩家？">
否。它仅在您的客户端上呈现，从不发送到服务器或显示给游戏中的任何其他玩家。
</FAQItem>

</FAQList>
