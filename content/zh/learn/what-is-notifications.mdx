<KeyTakeaways>

- 通知模块在特定客户端事件触发时会显示一个简短的屏幕弹出提示，通常是模块的开启或关闭，或自动化模块（如宝箱窃取器）完成任务。
- 它采用事件驱动的方式，而非持续显示：提示出现后，会持续一到两秒钟，然后自动消失，与始终可见的 HUD 不同。
- 它仅在本地绘制。没有数据包，没有移动或战斗变化，没有任何内容供服务器端反作弊系统检测。
- 在像 Fan 和 Tenacity 这样的客户端中，它被视为一种提升用户体验的附加层，而非游戏玩法功能.

</KeyTakeaways>

通知模块是某些 Minecraft 实用工具客户端中的一个小型警报系统，每当发生特定事件时，都会在屏幕上弹出一个简短的弹出窗口，最常见的情况是模块的开启或关闭，或后台任务完成。它不会改变游戏玩法本身。把它当作确认，而不是自动化：那种在你同时处理六个模块时，无需停止检查，就能知道你刚刚切换的模块是否生效的功能。

## 工作原理

通知模块监听客户端内部的事件，而不是服务器上发生的任何事情。最常见的触发方式是热键或点击 GUI 切换：开启 Killaura 后，一个快速的横幅会确认其状态。另一种常见的触发方式是自动化模块按照自己的时间表完成任务，最典型的例子是 [箱子窃贼](/zh/learn/what-is-chest-stealer) 完成对你刚刚打开的箱子的扫描.

弹出窗口本身很小：通常是一行文字或一个紧凑的横幅，通常停留在屏幕的角落，在淡出之前显示一两秒。它的工作方式与原版 Minecraft 的吐司消息相同（例如，进度弹出窗口、食谱解锁横幅），只是将其重新利用于客户端事件，而不是原版事件。重要的是它不做什么：显示通知对它所报告的事件没有任何影响。切换已经发生。弹出窗口只是事后告诉你这件事。

## 通知与 HUD 模块

很容易将此归类到 HUD 中，但两者解决不同的问题。HUD 持续显示在屏幕上，无论是否发生任何事情，都会显示持续的读数，例如坐标或 FPS。通知则相反：默认情况下不可见，只有在发生特定事件时才会显示。

| | 通知 | HUD |
|---|---|---|
| 默认可见 | 否，仅在事件发生时 | 是，持续显示 |
| 持续时间 | 一两秒，然后清除 | 保持直到取消选中 |
| 显示内容 | 单个事件（切换、任务完成） | 持续统计信息（坐标、FPS、战斗） |
| 服务器交互 | 无 | 无 |

像范和坚韧这样的客户都将它们一同发送，将它们视为同一类被动显示的不同部分，而不是一个覆盖另一个。 如果您想要那个组合的持续读数部分，请访问[抬头显示模块](/zh/learn/what-is-hud)页面。

## 是否安全使用

这几乎是客户功能所能达到的最安全的状态。 通知在本地绘制并在此停止。 没有额外的数据包发送，没有任何关于移动或战斗的信息，并且没有输入被自动化。 服务器端的反作弊系统评估您的客户端发送的内容以及您的玩家的移动方式，而仅存在于您屏幕上的弹出窗口从未触及服务器端的任何数据。 没有旋转签名，没有时间提示，没有任何可以关联的内容。

唯一的例外不是检测风险，而是一种可见性问题。 由于弹出窗口直接在屏幕上渲染，因此任何观看该屏幕的人（例如，版主、屏幕共享、直播）都会看到它显示的内容。 了解这一点很重要，但它与反作弊系统如何评估玩家无关.
