<KeyTakeaways>

- ESP 2D 是一种视觉模块，它会在屏幕空间中为玩家和实体绘制一个扁平的框或标记，而不是在世界中包裹实体的 3D 轮廓。
- 这是两种常见的 ESP 渲染风格之一，另一种是发光 ESP；发货时同时包含这两种风格的客户端允许你进行选择。
- 屏幕空间绘制意味着标记不会被地形遮挡，并且无论距离如何，标记始终以一致的大小显示。
- 它会显示实体的位置，穿透墙壁，因此反作弊系统会将它视为墙hack级别的功能，无论哪种渲染风格绘制了该框.

</KeyTakeaways>

ESP 2D 是一种视觉模块，它会在你的屏幕上直接绘制一个扁平的方框、点或标签来标记其他玩家和实体，而不是在世界中实体的 3D 轮廓周围绘制。它是客户端渲染实体 ESP 的两种常见方式之一，另一种是发光 ESP，并且任何同时提供这两种方式的客户端都允许你选择更喜欢查看哪一种。无论如何，目的都是一样的：无需转动你的摄像头来查看，就能知道每个人的位置。

## ESP 2D 的渲染方式

ESP 2D 每一帧都会将每个被追踪实体的实际位置投影到你的屏幕空间，并在该点绘制一个标记，一个方框轮廓，一个点，或一个小标签。这与 3D 或“世界” ESP 不同，后者直接在实体本身上绘制轮廓，因此它会随着你的摄像头旋转而移动和缩放三维模型。2D 标记的行为更像一个悬挂在某人头顶上的标签：扁平、屏幕锁定，并且对实际模型下方角度或动画的变化不敏感.

由于绘制调用发生在屏幕空间而不是世界空间，一个 2D 框不会像世界空间轮廓有时会发生的那样，被墙壁或山坡遮挡，并且无论实体距离多远，它都可以在屏幕上保持恒定的尺寸。某些实现方式仍然会根据距离缩放框，因此，站在你旁边的目标看起来比地图另一端的目标更大。无论如何，底层实体位置驱动标记，因此，即使实体躲在岩石后面或你的相机处于不便的角度，读取结果仍然存在。

## ESP 2D 与发光 ESP

| 方面 | ESP 2D | 发光 ESP |
|---|---|---|
| 绘制位置 | 屏幕空间的扁平框、点或标签 | 实体周围的世界空间轮廓 |
| 地形遮挡 | 始终可见于墙壁和障碍物后 | 也能穿透地形，但显示为世界中的发光效果 |
| 远距离尺寸 | 通常恒定；某些版本会根据范围进行缩放 | 自然缩放，就像一个真实的在游戏效果一样 |
| 视觉读取 | 在整个地图上扫描时，统一且易于浏览 | 更接近正常发光药剂或幽灵效果的外观 |

这两种风格都使用相同的底层实体追踪。选择纯粹是关于易读性和沉浸感：2D 用于在任何距离下都提供干净、一致的 HUD 风格读取，而发光用于在不显得像 HUD 的情况下显示信息.

<Callout variant="tip" title="Opal 在此处的定位">
Opal 并未将实体 ESP 分割成 2D 切换和发光切换。它的目录列出了一个视觉类别模块，即实体 ESP，描述为在世界中突出显示玩家和实体。一个开关，在您能看到任何人之前无需做出任何渲染样式决策。
</Callout>

<Callout variant="warning" title="参考，非使用指南">
没有任何形式的实体 ESP 是安全的，无论是假设其不可检测的方框还是光晕。它会向您提供您不应拥有的信息，而无需实际的视觉接触，并且旨在检测 ESP 的反作弊系统通常不关心哪个形状绘制了标记。墙hack 和 X 光规则适用于 ESP 2D，与任何 3D 轮廓相同。在具有反作弊保护的服务器上将其视为自担风险使用。
</Callout>

## 常见问题

<FAQList>

<FAQItem question="ESP 2D 和 Glow ESP 有什么区别？">
ESP 2D 在屏幕空间中绘制一个扁平的框或标记，它不遵循实体形状。Glow ESP 则勾勒出游戏世界中的实体本身，因此它会随着相机旋转而移动和缩放，就像一个真实的、在世界中的效果一样。
</FAQItem>

<FAQItem question="ESP 2D 能否穿墙？">
是的。它会为每个被追踪的实体绘制标记，无论你和它们之间是否有墙壁或地形阻挡，这正是它成为墙hack级别功能的根本原因。
</FAQItem>

<FAQItem question="反作弊系统能否检测到特定的2D ESP？">
没有专门针对框形设计的检测方法。能够检测到 ESP 的反作弊系统通常并不关心它是否以 2D 框或 3D 轮廓的形式呈现。风险在于拥有并采取你不应该拥有的信息，而不是标记的几何形状。
</FAQItem>

<FAQItem question="哪些客户端包含 ESP 2D？">
它通常出现在像 Fan 和 Tenacity 这样的客户端中，通常与 Glow ESP 作为一个第二渲染选项并排存在，用于相同的实体数据。Opal 采取不同的方法，提供一个统一的实体 ESP 模块，而不是将这两种样式分开。
</FAQItem>

</FAQList>
