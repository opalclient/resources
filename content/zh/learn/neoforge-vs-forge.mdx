## 各自的特点

Forge 是最初的 Minecraft 模组加载器。它在 1.6 时代及之后的大部分大型模组包场景中运行，拥有基于注解的事件总线、有主观性的生命周期以及底层的 ASM 类转换器。

NeoForge 是 Forge 在 2023 年分叉的项目，当时 Forge 项目的领导层冲突达到高潮。它保留了相同的广泛架构（注册表、能力、事件总线），但采用社区治理，并拥有更开放的贡献模式。

## 分叉的原因

分叉是关于治理，而不是重写。一群 Forge 贡献者在对 Forge 的运行方式产生分歧后，迁移到了一个社区拥有的项目。由于 NeoForge 起源于 Forge 的代码库，这两个加载器共享大部分核心理念，熟悉 Forge 的模组作者会立即认出 NeoForge。

随着时间的推移，这两个项目逐渐分道扬镳。NeoForge 已经重构了 API 的部分内容，并提供了自己的约定，因此为当前 NeoForge 构建的模组不能保证在 Forge 上加载，反之亦然。

## 它们的不同之处

| | NeoForge | Forge |
| :--- | :--- | :--- |
| 起源 | 2023 年 Forge 社区分支 | 最初的加载器 |
| 治理 | 社区拥有，开放贡献 | 传统 Forge 项目领导 |
| 架构 | 相同基础（注册器、能力、事件总线） | 注册器、能力、事件总线 |
| 新版本重点 | 当前的 Minecraft 版本 | 承载着旧的模组场景 |
| 模组目录 | 在现代版本中不断增长 | 在旧版本中拥有丰富的目录 |

## 常见问题

<FAQList>

<FAQItem question="NeoForge 是否是 Forge 的替代品？">
在当前的 Minecraft 版本中，大多数社区将 NeoForge 视为 Forge 谱系模组的继任者。Forge 仍然存在并继续支持旧版本，因此两者尚未完全取代对方。
</FAQItem>

<FAQItem question="我可以在 NeoForge 上运行 Forge 模组吗？">
不可靠。两者有共同的根源，因此一些简单的模组可能可以运行，但 API 已经分歧。针对特定加载器开发的模组不能保证能在另一加载器上加载。请安装与您的加载器匹配的构建版本。
</FAQItem>

<FAQItem question="NeoForge 和 Fabric 是同一回事吗？">
不是。NeoForge 源自 Forge 系列，并共享其事件总线和注册模型。[Fabric](https://fabricmc.net/) 是一个独立的、更轻量级的加载器，拥有自己的基于 Mixin 的补丁模型。它们是不同的生态系统。
</FAQItem>

</FAQList>
