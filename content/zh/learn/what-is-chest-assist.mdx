<KeyTakeaways>
- ChestAssist 是一个胸部交互辅助模块，在 Fan 客户端中可见，它能让你更轻松地操作容器的物品栏，而无需为你自动操作。
- 它不是“箱子窃贼”（Chest Stealer），后者会不经任何手动操作即可自动清空箱子。
- 它本身不会移动物品或打开容器，因此几乎没有与自动化相关的检测风险。
- 除了名称和类别之外，其具体机制尚未公开记录。
</KeyTakeaways>

胸部辅助 (Xiōngbù fǔzhù) 是内置于某些 Minecraft 实用工具客户端中的一个胸部交互辅助模块，其中最显眼的是 Fan，它就位于一个完全自动化的模块“箱子窃取器” (Xiāngzi qiěqǔqì) 旁边。这个名称已经告诉你大部分你需要知道的内容：它会协助你处理一个箱子，而不是为你处理箱子。这个区别是模块作为独立条目存在而不是被整合到“箱子窃取器”中的全部原因。

## 工作原理

关于胸部辅助的机械细节很少，直接说明这一点比用华丽的辞藻来掩盖它更有价值。已确认的是它的类别和边界：它是一个面向玩家的便利功能，用于处理箱子的物品栏，并且它不会在未经你指示的情况下移动物品或打开容器。相比之下，“箱子窃取器” 会自行触发，并从头到尾清空一个容器，无需进一步点击。Fan 将两者作为单独的列表条目一同发布，本身就证明了胸部辅助比其邻居覆盖了更窄、更不自动化的箱子处理范围.

超越那个界限，具体实现可能会有所不同。客户端可以选择将 ChestAssist 构建成纯粹的视觉或 UI 辅助工具（突出显示有价值的物品、简化排序、清理交互），也可以悄悄地融入自动化步骤，使其更接近 Chest Stealer 的领域。实际上，Fan 的模块列表中名称及其与 Chest Stealer 的配对是已确认的事项。除此之外的一切都取决于具体的实现。

## 是否安全使用

在 Minecraft 反作弊系统中，检测通常针对服务器能够观察到的不可能或统计上不符合人类行为的事情：没有匹配客户端输入的物品移动、没有预期数据包的容器打开，或者发生的动作速度超过了人类能够实际点击的速度。一个仅辅助交互，从不代表你采取行动的模块，几乎可以肯定地避免了这种风险。

| | ChestAssist | Chest Stealer |
|---|---|---|
| 触发条件 | 仅在玩家输入时 | 自动 |
| 是否自行移动物品 | 否 | 是 |
| 检测特征 | 低，更接近 UI 便利工具 | 高，匹配自动化特征 |

这张表格并非保证。 “Assist” 只是一个标签，而不是规范，客户端可以根据自己的意愿进行技术性实现。如果你正在评估某个特定的构建版本，请根据它实际的功能进行判断，而不是根据它的名称.

<Callout variant="tip" title="未包含在 Opal 的模块列表中，这是故意的">
Opal 没有提供 ChestAssist 模块，容器自动化通常也不是其模块列表关注的重点。Opal 的目录更倾向于战斗、移动和渲染。如果您专门需要胸部物品处理功能，Opal 与 Fan 这样的软件之间存在明显的差距，建议您在决定之前了解清楚。
</Callout>
