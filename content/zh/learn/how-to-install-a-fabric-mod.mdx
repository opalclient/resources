<KeyTakeaways>

- 安装 Fabric 模组有三个步骤：加载器、Fabric API，然后是模组 jar 文件。
- 每个组件（加载器、API、模组）都必须针对相同的 Minecraft 版本。
- 模组位于你的 Minecraft 目录下的 mods 文件夹中。
- 大多数安装失败的原因是版本不匹配或缺少 Fabric API.

</KeyTakeaways>

安装 [Fabric](https://fabricmc.net/) 模组需要三个步骤：为你的 Minecraft 版本的确切版本安装 Fabric 载入器，将 Fabric API 放入你的模组文件夹中，然后将模组的 jar 文件放在旁边。启动 Fabric 配置文件，模组就会加载。 大多数安装问题都源于版本不匹配，所以请确保所有组件都与相同的 Minecraft 版本相对应.

<Callout variant="info" title="安装 Opal">
Opal 的安装步骤与之相同：它是一个 Fabric 模组。有关确切版本和推荐的 JVM 参数，请参考[设置指南](/zh/docs/setup)。
</Callout>

## 三步

<Steps>

<Step number={1} title="安装 Fabric 载入器">为您的 Minecraft 版本安装 [Fabric](https://fabricmc.net/) 载入器，可以使用官方的 Fabric 安装程序或支持 Fabric 的启动器。请参阅 [Fabric 载入器安装指南](/zh/learn/how-to-install-fabric-loader)，以获取完整的安装步骤</Step>

<Step number={2} title="添加FabricAPI">
将 [Fabric API](/zh/learn/what-is-the-fabric-api) jar 文件放入你的 mods 文件夹中。 大多数模组都依赖于它。
</Step>

<Step number={3} title="添加模组">将模组的 jar 文件放在与 Fabric API 相同的 mods 文件夹中</Step>

</Steps>

然后启动 Fabric 档案，模组将在启动时加载。如果未出现，请查看以下故障排除列表。

## 模组文件夹的位置

| 操作系统 | 路径 |
| :--- | :--- |
| Windows | `%appdata%\.minecraft\mods` |
| macOS | `~/Library/Application Support/minecraft/mods` |
| Linux | `~/.minecraft/mods` |

如果 `mods` 文件夹尚未存在，请运行 Fabric 档案一次，它将为您自动创建。

## 模组无法加载时

<Checklist>
<ChecklistItem>模组未显示：模组文件位于错误文件夹中，或者您启动了非 Fabric 的配置文件.</ChecklistItem>
<ChecklistItem>启动时发生崩溃，缺少类错误：Fabric API 缺失或版本不正确.</ChecklistItem>
<ChecklistItem>版本不匹配：加载器、Fabric API 和模组都必须针对相同的 Minecraft 版本.</ChecklistItem>
</Checklist>
