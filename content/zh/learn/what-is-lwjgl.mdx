<KeyTakeaways>

- LWJGL 是轻量级 Java 游戏库，它是 Java 与原生图形、音频和输入之间的桥梁。
- 它是一系列精简的绑定（OpenGL、OpenAL、GLFW 等），而不是一个单独的库。
- Minecraft 的窗口、渲染和输入栈都建立在 LWJGL 之上。
- 它是库，而不是模组或模组加载器，而且 Minecraft 已经包含了它.

</KeyTakeaways>

LWJGL 是轻量级 Java 游戏库。它是连接 Java 代码与图形、音频和输入原生 API 的桥梁。 像 Minecraft 这样的 Java 游戏，会使用 LWJGL 来与你的 GPU 和声卡通信。绘制自定义界面或播放声音的模组也会通过它访问相同的原生层.

</Callout>

## LWJGL 实际上做什么

LWJGL 为 Java 提供了对原生 C 库的精简、直接的封装。Java 本身无法调用 OpenGL 或 OpenAL；它们是用 C 编写的，并存在于操作系统或驱动程序中。LWJGL 提供了绑定代码和原生二进制文件，因此 Java 方法几乎可以与底层的 C 函数一一对应。

这个封装层是故意保持精简的。LWJGL 并没有用场景图或引擎来隐藏 OpenGL。您编写的图形代码会紧贴 API，这正是它在希望获得控制权和可预测性能的游戏中如此受欢迎的原因。

## 它绑定了什么

LWJGL 是一组绑定，而不是单个库。与 Minecraft 及其模组相关的组件包括：

| 绑定 | 用途 |
| :--- | :--- |
| OpenGL | 向屏幕绘制：游戏使用的渲染 API |
| OpenAL | 3D 位置音频 |
| GLFW | 窗口创建、游戏循环以及键盘/鼠标输入 |
| stb | 图像和字体加载辅助工具 |
| NanoVG | 抗锯齿 2D 矢量绘图，某些 UI 代码使用 |

每个绑定都是可选的。项目只需引入其需要的模块。

## 为什么 Minecraft 及其模组依赖它

Minecraft 的整个窗口、渲染和输入堆栈都建立在 LWJGL 之上。当游戏打开一个窗口、读取鼠标或将三角形推送到 GPU 时，该调用会通过 LWJGL 传递到原生层.

模组会继承这个。那些绘制游戏内覆盖层或自定义菜单的模组，会使用游戏通过 LWJGL 设置的同一个 OpenGL 上下文。一些模组也使用 [NanoVG](/zh/learn/what-is-nanovg) 来绘制界面，而 NanoVG 本身就是 LWJGL 的一个绑定。这种共享的基础是模组能够与游戏一起渲染，而无需建立自己的图形系统的一部分原因。

## LWJGL 是模组吗？

不是。LWJGL 是一个库，而不是模组，也不是模组加载器。你不会像安装模组那样安装它。Minecraft 已经捆绑了它所需的版本，并且启动器会下载适用于你的操作系统的匹配的本地二进制文件。模组开发者会针对 LWJGL 进行编译，但很少会附带自己的副本。
