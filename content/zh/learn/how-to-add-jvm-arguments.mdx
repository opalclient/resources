<KeyTakeaways>

- JVM 参数存在于启动器的配置文件中，而不是游戏内选项中。
- 它们配置了托管游戏的 [Java 运行时](/zh/learn/what-is-the-jvm)，因此一个打字错误会影响启动，而不是你的世界。
- 大多数玩家想要的标志是 `-Xmx`，它限制了游戏可以使用的内存量。
- 对于大多数设置，4 到 6 千兆字节就足够了。 更多可能会使卡顿更严重.

</KeyTakeaways>

你可以在启动器的配置文件的设置中为 Minecraft 添加 JVM 参数。打开你用来启动的配置文件，找到 JVM 参数字段，并编辑该字段中的空格分隔的标志列表。这些标志配置了[Java 虚拟机](/zh/learn/what-is-the-jvm)，而不是游戏本身，因此一个打字错误会影响启动，而不会影响你的世界.

</Callout>

## JVM 参数是什么
JVM 参数是在游戏启动时传递给 Java 运行时环境的标志。启动器每次按下“开始”按钮时都会构建一个很长的命令行，而 JVM 参数是该命令行中用于调整运行时的部分。最常见的参数之一是设置游戏可以使用的内存量。

这些标志不是 Minecraft 的设置。它们由 JVM 在游戏加载之前读取，这就是为什么你需要在启动器中设置它们，而不是在游戏内的选项菜单中设置
## 把它们放在哪里？
几乎所有的启动器都在每个配置中保留一个 JVM 参数字段。标签可能有所不同，但位置是始终如一的。

| 启动器类型 | 字段位置 |
| :--- | :--- |
| 官方启动器 | 安装，编辑配置，更多选项，JVM 参数 |
| 第三方启动器 | 配置或实例设置，通常在 Java 或 高级 选项卡下 |
| 命令行 / 脚本 | 你运行的 `java` 命令，在 `-jar` 之前 |

编辑现有的列表，而不是替换它。启动器通常会提供合理的默认设置，清除它们可能会导致启动失败

## 多数人希望的参数：堆大小

大多数玩家关注的标志是 `-Xmx`，它限制了 JVM 可以使用的最大堆大小。`-Xmx4G` 允许最多 4 千兆字节。 还有 `-Xms`，它是起始堆。

```
-Xmx4G -Xms2G
```

两条规则保证了安全。 不要分配超过你拥有的可用空间，因为 Java 进程需要喘息空间，操作系统需要剩余的空间。 更多并不总是更好； 非常大的堆可以使垃圾回收暂停变得更糟，而不是更小。 对于大多数设置，4 到 6 千兆字节就足够了。

## 常见的参数

- `-Xmx<size>` 设置最大堆，例如 `-Xmx4G`。
- `-Xms<size>` 设置起始堆，例如 `-Xms2G`。
- `-XX:+UseG1GC` 选择 G1 垃圾回收器，这是一个常见的默认设置。
- `-Dproperty=value` 设置游戏或模组在启动时读取的系统属性。

你很少需要超过堆参数。 从旧指南中复制粘贴的冗长参数列表通常会比帮助更多，尤其是在现代 Java 运行时中。

## 当出现问题时

<Checklist>
<ChecklistItem>编辑后游戏无法启动：标志中存在打字错误。恢复默认参数，然后一次添加更改.</ChecklistItem>
<ChecklistItem>`-Xmx` 参数会被忽略或报错：单位或语法错误。请使用 `G` 或 `M`，不带空格，例如 `-Xmx4G`，而不是 `-Xmx 4 GB`.</ChecklistItem>
<ChecklistItem>卡顿问题在增加内存后加剧：过大的堆空间可能会延长垃圾回收暂停时间。将 `-Xmx` 参数值调低至 4 到 6 千兆字节.</ChecklistItem>
</Checklist>

## 常见问题

<FAQList>

<FAQItem question="在 Minecraft 中，-Xmx 是做什么用的？">
它设置了 Java 运行时允许使用的最大堆大小。`-Xmx4G` 将游戏限制在 4 千兆字节。这是每个“分配更多 RAM”指南背后的标志。
</FAQItem>

<FAQItem question="我应该给 Minecraft 分配多少 RAM？">
对于大多数玩家来说，4 到 6 GB 足够了。分配过多的内存很少有帮助，反而可能导致垃圾回收暂停时间更长，所以不要将所有内存都分配给 JVM。
</FAQItem>

<FAQItem question="模组会改变我需要的 JVM 参数吗？">
通常不会。模组在现有的 Java 进程中加载，因此它们共享您已经设置的堆。少数模组会记录特定的系统属性，但堆标志通常涵盖了大多数情况。
</FAQItem>

<FAQItem question="JVM参数和游戏设置是一样的吗？">
不是的。JVM参数是在游戏加载前配置Java运行时环境的，它们存在于启动器中。游戏设置则存在于游戏内的选项菜单中。
</FAQItem>

</FAQList>
