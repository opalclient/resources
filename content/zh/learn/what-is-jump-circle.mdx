<KeyTakeaways>
- 跳跃圆圈是一个外观模块，会在玩家每次跳跃时，在玩家脚下的地面上绘制一个圆环。
- 它仅改变本地渲染，因此跳跃高度、时机和物理效果保持纯粹原版。
- 服务器无法察觉这个效果，意味着零风险被检测到的可能性。
- 除非其他玩家也运行相同的模块，否则他们将看不到它。
</KeyTakeaways>

跳跃圆圈是一个化妆品级别的 Minecraft 客户端模块，每次玩家跳跃时，它会在玩家脚下绘制一个环形或圆圈。它仅改变本地屏幕上显示的内容：不影响移动、碰撞检测或服务器处理的任何内容。你可以把它归类为粒子轨迹或自定义披风，是一种视觉上的点缀，而不是游戏功能。

## 工作原理

跳跃圆圈会监听玩家的跳跃动作，并在跳跃发生的那一刻，在玩家脚下渲染一个环形或圆圈形状。这个效果仅在客户端绘制，叠加在正常的场景几何体之上，并在短时间内淡出。跳跃本身没有任何改变：高度、时机和物理效果都与原版游戏完全一致，服务器看到的跳跃数据包也与模块关闭时相同。

由于这个环形只是渲染上的添加，而不是对游戏逻辑实际依赖的任何值的更改，因此它不会与碰撞、碰撞箱或任何其他玩家客户端正在计算的内容发生交互。它的存在是为了被看见，而不是被处理.

吸引力很简单：在每次跳跃时添加一个环，就能为朴素的移动动作增添一些亮点，就像人们选择自定义动画套装或粒子轨迹的原因一样。 这种细节在短视频中表现良好，可见的跳跃提示能增加一些朴素版本所不具备的精致感。 将其发售的客户，例如Tenacity，将其与其他外观显示选项分组，而不是将其与移动或战斗模块分组。

## 跳跃圆圈 vs. 粒子轨迹

两者都是装饰性的，都与移动相关，并且经常被归类到同一个视觉设置选项卡中。 区别在于它们的触发时机。

| | 跳跃圆圈 | 粒子轨迹 |
|---|---|---|
| 触发 | 在跳跃动作时触发 | 在移动时持续运行 |
| 持续时间 | 简短，着陆后淡出 | 持久，跟随每一步 |
| 类别 | 视觉 / 外观 | 视觉 / 外观 |
| 对他人可见 | 否，仅本地渲染 | 否，仅本地渲染 |
| 影响游戏玩法 | 无 | 无 |

两者都不影响服务器跟踪的任何内容。 它们之间的选择纯粹是美学上的：持续的亮点与每次跳跃的标点符号.

<Callout variant="tip" title="Opal 的界限">
Opal 并没有以“跳跃圈模块”这样的名称提供。但它所代表的分离，即装饰效果被保留在独立的类别中，与任何影响移动或战斗的元素分开，正是 Opal 组织其视觉类别的方式：你可以随心所欲地设计外观，而不会悄无声息地改变游戏方式。
</Callout>
