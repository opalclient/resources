<KeyTakeaways>

- Fabric 是性能和实用模组偏爱的轻量级、快速更新的加载器。
- Forge 是较旧、更重的加载器，拥有最丰富的大型内容模组目录。
- 如果你需要早期版本支持和较小的占用空间，请选择 Fabric；如果需要大型模组包，请选择 Forge。
- 模组通常针对一个加载器，因此请根据你运行的加载器来选择模组.

</KeyTakeaways>

Fabric 是一个轻量级、运行快速的模组加载器，能够在几天内支持新的 Minecraft 版本，并且是性能和实用型模组的首选。Forge 是一个较旧、更重的加载器，拥有最丰富的大型内容模组目录。如果追求速度和早期版本支持，请选择 Fabric；如果需要大型模组包和只在该平台上发布的模组，请选择 Forge.

## 它们各自是什么

Fabric 是一个围绕小型核心以及独立版本发布的 [Fabric API](/zh/learn/what-is-the-fabric-api) 构建的模块化模组加载器。它以快速更新到新的 Minecraft 版本以及干净的、基于 Mixin 的补丁模型而闻名。

Forge 是一个历史悠久的加载器，多年来为大多数大型模组包提供动力。它自带广泛的 API 表面，这使得构建大型内容模组更容易，但加载器更新每个版本的速度较慢。

## 它们有何不同

| | Fabric | Forge |
| :--- | :--- | :--- |
| 体积 | 轻量级核心 + Fabric API | 庞大，内置大型 API |
| 新版本支持 | 版本发布后的几天 | 数周到数月 |
| 擅长 | 性能 + 实用模组 | 大型内容模组 + 模组包 |
| 补丁模型 | Mixins | 核心模组及其自己的钩子 |
| 模组间兼容性 | 每个模组的 API | 广泛的共享 API |

## 如何选择

- 如果您希望尽早获得最新的 Minecraft 版本、运行性能模组或希望尽可能小的占用空间，请选择 Fabric。
- 如果您最喜欢的模组或模组包是 Forge 专属的，或者您希望有一个大型的共享 API 来构建，请选择 Forge。
- 许多玩家会同时保留两者，每个加载器一个配置文件，因为大多数模组针对其中一个。

## 常见问题解答

<FAQList>

<FAQItem question="Fabric 是否比 Forge 更快？">
加载器本身更轻量，而与它搭配的流行性能模组（Sodium，“Lithium”）运行在 Fabric 上。原始速度取决于你安装的模组，而不仅仅是加载器本身。
</FAQItem>

<FAQItem question="我可以在 Fabric 上运行 Forge 模组吗？">
不能直接运行。为一种加载器制作的模组无法在没有兼容层的情况下在另一种加载器上加载，因此在安装模组之前，请务必检查该模组的目标加载器。
</FAQItem>

<FAQItem question="哪个更新更快？">
Fabric 通常在 Minecraft 发布后的几天内即可使用。Forge 通常需要更长的时间，因为它拥有更大的 API 表面积。
</FAQItem>

</FAQList>
