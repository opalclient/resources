<KeyTakeaways>

- 一个HUD模块将实时信息、坐标、帧数和战斗统计数据直接叠加在游戏视图上，无需进入菜单即可查看。
- 它为只读模式：HUD从不自动执行操作或更改游戏机制，它仅改变玩家已可用的信息展示方式。
- 不同的客户端对其有不同的称呼。Juul称其为HUD，Fan和Tenacity则将相同的概念称为HUDMod。
- 由于它从不触及游戏逻辑，因此HUD的风险概况与Minecraft自身的F3调试屏幕大致相同：无风险.

</KeyTakeaways>

抬头显示模块是一种屏幕覆盖层，在您游戏时，它会显示实时统计数据，例如坐标、帧数或战斗信息，直接覆盖在游戏视图之上。无需打开菜单或点击任何内容。这些数字只是存在于那里，实时更新，就像原版 Minecraft 的界面元素无需请求输入即可显示在屏幕上一样。它是一种显示功能，与客户端交互式点击 GUI 设置菜单是完全不同的模块类型。

## 工作原理

抬头显示模块每帧渲染其元素，叠加在正常的游戏视图之上，就像原版 Minecraft 的快捷栏或饥饿条一样。它不依赖于任何菜单的打开。无论您移动、战斗还是建造，信息都会保持可见，并以游戏其余部分渲染的速度刷新。

实际显示的内容因客户端而异，但常见的包括坐标、帧数计数器以及玩家通常需要以其他方式（例如暂停并打开 F3）检查的与战斗相关的信息。这些都不是新的数据。抬头显示模块只是获取游戏已经公开的数字，并将它们放在您无需停止游戏即可查看的地方.

这同样是它与点击式 GUI 的区别所在。点击式 GUI 是您打开以浏览和配置客户端模块的交互式菜单，它接收输入，并且需要被打开。HUD 则完全不需要输入。它绘制信息并退居幕后，在普通游戏背景下持续运行，而不是作为您需要导航的屏幕。

## HUD 与 点击式 GUI

| | HUD | 点击式 GUI |
|---|---|---|
| 接收输入 | 否 | 是 |
| 游戏期间可见 | 默认情况下始终可见 | 仅在打开时可见 |
| 目的 | 显示实时统计数据 | 配置模块和设置 |
| 最接近原版等效项 | F3 调试屏幕 | 选项 / 暂停菜单 |

便利性方面的优势显而易见。始终保持坐标、FPS 或战斗信息在视野中，可以避免您反复打开菜单或调试屏幕来检查相同的数据，尤其是在战斗中，将这些信息始终显示在屏幕上意味着您无需花费时间从实际威胁的攻击中分散注意力.

<Callout variant="tip" title="在 Opal 中的位置">
Opal 目前没有以“HUD”这个名称发布模块。覆盖式信息显示是实用程序客户端视觉类别中的常见功能，因此如果您以持续的统计信息显示为决定因素，最好直接检查任何客户端的当前模块列表，而不是假设命名一致。
</Callout>

<FAQItem question="HUD 模块是否被视为作弊行为？ Minecraft?">
不。HUD 模块仅显示信息、坐标、帧数、战斗数据等。它不会自动执行任何操作，也不会改变游戏的行为，因此它所带来的风险与其他情况无异 Minecraft'他自己的 F3 调试屏幕.

Juul 将其版本称为 HUD。Fan 和 Tenacity 以 HUDMod 的名称发布了相同的想法。名称不同，功能相同。

</FAQItem>
