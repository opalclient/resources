## LiteLoader 是什么

LiteLoader 是一款刻意保持小巧的模组加载器。它专注于客户端实用型模组，并未试图成为一个完整的游戏内容模组平台。其理念很简单：一个轻量级的层，用于加载无需大型 API 也能运行的轻量级模组。

正是这种专注使其变得有用。一个 minimap、聊天调整或小型 HUD 模组不需要大型内容模组所依赖的复杂机制，因此更纤细的加载器更适合。

## 它从何而来

LiteLoader 诞生于 2010 年代初到中期，那时客户端调整正在迅速传播。它通过启动 tweaker 插入游戏，与其他同期加载器采用的通用方法相同，在游戏启动前插入自身。

一个关键特征是它可以与内容加载器共存。玩家可以在一个加载器中运行大型内容模组，同时在同一实例中通过 LiteLoader 加载小型客户端模组，这使得它成为常见的伴侣而非竞争对手。

## 它擅长什么

LiteLoader 在纯客户端模组方面表现最佳。因为它本身不携带大型 API，因此在处理简单任务时保持轻量级和快速设置.

| 强度 | 实际意义 |
| :--- | :--- |
| 轻量级核心 | 加载速度快，占用空间小 |
| 客户端重点 | 专为 HUD、UI 调整和小型工具设计 |
| 共存 | 能够与内容加载器并行运行 |
| 基于 Tweaker 的启动 | 像同类产品一样在游戏启动时进行挂钩 |

## 衰落的原因

LiteLoader 衰落的原因有两个。首先，当时的主流内容加载器不断吸收了模组玩家，许多玩家都选择通过它来运行所有内容，而不是管理第二个加载器。其次，当 [Fabric](https://fabricmc.net/) 出现时，它为较新的 Minecraft 版本提供了轻量级、快速更新的选择，覆盖了 LiteLoader 吸引人的“小而快速”的利基市场。

较新的 Minecraft 版本从未获得与旧版本相同的 LiteLoader 支持，因此它的活跃窗口随着它所服务的旧版本而关闭。今天，它最好被理解为模组加载器历史的一部分，是客户端侧调整需要自己轻量级环境的时刻的快照。

## 常见问题

<FAQList>

<FAQItem question="LiteLoader 还在使用吗？">
在现代的 Minecraft 中，几乎无人再有意义地使用它。LiteLoader 属于旧版本时代；现在的配置通常使用 [Fabric](https://fabricmc.net/) 或 Forge。
</FAQItem>

<FAQItem question="LiteLoader 的用途是什么？">
小型客户端修改，例如 HUD、界面调整和一些小工具，而不是添加方块、物品或生物的大型内容修改。
</FAQItem>

<FAQItem question="LiteLoader 与内容加载器有什么不同？">
它有意保持轻量级，没有附带大型 API。内容加载器更重，旨在支持大型模组，而 LiteLoader 则专注于小规模的内容，并且可以与一个模组并行运行。
</FAQItem>

<FAQItem question="人们为什么停止使用 LiteLoader？">
主要内容加载器保留了大部分用户，而 [Fabric](https://fabricmc.net/) 后来填补了较新版本中轻量级加载器的需求，因此没有必要保留一个独立的、体积较小的加载器了。
</FAQItem>

</FAQList>
