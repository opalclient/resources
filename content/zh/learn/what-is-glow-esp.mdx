Glow ESP 是一款 Minecraft 视觉模块，它会用纯色勾勒出玩家和生物的轮廓，并保持这些轮廓在墙壁、地板以及任何障碍物之间可见。它的渲染方式与原版 Minecraft 中“发光”状态效果相同，就像幽灵箭矢施放的闪光效果，只不过它会持续地应用于你周围的所有相关实体，而不是仅针对一个目标持续几秒钟。客户端通常会附带第二种样式——一个扁平的 2D 方框叠加层，并允许你选择哪种样式在战斗中反应更快。无论如何，结果都是一样的：你能在看到他们之前就知道他们在哪里。

## 工作原理

Glow ESP 会用彩色轮廓渲染每个目标实体，这个轮廓会跳过深度测试，因此即使墙壁、地板或另一个生物在你和目标之间，它也能持续绘制。原版 Minecraft 已经内置了这种渲染方式，用于“发光”效果，这就是为什么这种外观如此熟悉的原因：该模块只是根据需要将轮廓应用于实体，而不是等待游戏通过正常方式授予它。

大多数实现方式允许轮廓颜色根据实体类型或团队而变化，这样玩家和敌对生物就不会在第一眼就被误判为相同的威胁。而且，由于轮廓会跟踪实体的实际模型，而不是一个扁平的方框，它会随着目标移动和旋转而移动，而不是仅仅跟随其坐标。

## Glow ESP 与 ESP 2D

Glow ESP 和 ESP 2D 解决相同的问题，即透过地形看到实体，但渲染方式不同。

| | Glow ESP | ESP 2D |
|---|---|---|
| 渲染方式 | 沿着实体 3D 模型绘制轮廓 | 在实体屏幕位置绘制一个扁平矩形 |
| 行为更像 | 贴附在目标形状上的游戏内高亮 | 位于游戏顶部的 UI 标记 |
| 包含在内 | Tenacity (选择其中一种风格) | Tenacity (选择其中一种风格) |

两者都不会在网络上传输额外数据，因此选择取决于个人喜好。如果想要轮廓和旋转，就选择轮廓；如果想要一瞥屏幕就能立即捕捉到的东西，就选择方框。

## 为什么会被标记

Glow ESP 仅更改本地渲染内容。它不会发送额外数据包、改变移动方式或影响战斗时机，因此不会像 [Reach](/zh/learn/what-is-reach) 这样的模块那样，它会更改客户端实际声称的内容，从而产生数据包级别的特征.

检测最终归结于行为模式。追踪墙后目标、预瞄拐角或在敌人进入视野前做出反应的玩家，会给反作弊软件和工作人员留下值得关注的模式。这种检测方式依赖于观察你的行为，而非拦截数据包，这正是 Glow ESP 始终保持“自担风险使用”状态的原因：它在网络上传输的数据量比战斗作弊程序更少，但并非对审查录像的人员不可见.

## 常见问题

<FAQList>
<FAQItem question="Glow ESP 实际显示什么？">
每个透过墙壁可见的实体周围都会显示一个彩色轮廓，其形状与该实体的模型相匹配。它不显示物品、背包或超出位置和轮廓的任何信息。
</FAQItem>
<FAQItem question="Glow ESP 与原版游戏中的发光效果相同吗？">
渲染路径与原版游戏中使用来显示发光状态效果的路径相同，例如时钟上附带的幽灵箭头的发光效果。Glow ESP 只是将该轮廓直接应用于实体，而无需任何游戏内效果。
</FAQItem>
<FAQItem question="Glow ESP 比 Killaura 或 Reach 更安全吗？">
它不修改移动、旋转或攻击数据包，因此可以避免那些反作弊系统用于检测战斗作弊的底层数据包检查。但如果您的行为明显依赖于您不应拥有的信息，仍然可能被标记。
</FAQItem>
<FAQItem question="Glow ESP 会导致封禁吗？">
是的，有可能。与战斗模块相比，它风险较低，因为它从不更改客户端发送的内容，但服务器或工作人员可能会监视那些对不可见目标做出一致反应的玩家，从而发现它。
</FAQItem>
</FAQList>
