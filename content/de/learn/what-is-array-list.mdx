<KeyTakeaways>

- Array List ist eine HUD-Überlagerung, die eine Liste der aktuell aktiven Module eines Clients in einer Ecke des Bildschirms anzeigt und sich live aktualisiert, während Module ein- und ausgeschaltet werden.
- Sie wird ausschließlich clientseitig gerendert, berührt also niemals den Server und Anticheat-Systeme können sie nicht erkennen.
- Einige Clients erlauben es, Module versteckt laufen zu lassen: aktiv, aber nicht in der Liste angezeigt.
- In Clients wie Fan, Tenacity und Rose zu finden.

</KeyTakeaways>

Die Array List ist ein HUD-Element in Minecraft-Utility-Clients, das den Namen jedes aktuell aktivierten Moduls anzeigt, typischerweise gestapelt in einer Ecke des Bildschirms. Wechselt man ein Modul um, erscheint sein Name; schaltet man es ab, verschwindet es wieder, ohne dass ein Menü erforderlich ist. Sie findet sich in Clients wie Fan, Tenacity und Rose, und obwohl Position und Gestaltung sich unterscheiden, bleibt die Funktion überall gleich: eine konstante, auf einen Blick erfassbare Anzeige, welche Module tatsächlich laufen.

## Wie es funktioniert

Jeden Frame überprüft der Client, welche Module als aktiviert markiert sind, und zeichnet ihre Namen als einen kurzen Textstapel, der an einer Ecke des Bildschirms verankert ist. Aktiviert man ein Modul, wird eine Zeile hinzugefügt; deaktiviert man es, verschwindet die Zeile. Das ist die gesamte Mechanik. Die Liste wird lokal über die Spielansicht gezeichnet, berührt den Server nicht, sendet keine Pakete und hat keinen Einfluss darauf, wie das Spiel tatsächlich abläuft. Es ist eine Anzeige, keine Aktion.

Der Reiz liegt hauptsächlich in der Bequemlichkeit. Sobald ein Client mehr als eine Handvoll Module mitbringt, wird es unzuverlässig, sich zu merken, welche Module aktiviert sind, und das wiederholte Öffnen einer vollständigen Klick-GUI nur um nachzusehen, wird schnell lästig. Ein Blick in eine Ecke des Bildschirms löst das Problem in einer halben Sekunde.

Einige Kunden kombinieren die Liste mit einer Kategorie für versteckte Module. Ein so markiertes Modul läuft weiterhin wie gewohnt, wird aber nicht in der gerenderten Ausgabe angezeigt. Das bedeutet, dass die Array-Liste auf dem Bildschirm nicht immer ein vollständiges Inventar dessen darstellt, was aktiv ist, was vor allem für das Aussehen eines Bildschirms für einen Betrachter wichtiger ist als für Vorgänge auf Serverseite.

## Ist es sicher, es zu verwenden?

Die Array-Liste selbst birgt keine der Risiken, die mit Modulen verbunden sind, die tatsächlich die Spielweise verändern. Sie rendert Text lokal und kommuniziert niemals mit dem Server, sodass ein serverseitiges Anti-Cheat-System nichts zu beobachten hat, geschweige denn etwas zu markieren. Vergleichen Sie das mit etwas wie einem Aim-Bot oder Reichweite, bei dem der Server Winkel und Entfernungen von Treffern messen und die Lücke zwischen dem, was physikalisch möglich ist, und dem, was gerade passiert ist, erkennen kann. Es gibt hier kein äquivalentes Signal, da es überhaupt keine Serverinteraktion gibt.

Die Kombination mit versteckten Modulen ist eine andere Betrachtungsweise. Ihr Zweck ist es, Informationen vor einer Person zu verbergen, die den Bildschirm betrachtet, vor allem bei einem Bildschirmfreigabe-Vorgang, anstatt ein Erkennungssystem zu umgehen. Es ist wichtig zu wissen, wenn man eine Array-Liste als vollständiges Bild davon betrachtet, was läuft: Sie ist es möglicherweise nicht.
