## Was sie ausmacht

Fabric ist ein leichtgewichtiges, modulares Mod-Loader-System, das von FabricMC gepflegt wird. Es kombiniert einen kleinen Kern mit der separat versionierten [Fabric API](/de/learn/what-is-the-fabric-api) und verwendet ein Mixin-basiertes Patching-Modell. Es aktualisiert sich schnell auf neue Minecraft-Versionen und verfügt über den umfangreichsten Katalog an Performance- und Utility-Mods.

Quilt ist eine Abspaltung von Fabric, die 2021 nach Governance-Streitigkeiten innerhalb von FabricMC entstand. Es führt Fabric-Mods unverändert aus, liefert seine eigene Quilted Fabric API als Fabric API-Kompatibilitätsschicht mit und fügt QSL, die Quilt Standard Libraries, für seine eigenen APIs hinzu. Seine Verbreitung ist im Vergleich zu Fabric gering.

## Worin sie sich unterscheiden

Der große praktische Punkt ist die Richtung der Kompatibilität. Quilt kann Fabric-Mods laden, aber Fabric kann keine Mods laden, die von Quilt-spezifischen QSL-Funktionen abhängen. Ein für reines Fabric geschriebener Mod erreicht also beide Zielgruppen, während ein QSL-abhängiger Mod nur Quilt-Benutzer erreicht.

| | Fabric | Quilt |
| :--- | :--- | :--- |
| Ursprung | Ursprünglicher Loader von FabricMC | 2021 Community-Fork von Fabric |
| Mod-Ökosystem | Größtes für moderne Minecraft | Lädt Fabric-Mods plus QSL-Mods |
| Eigene API | Fabric API | Quilted Fabric API (Kompatibilität) + QSL |
| Kompatibilität | Kann Quilt-spezifische Mods nicht laden | Lädt Fabric-Mods unverändert |
| Verbreitung | Sehr groß | Gering im Vergleich zu Fabric |
| Patching-Modell | Mixins | Mixins (von Fabric geerbt) |

## Welche Wahl treffen

<Checklist>

<ChecklistItem>Wähle Quilt, wenn du seine QSL-Funktionen nutzen möchtest, einen Quilt-spezifischen Mod ausführen oder die Governance der Abzweigung bevorzugst. Deine bestehenden Fabric-Mods funktionieren weiterhin.</ChecklistItem>

## Häufig gestellte Fragen

<FAQList>

<FAQItem question="Ist Quilt mit Fabric-Mods kompatibel?">
Ja. Quilt behält die binäre Kompatibilität bei und lädt Fabric-Mods über seine Quilted Fabric API-Schicht. Das Gegenteil ist nicht der Fall: Fabric kann keine Mods ausführen, die Quilt-spezifische QSL-APIs benötigen.
</FAQItem>

<FAQItem question="Warum existiert Quilt, wenn Fabric bereits funktioniert?">
Quilt spaltete sich 2021 aufgrund von Meinungsverschiedenheiten über die Leitung von FabricMC ab. Es zielt darauf ab, mit Fabric kompatibel zu bleiben und gleichzeitig durch QSL ein eigenes API-Design anzubieten.
</FAQItem>

<FAQItem question="Sollte ein Utility-Client Fabric oder Quilt anvisieren?">
Die meisten entscheiden sich für Fabric, da dessen Nutzerbasis und Versionsunterstützung größer sind, und ein Fabric-Build auch auf Quilt läuft. Ein Quilt-only-Build würde die Zielgruppe in den meisten Fällen ohne Vorteil verkleinern.
</FAQItem>

<FAQItem question="Ist eines schneller als das andere?">
Es gibt keinen nennenswerten Unterschied. Quilt erbt das Mixin-Modell von Fabric, daher hängt die Leistung von den installierten Mods ab, nicht vom Loader.
</FAQItem>

</FAQList>
