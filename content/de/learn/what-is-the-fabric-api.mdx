<KeyTakeaways>

- Die Fabric API ist eine gemeinsam genutzte Bibliothek aus Hooks und Hilfsfunktionen, auf die die meisten Fabric-Mods aufbauen.
- Sie ist aus Gründen der Trennung vom Loader separat: Ein kleiner Loader lässt sich schnell aktualisieren, die API entwickelt sich unabhängig weiter.
- Sie bietet Events, Netzwerkfunktionen, ein Registry und Rendering-Hilfsfunktionen, auf die viele Mods angewiesen sind.
- Eine fehlende oder veraltete Fabric API ist die übliche Ursache dafür, dass ein Mod beim Starten abstürzt.

</KeyTakeaways>

Die Fabric API ist eine gemeinsam genutzte Bibliothek aus Hooks und Hilfsmitteln, auf die die meisten [Fabric](https://fabricmc.net/) Mods basieren. Der Kern-Loader von Fabric ist absichtlich klein gehalten, sodass die API die Lücken füllt: Ereignisse, Netzwerkfunktionen, Registry-Hilfen, Rendering-Dienstprogramme. Wenn ein Fabric Mod beim Start aufgrund einer fehlenden Klasse abstürzt, ist in der Regel eine fehlende oder veraltete Fabric API die Ursache.

## Warum es vom Loader getrennt ist

Fabric teilt das System absichtlich in zwei Teile auf. Der Loader bleibt klein und stabil, und die Fabric API liefert den größeren, sich schneller bewegenden Satz gemeinsamer Schnittstellen. Diese Trennung ermöglicht es dem Loader, schnell auf eine neue Minecraft-Version zu aktualisieren, während sich die API in ihrem eigenen Rhythmus weiterentwickelt.

## Was es bietet

Die API bietet Mods eine gemeinsame, versionsstabile Möglichkeit, die Dinge zu tun, die viele Mods benötigen:

- Lebenszyklus- und Spielereignisse, an die man sich anhaken kann.
- Netzwerk-Helfer für benutzerdefinierte Pakete.
- Registry- und Inhalts-Helfer zum Hinzufügen von Items, Blöcken und Entitäten.
- Rendering- und Ressourcen-Helfer, die von Mods gemeinsam genutzt werden.

Ohne sie würde jeder Mod diese Teile neu erfinden und sich gegenseitig unterbrechen.

## Benötigen Sie es?

Fast immer, ja. Die meisten Fabric-Mods listen die Fabric API als Abhängigkeit auf, sodass Sie sie einmal zusammen mit den anderen als Mod installieren. Passen Sie ihre Version an Ihre Minecraft-Version an.

## FAQ

<FAQList>

<FAQItem question="Ist Fabric erforderlich?">
Für die meisten Mods ja. Ein Mod, der davon abhängt, lädt ohne Fabric nicht. Einige eigenständige Mods benötigen es nicht, aber die Installation ist die sichere Standardeinstellung.
</FAQItem>

<FAQItem question="Wo bekomme ich es?">
Über die offiziellen Fabric-Kanäle und bei den üblichen Mod-Hosts. Achten Sie immer darauf, die Version zu wählen, die zu Ihrer Minecraft-Version passt.
</FAQItem>

<FAQItem question="Warum ist es nicht im Loader integriert?">
Damit der Loader klein bleibt und sich schnell aktualisieren kann, während die gemeinsame API separat weiterentwickelt wird. Dies ist eine bewusste Designentscheidung in Fabric.
</FAQItem>

</FAQList>
