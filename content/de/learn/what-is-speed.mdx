<KeyTakeaways>

- Geschwindigkeit ist ein Bewegungsmodul, das die Geh- und Sprintgeschwindigkeit über den festen Wert in Minecraft Vanilla hinaus erhöht.
- Es überschreibt direkt die Bewegungsberechnungen des Clients, ohne dass ein Trank oder ein Block für die zusätzliche Geschwindigkeit verantwortlich ist.
- Der Boost wirkt sowohl beim Gehen als auch beim Sprinten, solange das Modul aktiviert ist.
- Bewegungsprüfungen gehören zu den ältesten Werkzeugen der Anti-Cheat-Software, weshalb das Speed-Modul so oft aufgespürt wird.

</KeyTakeaways>

Geschwindigkeit ist ein Bewegungsmodul, das die Geh- und Laufgeschwindigkeit eines Spielers über das hinaus erhöht, was Vanilla Minecraft erlaubt. Aktiviere es, und du überquerst das Terrain schneller als ein unveränderter Client es je könnte, ohne dass im Spielverlauf eine Erklärung für das zusätzliche Tempo vorhanden ist. Es erscheint in Clients wie Fan und Tenacity, und Server behandeln es wie jede andere Bewegungs-Cheaterie: als einen unfairen Vorteil, nicht als eine Einstellung.

## Funktionsweise

Vanilla Minecraft berechnet für jeden Spieler eine feste Gehgeschwindigkeit und eine separate, schnellere Laufgeschwindigkeit. Normalerweise greifen nur Dinge wie eine Trank der Schnelligkeit oder das Stehen auf einem geschwindigkeitsbeeinflussenden Block in diese Werte ein. Ein Speed-Modul überspringt all das und überschreibt die Berechnung direkt, indem es sie erhöht, sodass der Spieler pro Tick mehr Fläche zurücklegt, sowohl beim Gehen als auch beim Laufen, ohne Trank, Block oder andere Vanilla-Erklärung.

Da sich die Änderung vollständig auf dem Client-Rechner vollzieht, zeigt der Bildschirm des Spielers eine flüssige, schnellere Bewegung, ohne Ruckler oder Stottern, die es visuell verraten würden. Was der Server empfängt, ist eine andere Geschichte: ein Strom von Positionsaktualisierungen, die pro Tick mehr Strecke zurücklegen, als es der bekannte Zustand des Spielers zulassen würde. Diese Lücke zwischen dem, was plausibel ist, und dem, was ankommt, ist genau das, worauf eine Bewegungsprüfung ausgelegt ist, um es zu bemerken.

## Warum es entdeckt wird

Geschwindigkeit ist einer der einfacheren Tricks, den ein Server aufdecken kann, da der Server bereits weiß, wie schnell ein Spieler sich bewegen darf, unabhängig davon, was der Client behauptet.

| Faktor, den die Anticheat-Software berücksichtigt | Warum das wichtig ist |
|---|---|
| Distanz, die pro Tick relativ zur bekannten Basisgeschwindigkeit des Spielers zurückgelegt wird | Direkter Beweis für eine Bewegung, die schneller ist, als das Spiel zulässt |
| Aktive Trankeffekte und geschwindigkeitsbeeinflussende Blöcke | Legitimitätsgründe, warum ein Spieler schnell sein kann, die der Server zuerst ausschließen muss |
| Sprintstatus und Richtungsänderungen | Sowohl Sprinten als auch Drehen beeinflussen, welche Geschwindigkeit der Server erwartet |
| Konsistenz über die Zeit | Ein einzelner schneller Tick kann Lag sein; anhaltend hohe Geschwindigkeit über die Grenze ohne Erklärung ist ein stärkeres Signal |

Server-seitige Bewegungsprüfungen gehören zu den ältesten und ausgereiftesten Teilen des Anticheat-Designs. Das Ausnutzen der Geschwindigkeit über einen längeren Zeitraum auf einem Server mit einer kompetenten Prüfung wird kaum unbemerkt bleiben.

## Häufig gestellte Fragen

<FAQList>

<FAQItem question="Ist Geschwindigkeit dasselbe wie ein Trankeffekt?">
Nein. Ein regulärer Trank der Schnelligkeit ist ein temporärer, serverseitiger Effekt. Ein Geschwindigkeitsmodul verändert die Bewegung hingegen auf Klientebene, sodass der Server Geschwindigkeit ohne Trank oder andere reguläre Quelle wahrnimmt.
</FAQItem>

<FAQItem question="Beeinflusst Geschwindigkeit Gehen, Sprinten oder beides?">
Beides. Das Modul erhöht die Rate sowohl beim Gehen als auch beim Sprinten, da beide aus derselben Bewegungsberechnung resultieren, die sie überschreibt.
</FAQItem>

<FAQItem question="Warum ist Speed einfacher zu fangen als andere Module?">
Bewegung wird vom Server bereits für jeden Spieler in jedem Takt erfasst, unabhängig davon, was der Client behauptet. Überhöhte Geschwindigkeit ist daher ein zuverlässigeres Signal als bei Betrugsversuchen, die auf subtile Timing- oder Rotationsinformationen angewiesen sind.
</FAQItem>

<FAQItem question="Ist Speed auf einem Server mit Anticheat-Schutz sicher nutzbar?">
Nein. Bewegungsdaten sind eines der zuverlässigsten Signale, die ein Server hat, daher riskiert ein Account, der Speed auf einem Server mit einer funktionierenden Bewegungsprüfung verwendet, eine Sperrung oder einen Bann.
</FAQItem>

</FAQList>
