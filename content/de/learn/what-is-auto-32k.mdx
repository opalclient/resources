- Der zugrunde liegende Fehler missbraucht die Stapelung von Verzauberungen, um Schaden zu verursachen, der weit über das hinausgeht, was ein normales Schwert bewirken kann, und das Modul automatisiert lediglich die Reproduktion.

Auto 32k ist ein Exploit-Modul, das den "32k"-Enchanted-Sword-Damage-Exploit automatisiert, ein Stück Anarchie-Server-Lore aus der Minecraft 1.12.2-Ära, bei dem eine Waffe mit gestapelten Verzauberungen Schaden verursacht, der weit über das hinausgeht, was die normalen Survival-Regeln erlauben. Der Name bezieht sich auf den Exploit selbst, nicht auf eine Einstellung oder einen Schwellenwert, den man anpassen kann. Das Modul entfernt lediglich den manuellen Aufwand, um ihn zu reproduzieren.

## Wie der zugrunde liegende Exploit funktioniert

Der Exploit resultiert aus dem Ausnutzen von Verzauberungsstapelung: Eine Waffe trägt weit mehr Verzauberungsstärke, als eine einzelne Waffe unter normalen Survival-Regeln tragen sollte. Ein normales, verzaubertes Schwert verursacht einen bescheidenen, vorhersehbaren Schadenszuwachs. Die gestapelte Version hingegen nicht. Sie verursacht bei einem einzigen Treffer um ein Vielfaches höheren Schaden, eine Zahl, für die es im Spiel keine legitime Erklärung gibt.

Auto 32k erfindet keinen neuen Trick zusätzlich dazu. Es automatisiert den bestehenden. Anstatt dass ein Spieler die Schritte manuell durchführt, um ein übermächtiges Schwert herzustellen, führt das Modul den Prozess für ihn aus, sodass er bei Bedarf wiederholt werden kann. Es ist als Exploit-Kategorie-Modul in SalHack dokumentiert, einem 1.12.2-Anarchie- und Crystal-Combat-Client, der genau auf diese Art von PvP ohne Regeln ausgelegt ist.

Der Reiz ist einfach: Anarchie-Server regeln Streitigkeiten durch rohe Kampfkraft, nicht durch Moderation, und ein Schwert, das Gegner mit einem Schlag ausschaltet, gewinnt direkt. Ein Treffer, der normalerweise nur einen kleinen Teil der Gesundheit eines Spielers reduziert, beendet den Kampf, bevor er überhaupt beginnt.

## Ist es sicher, es zu nutzen?

Nein, und zwar aus zwei Gründen, die beide unabhängig davon sind, welches Anticheat ein Server verwendet.

Erstens ist es von Natur aus fragil. Der Exploit funktioniert nur dort, wo der Fehler beim Stapeln von Verzauberungen aus der 1.12.2-Ära noch existiert. Patche den Fehler, aktualisiere den Server und Auto 32k erzeugt nichts Ungewöhnliches mehr. Es ist kein allgemeiner Kampfvorteil, sondern ein sehr spezifisches, sehr veraltetes Loch, das die meisten modernen Umgebungen bereits geschlossen haben.

Zweitens ist es, wenn es funktioniert, alles andere als unauffällig. Die Reichweite ist ein paar zusätzliche Blöcke, die niemand mitten im Kampf bemerkt. Ein Schwert, das beim einmaligen Schwung das mehrfache der normalen Schadens verursacht, ist dem Gegner und jedem, der den Kampf beobachtet, sofort offensichtlich. Die Automatisierung eines bekannten Schadensfehlers gilt außerdem gemäß den Regeln von praktisch jedem Server, der Regeln hat, als Fehlerausnutzung, unabhängig davon, welches Erkennungssystem aktiv ist oder nicht.

<Callout variant="warning" title="Ausnutzung eines Fehlers, kein Vorteil im Kampf">
Auto 32k automatisiert eine dokumentierte serverseitige Ausnutzung und nicht einen clientseitigen Rendering- oder Vorhersagetrick. Das Ausführen davon stellt eine Ausnutzung eines Fehlers auf jedem Server dar, der das Ausnutzen von Fehlern verbietet, und die Schadensausgabe ist für Gegner auch ohne das Eingreifen des Supports unverkennbar.
</Callout>

Opal versendet nichts Vergleichbares wie Auto 32k, und das ist kein Versehen. Es handelt sich um eine völlig andere Kategorie von Software: Die Automatisierung eines veralteten Schadensfehlers auf dem Server ist keine Kampfmechanik, sondern eine Belastung mit einer Abklingzeit, die durch die nächste Server-Patch-Version festgelegt wird. Opals Kampfmodule arbeiten auf der Client-Seite, lesen und prognostizieren, was das Spiel bereits erlaubt, anstatt umzuschreiben, wie viel Schaden ein Schwert anrichten kann.

## FAQ

<FAQList>
<FAQItem question="Was bedeutet „32k“ in Auto 32k?">
Es ist der Spitzname für den zugrunde liegenden Exploit für verzauberte-Schwert-Schaden, den das Modul automatisiert, der auf Anarchie-Server der Minecraft 1.12.2-Ära zurückgeht.
</FAQItem>
<FAQItem question="Funktioniert Auto 32k noch auf aktuellen Minecraft-Versionen?">
Nur wenn der Zielserver oder die Version immer noch den Stapel-Fehler bei Verzauberungen aufweist, von dem der Exploit abhängt. Es ist an die Mechaniken der 1.12.2-Ära gebunden, und überall dort, wo dieser Fehler behoben wurde, tut das Modul einfach nichts.
</FAQItem>
<FAQItem question="Ist Auto 32k ein heimlicher Cheat?">
Nein. Ein einziger Treffer, der einen Schaden verursacht, der weit über das hinausgeht, was ein legitimes verzaubertes Schwert erzeugen kann, ist für jeden im Kampf offensichtlich. Es liest sich wie ein brutaler Exploit, nicht wie ein versteckter Kampfvorteil.
</FAQItem>
<FAQItem question="Welcher Client dokumentiert Auto 32k?">
SalHack listet Auto 32k unter seinen Exploit-Modulen auf, zusammen mit anderen 1.12.2-Anarchie-Server-Exploits, die es mitliefert.
</FAQItem>
<FAQItem question="Ist die Verwendung von Auto 32k gegen die Serverregeln?">
Höchstwahrscheinlich. Die Automatisierung eines bekannten Schadens-Exploits gilt als Ausnutzung von Fehlern auf praktisch jedem Server mit Regeln gegen das Ausnutzen von Fehlern, unabhängig davon, ob dieser Server eine konventionelle Anti-Cheat-Software verwendet oder nicht.
</FAQItem>
</FAQList>
