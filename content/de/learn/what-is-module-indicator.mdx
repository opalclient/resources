<KeyTakeaways>

- Ein Modulindikator ist eine kompakte On-Screen-Liste, die anzeigt, welche Module in einem Minecraft-Utility-Client gerade aktiviert sind.
- Er aktualisiert sich live: Aktiviere ein Modul und sein Name erscheint, deaktiviere es und der Name verschwindet.
- Er wird vollständig clientseitig gerendert und kontaktiert niemals den Server, sodass er kein Anticheat-Risiko birgt.
- Er erledigt die gleiche Aufgabe wie eine Array-Liste, nur als ein einfacheres Modul, das separat erstellt wurde, anstatt Code zu teilen.
- Er erscheint in Clients wie Fan.

</KeyTakeaways>

Ein Modulindikator ist die kurze Textliste, die sich meist in einer Ecke des Bildschirms befindet und Ihnen anzeigt, welche Module aktuell in einem Minecraft-Utility-Client laufen. Schalten Sie Killaura ein, und sein Name erscheint. Schalten Sie es aus, und der Name verschwindet. Das ist die gesamte Funktion: eine Live-Anzeige Ihres Client-Zustands, nichts weiter. Er spielt die gleiche Rolle wie eine Array-Liste, und in Clients, die beide mitliefern, werden die beiden typischerweise als separate, unabhängig voneinander programmierte Funktionen und nicht als eine Liste mit zwei Skins geschrieben. Sie finden ihn in Clients wie Fan, die genau das tun, was der Name vermuten lässt.

## Funktionsweise

Jeden Frame überprüft der Client, welche Module aktuell als aktiviert markiert sind, und zeichnet ihre Namen als kurze Liste, die meist an einer Ecke des Bildschirms verankert ist. Aktivieren Sie ein Modul, und eine Zeile wird hinzugefügt. Deaktivieren Sie es, und die Zeile verschwindet sofort. An der Mechanik selbst gibt es nichts weiter.

Da die Liste nur den lokalen Client-Zustand ausliest und Text über die Spielansicht malt, kommuniziert sie niemals mit dem Server. Es werden keine Pakete gesendet, keine Spielmechanik wird verändert, sondern lediglich ein Draw-Call über das gelegt, was Sie bereits tun. Sie gehört in dieselben Bereich wie ein Koordinaten-HUD oder ein FPS-Zähler: vollständig lokal, vollständig kosmetisch und für jeden unsichtbar, der nicht gerade Ihren Bildschirm betrachtet.

## Modulindikator vs. Array-Liste

Die Überschneidung hier ist kein Zufall. Beide Funktionen existieren, um auf einen Blick dieselbe Frage zu beantworten: Was läuft gerade? Wo sie auseinandergehen, ist der Umfang, nicht der Zweck.

| | Modulindikator | Array-Liste |
|---|---|---|
| Zweck | Listet aktuell aktive Module auf | Gleiches: listet aktuell aktive Module auf |
| Funktionsumfang | Minimale, nur-Name-Liste | Oft umfangreicher, mit Extras wie Sortier- oder Styling-Optionen |
| Implementierungsansatz | Ein eigenes, einfaches Modul, separat programmiert, selbst wenn Kunden beide ausliefern | Häufig die weiterentwickelte, "primäre" Version der beiden |
| Serverinteraktion | Keine | Keine |
| Anticheat-Risiko | Keine | Keine |

Ein Kunde kann eines, das andere oder beide nebeneinander ausliefern. Welches Sie letztendlich betrachten oder ob Sie beide in einer Ecke gestapelt bekommen, hängt davon ab, wie das Entwicklungsteam ihr HUD strukturiert hat, und nicht von einem Unterschied darin, was Sie tatsächlich sehen. Wenn Sie die tiefere Ausführung des umfangreicher ausgestatteten Geschwisters wünschen, finden Sie diese auf der [Array-Liste](/de/learn/what-is-array-list) Seite.

<Callout variant="tip" title="Wie Opal dazu steht">
Opal versendet kein Modul namens Modulanzeige. Eine einfache Anzeige, was gerade läuft, ist Standard für einen Dienstprogramm-Client, kein Verkaufsargument. Ob ein solches Modul existiert, ist daher weniger wichtig als das, was es tatsächlich anzeigt. Konzentrieren Sie die Aufmerksamkeit auf die Module selbst.
</Callout>
