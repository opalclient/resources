<KeyTakeaways>

- Animationen sind ein kosmetisches Modul, das den Standard-Schwertschwung und die Standard-Gegenhaltungspose von Minecraft durch eine benutzerdefinierte ersetzt.
- Es ändert nur das, was lokal gerendert wird; Treffererkennung, Timing und Serverpakete bleiben identisch mit dem Originalzustand.
- Andere Spieler sehen nie deine benutzerdefinierten Animationen, sondern nur die Standard-Animationen.
- Es birgt kein Risiko einer Erkennung, da nichts an den Server gesendet wird, was sich ändert.

</KeyTakeaways>

Das Animations-Modul ist eine rein kosmetische Funktion, die die eingebauten Animationen von Minecraft ersetzt oder anpasst, am häufigsten den Schwertschlag und die Pose, die man beim Halten eines Gegenstands einnimmt, durch eine benutzerdefinierte Version. Es verändert, wie Aktionen auf dem Bildschirm aussehen, ohne die Treffererkennung, die Bewegung oder irgendetwas, was der Server tatsächlich verarbeitet, zu beeinflussen. Betrachten Sie es als eine Art Skin für Ihre Bewegungen, nicht als Vorteil im Kampf.

## Wie es funktioniert

Animations greift in die Client-Darstellung bestimmter Aktionen ein und ersetzt die Standardanimation durch eine benutzerdefinierte. Ein Schwertschlag kann anders gewölbt, zu einem anderen Zeitpunkt landen oder in einem anderen Winkel gehalten werden, je nach Preset. Das alles berührt nicht die zugrunde liegende Spielmechanik: dieselbe Interaktion wird immer noch ausgelöst, dasselbe Paket wird immer noch an den Server gesendet, zum gleichen Zeitpunkt, den der Server erwartet.

Das ist der ganze Trick. Das Modul berührt nur das, was gezeichnet wird, und muss daher nie mit Trefferfenstern, Angriffskühlzeiten oder irgendetwas anderem, was der Server im Auge behält, verhandeln. Sie sehen einen anders aussehenden Schwung. Der Server sieht einen gewöhnlichen Angriff, weil er genau das erhalten hat.

Clients, die es mitliefern, wie Fan, Tenacity und Rose, bündeln Animations in der Regel mit anderen Erscheinungsoptionen wie benutzerdefinierten HUDs, anstatt mit etwas, das die Art und Weise verändert, wie ein Kampf tatsächlich abläuft. Es gehört in dieselbe Kategorie von Funktionen wie die Auswahl eines Resource Packs, nur dass es über Code anstatt einer Texturdatei abgewickelt wird.

## Ist es sicher, zu verwenden?

Ja, und es gibt kaum Nuancen. Animationen ändern lediglich, was auf deinem eigenen Client gerendert wird. Sie verändern weder die Reichweite, das Timing noch einen Wert, den der Server zur Ausführung einer Aktion verwendet, sodass es serverseitig nichts zu markieren gibt. Die Animation selbst verlässt nie deinen Rechner; nur die zugrunde liegende Interaktion, die in genau der Form gesendet wird, wie sie auch in Vanilla gesendet würde, erreicht den Server.

Die einzige wirkliche Einschränkung: Einige Server verbieten Client-Modifikationen pauschal, unabhängig davon, was eine bestimmte Modifikation tatsächlich tut. Das ist eine Frage der Regeln, nicht der Erkennung, daher lohnt es sich immer noch, die Nutzungsbedingungen des Servers zu lesen, bevor du einen modifizierten Client ausführst.
