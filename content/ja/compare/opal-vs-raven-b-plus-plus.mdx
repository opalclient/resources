## フリーでオープンソース vs $8.99 とクローズド

Raven B++ は無料です。GPL-3.0 でライセンスされており、フルソースコードは GitHub で著者 K-ov と LycanLD の貢献のもと公開されています。ご自身でコードを読んだり、ビルドしたり、フォークしたりすることができます。モジュールが実際に何をしているか実行前に確認したい場合や、単にクライアントの費用を払いたくない場合に、これは真の利点です。 敬意を表して言いますが、これはクライアントを動作させる正当な方法であり、妥協ではありません。

Opal は $8.99 の費用がかかり、クローズドソースです。それによって得られるのは、自分でコンパイルするリポジトリではなく、維持されたランチャー製品です。ログインして、起動して、プレイするだけです。あなたは完成したビルドと、その背後にある作業に対してお金を払っているのです。コードを読む能力ではありません。

## 1.8.9 の MOD vs 現在のバージョン向けに構築されたクライアント

Raven B++ は MC 1.8.9 の Forge でのみ動作します。非アクティブな 1.8.8 のビルドパスは、build.gradle にコメントアウトされたまま残っており、プロジェクトがその時代に完全に定着していることを示す小さな兆候です。まだ 1.8.9 スタイルの戦闘を実行しているサーバー、特に Hypixel 風のランク PvP をプレイする場合、そのバージョンロックはクライアントの目的であり、弱点ではありません。しかし、それは Raven B++ が MC 26.2 や現在のリリース向けに構築されたものと動作しないことを意味します。

Opal は Fabric で MC 26.2 用に構築されています。サーバーが現在のバージョンのコンテンツを実行している場合、Raven B++ は単に選択肢になりません。この比較は、どの時代にプレイしているかを知ってから意味をなします.

## より汎用的なモジュール vs 目的のために構築されたモジュール

Raven B++ は、戦闘、移動、プレイヤー、レンダリング、Hypixel ミニゲームヘルパーのカテゴリーに、約 110 のモジュールクラスを搭載しています。具体的には、ベッドウォーズのオーバーレイ、デュエルの統計、相撲のフェンスなどが含まれます。これは、Opal の 70 個よりも多くのモジュールを搭載していることになります。モジュール数が多いことは、一見すると価値があるように思えますが、それぞれのモジュールが実際に何をしているのかを尋ねてみるとどうでしょうか。

そのソースコードの注釈は、正直に答えています。クライアントは「特定のバイパスに関する主張は一切行わない」、難読化は一切行わず、Disabler モジュールも、いかなるアンチチート回避機能も明示的に記載されていません。これは批判ではありません。単に、オープンで難読化されていない 1.8.9 の MOD であるという事実です。Opal の 70 個のモジュールは、主にサーバー側のアンチチートを中心に構築されており、専用のディセーブルモードと足場回避モードを備えています。これは、誰でも検査して反論できるオープンな JAR ではなく、クローズドで積極的にメンテナンスされているビルドによって裏付けられています。

## MOD を自分でインストールする vs ランチャーを使用する

Raven B++ では、GitHub から Forge の JAR をビルドまたはダウンロードし、Forge 1.8.9-11.15.1.2318 に対してロードし、実行時に Essential MOD をインストールする必要があります。これは、標準的なオープンソース MOD の手順です。JAR を取得し、ローダーをロードし、開始します。以前に行ったことがある場合は、5 分の作業で済みます。そうでない場合は、数時間かかるでしょう。

Opal は独自のランチャーを通じて配布されます。ログインすると、ランチャーが残りの処理をすべて処理します。これは、GitHub から無料のリポジトリをプルするよりも、$8.99 が提供する価値の大部分です.
