高いフレームレートと高品質なシェーダーを同時に手に入れたいですか？[Fabric](https://fabricmc.net) なら、必要な要素はシンプルです。Sodium でフレームを最適化し、Iris でシェーダーを適用し、Lithium でゲームロジックを処理します。すべて無料で、互いに連携するように設計されています。Opal ユーザーへの注意点として、Lithium は Opal と互換性がないため、Sodium と Iris を中心に構築してください.

<KeyTakeaways>

- Sodiumはフレームレートを、Irisはシェーダーを提供し、それらは一緒に動作するように設計されています。
- 「Lithium」はゲームロジックの助けになりますが、Opalとは互換性がありません。
- インストール順序：Fabricローダー、FabricAPI、そして残りの部分です。
- Opalは機能レイヤーであり、Sodiumがそれを実行するための余裕を与えます.

</KeyTakeaways>

## ナトリウム：フレームレート

[Sodium](https://modrinth.com/mod/sodium) は、Minecraft のレンダリングを高速かつスムーズに書き換えます。その差は、特に性能の低い GPU で顕著になります。現代のパフォーマンス設定の基盤となるものであり、Opal と並行して問題なく動作します。インストール後、Iris を使用してシェーダーを重ねてください。

## Iris：シェーダー

[Iris](https://modrinth.com/mod/iris) は、Fabric 上でシェーダーパックを実行するように設計されており、Sodium と連携することで、Sodium によって得られたフレームレートを消費することなくシェーダーを実行できます。従来の方法でシェーダーを実行していた場合、これが最新のエコシステムとの連携がスムーズな方法です。

## リチウム：ゲームロジック関連 (Opal とは互換性なし)

[Lithium](https://modrinth.com/mod/lithium) は、ゲームロジック、Mob の AI、物理演算、ティック処理を高速化します。見た目の変化はありません。バニラプラス設定に加えることで効果を発揮します。ただし、Opal とは互換性がないため、Opal を使用している場合は、インストールせずに Sodium に依存してください。

## 一目でわかる

| Mod | 役割 | Opal との互換性 |
| :--- | :--- | :--- |
| Sodium | レンダリング FPS | はい |
| Iris | シェーダー | はい |
| Fabric API | 必須の基盤 | はい |
| Lithium | ゲームロジックのパフォーマンス | いいえ |

<Callout variant="info" title="インストール順序">
Fabricローダーを最初に、次にFabricAPIをインストールし、その後、残りのものをインストールしてください。 ほとんどのMODはAPIがないとロードされません。そのため、Sodium、Iris、またはOpalの前にインストールする必要があります。
</Callout>

## Opalの配置場所

OpalはMinecraft 1.21.10用のFabricModです。そのため、SodiumやIrisと同じ`mods`フォルダに配置します。シェーダー対応ゲームに加え、戦闘、移動、視覚、ユーティリティのモジュール、ガラス製のUI、そしてスクリプトエンジンといったフルクライアント機能が利用できます。Opalは大規模なModなので、Sodiumを導入しておくと余裕があり、Lithiumは不要です。この構成で全てがうまく連携します。
