OpalはGraalVMJavaScript エンジンを搭載しており、これはMinecraftクライアント内で動作するフル JS ランタイムです。ゲーム実行中にOpalの内部 API と対話するスクリプトを作成できます。これはマクロレコーダーやキーバインドシステムではありません。クライアントのイベントシステム、モジュール設定、ゲーム状態にアクセスできる、本格的なプログラミング環境です.

<KeyTakeaways>

- フル GraalVM JavaScript ランタイムであり、マクロレコーダーではありません。
- ゲームイベントに反応し、モジュール設定を変更し、独自の HUD を描画できます。
- スクリプトは実行時にロードされ、再起動なしで、Opal の API に対してサンドボックス化されて実行されます。
- 他のほとんどのクライアントは基本的なマクロで止まります.

</KeyTakeaways>

## なぜGraalVM

GraalVMはOracleによるポリグロットランタイムです。具体的な理由から、採用しました。

- 速度: JITコンパイルにより、スクリプトは毎ティック実行するのに十分な速さを維持します。
- JavaScript: 多くの人がすでに知っているので、学習曲線は短いです。
- スレッドセーフティ: JSからJavaへのバインディングは、競合状態を適切に処理し、それに引っかかるのを防ぎます。
- 慣れ親しんだパターン: 標準的なJavaScript APIは、期待どおりに動作します。

## 構築できるもの

### カスタム自動化

ゲーム内イベントに応答します。

```javascript
// Auto-respond to certain chat messages
on('chat', (event) => {
  if (event.message.includes('1v1?')) {
    sendChat('/duel accept');
  }
});
```

### 動的なモジュール設定

ゲームの状態に基づいてモジュールの設定を変更します。

```javascript
// Increase reach when health is low
on('tick', () => {
  const health = player.getHealth();
  if (health < 8) {
    modules.get('Reach').setSetting('distance', 3.3);
  } else {
    modules.get('Reach').setSetting('distance', 3.1);
  }
});
```

### カスタムHUD要素

独自のオーバーレイを描画します。

```javascript
// Display a combo counter on screen
let combo = 0;
on('attackEntity', () => { combo++; });
on('damage', () => { combo = 0; });
on('render2d', (ctx) => {
  ctx.drawText(`Combo: ${combo}`, 10, 10, 0xFFFFFF);
});
```

### サーバー固有のプロファイル

異なるサーバーに参加すると、設定を自動的に切り替えます。

```javascript
on('serverJoin', (event) => {
  if (event.ip.includes('pvp.example.com')) {
    loadConfig('server-bypass');
  } else {
    loadConfig('ghost-default');
  }
});
```
