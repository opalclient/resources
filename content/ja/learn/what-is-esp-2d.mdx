<KeyTakeaways>

- ESP 2Dは、画面空間に描画された平面のボックスやマーカーでプレイヤーやエンティティを識別するビジュアルモジュールです。ワールド内のエンティティを包む3Dアウトラインとは異なります。
- これは、2つの一般的なESPレンダースタイルの一つで、もう一つはグローESPです。両方搭載しているクライアントでは、どちらかを選択できます。
- 画面空間への描画は、地形によってマーカーが隠されることがなく、距離に関わらず一貫したサイズで表示されることを意味します。
- 壁を越えてエンティティの位置を明らかにするため、どのレンダースタイルでボックスが描画されたかに関わらず、アンチチートシステムはこれを壁ハックと同等の機能として扱います.

</KeyTakeaways>

ESP 2Dは、他のプレイヤーやエンティティを、画面上に直接描画された平面のボックス、ドット、またはラベルでマークするビジュアルモジュールです。ワールド上に包み込まれた3Dアウトラインではなく、画面上に直接描画されます。エンティティESPのレンダー方法として一般的な2つの方法のうちの一つで、もう一つはグローESPです。両方を搭載しているクライアントでは、どちらを見たいかを選択できます。いずれの場合も目的は同じです。カメラを向けることなく、誰がどこにいるかを知ることです。

## ESP 2Dのレンダー方法

ESP 2Dは、追跡対象のエンティティの実際のポジションを毎フレーム画面空間に投影し、その点にマーカー、ボックスのアウトライン、ドット、または小さなラベルを描画します。これは、エンティティ自体に直接アウトラインを描画する3Dまたは「ワールド」ESPとは異なります。そのため、カメラが回転するにつれて、モデルの3次元的な動きやスケールに合わせて動きます。2Dマーカーは、誰かの頭上にピンで留められたネームタグのように振る舞います。平面で、画面にロックされており、その下にモデルがどのように角度付けまたはアニメーションされているかに無関係です.

描画コールがワールドスペースではなくスクリーン空間で行われるため、2Dボックスは壁や丘陵地によって遮られることはありません。ワールドスペースのアウトラインが時々そうなるのとは異なり、エンティティがどれだけ遠く離れていても、常に画面上の一定のサイズを維持できます。一部の実装では、ボックスを距離に応じてスケーリングすることもあります。その結果、あなたの隣にいるターゲットは、マップの反対側にいるターゲットよりも大きく見えます。いずれにせよ、エンティティの位置がマーカーを駆動するため、ターゲットが岩の陰に隠れたり、カメラが不自然な角度に移動したりしても、その表示は維持されます。

## ESP 2D vs グローESP

| 側面 | ESP 2D | グローESP |
|---|---|---|
| 描画場所 | スクリーン空間の平らなボックス、ドット、またはラベル | ワールドスペースのエンティティに巻き付いたアウトライン |
| 地形による遮蔽 | 壁やブロックを貫通して表示され続ける | 地形を貫通するが、ゲーム内のグローとして表示される |
| 距離でのサイズ | 多くの場合一定。一部のビルドでは、範囲に応じてスケーリングされる | 実際のゲーム内効果のように自然にスケーリングされる |
| 視覚的な読み取り | マップ全体で一貫性があり、スキャンしやすい | 通常のグローポーションやスペクトラルエフェクトのように見える |

どちらのスタイルも、同じエンティティトラッキングから派生しています。選択は純粋に可読性と没入感の問題です。2Dは、あらゆる距離でクリーンで一貫性のあるHUDスタイルの読み取りを実現し、グローは、HUDのように見えすぎることなく情報を表示したい場合に適しています.

<Callout variant="tip" title="どこ Opal ここに着陸">
Opal エンティティESPを2Dの切り替えとグローの切り替えに分割しません。そのカタログには、Entity ESPという1つのビジュアルカテゴリのモジュールが記載されており、世界中のプレイヤーやエンティティをハイライトするというシンプルな説明がされています。切り替えは1つだけで、誰かを視認する前にレンダースタイルに関する決定をする必要はありません.
</Callout>

<Callout variant="warning" title="参照、ハウツーではない">
いかなるエンティティESPも、検出不可能であると安全に仮定することはできません。箱や光であろうと関係ありません。視覚的な接触なしに、本来手に入れるべきではない情報をあなたに与え、ESPを検知するように設計されたアンチチートシステムは、どの形状がマーカーを描いたかを気にするものではありません。壁ハックやX線ハックのルールは、ESP 2Dにも3Dのアウトラインと同様に適用されます。アンチチート保護サーバーでは、自己責任で使用してください。
</Callout>

## よくある質問

<FAQList>

<FAQItem question="ESP 2DとGlow ESPの違いは何ですか？">
ESP 2Dは、エンティティの形状に沿わず、画面空間に平坦なボックスやマーカーを描画します。Glow ESPは、ゲーム内のエンティティそのものを輪郭で示し、カメラの動きに合わせてエンティティが移動し、スケールします。
</FAQItem>

<FAQItem question="ESP 2D は壁を貫通できますか？">
はい。追跡対象のエンティティごとにマーカーを描画します。壁や地形があなたとそれらの間に存在しても関係なく、まさにそれが壁ハック級の機能である理由です。
</FAQItem>

<FAQItem question="アンチチートは2DのESPを特定できますか？">
2Dボックスの形状に特化した検出方法は存在しません。一般的にESPを検知できるアンチチートシステムは、それが2Dボックスとしてレンダリングされたか、3Dのアウトラインとして表示されたかには関心がありません。リスクが生じるのは、本来持っているべきではない情報を取得し、それに基づいて行動することから、マーカーの形状自体ではありません。
</FAQItem>

<FAQItem question="ESP 2D はどのクライアントに含まれていますか？">
Fan や Tenacity などのクライアントで確認されており、通常、同じエンティティデータに対するセカンドレンダリングオプションとして Glow ESP の隣に表示されます。Opal は異なるアプローチを採用しており、2 つのスタイルを分割する代わりに、単一の統合された Entity ESP モジュールを搭載しています。
</FAQItem>

</FAQList>
