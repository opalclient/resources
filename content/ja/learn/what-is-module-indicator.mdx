<KeyTakeaways>

- モジュールインジケーターは、Minecraftユーティリティクライアントで現在有効になっているモジュールをコンパクトなオン画面リストで表示するものです。
- ライブで更新されます。モジュールを有効にすると名前が表示され、無効にすると名前が消えます。
- クライアント側でのみレンダリングされ、サーバーに一切連絡を取らないため、アンチチートのリスクはありません。
- アレイリストと同じ仕事をしますが、コードを共有するのではなく、個別に構築されたよりシンプルなモジュールとして機能します。
- Fanのようなクライアントに表示されます.

</KeyTakeaways>

モジュールインジケーターは、通常、画面の隅に表示される短いテキストリストで、Minecraftユーティリティクライアントで現在実行されているモジュールを何が動作しているかを示します。Killauraをオンにすると、その名前が表示されます。オフにすると、名前は消えます。これが機能全体です。自分のクライアントの状態をリアルタイムで読み取るもので、それ以上のものではありません。Array Listと同じ役割を果たし、両方搭載されているクライアントでは、通常、1つのリストに2つのスキンを持つのではなく、別々に、独立してコードされた機能として記述されます。Fanのようなクライアントでは、名前が示すとおり、まさにその機能を使用しています。

## 仕組み

毎フレーム、クライアントは現在有効になっているモジュールを確認し、その名前を短いリストとして描画します。通常、画面の隅に固定されています。モジュールを有効にすると、行が追加されます。無効にすると、その行はすぐに消えます。このメカニズムにはそれ以上のものはありません。

このリストはローカルクライアントの状態のみを読み取り、ゲームビューにテキストを描画するため、サーバーとは通信しません。パケットは送信されず、ゲームプレイのロジックは変更されません。単に、すでに実行しているものの上に描画されるdraw callです。座標HUDやFPSカウンターと同じカテゴリに属します。完全にローカルで、完全に装飾的で、自分の画面を見ていない人には見えません。

## モジュールインジケーターとArray Listの比較

この重複は偶然ではありません。どちらの機能も、一目で同じ質問に答えるために存在します。それは何か？今、何が動いているのか？両者が異なるのは、範囲であり、目的ではありません。

| | モジュールインジケーター | アレイリスト |
|---|---|---|
| 目的 | 現在アクティブなモジュールをリスト表示 | 同じ：現在アクティブなモジュールをリスト表示 |
| 機能の深さ | 最小限、名前のみのリスト | 頻繁により詳細で、ソートやスタイリングオプションなどの追加機能が含まれる |
| 構築アプローチ | 独自のシンプルなモジュールとして、両方を搭載するクライアントでも個別にコード化される | 頻繁により開発が進んだ、「主要な」バージョンである |
| サーバーとの連携 | なし | なし |
| チート対策リスク | なし | なし |

クライアントは、どちらか一方、または両方を並行して搭載できます。どちらの表示を見るか、あるいは両方が隅に積み重なって表示されるかどうかにかかわらず、実際に何をしているかという機能の違いではなく、開発チームがHUDをどのように構造化したかに依存します。より詳細な機能を持つ兄弟機能について知りたい場合は、[アレイリスト](/ja/learn/what-is-array-list)ページをご覧ください.

<Callout variant="tip" title="どこ Opal ここに立つ">
Opal モジュールインジケーターと呼ばれるモジュールは搭載されていません。ユーティリティクライアントにとって、「現在何がオンになっているか」という基本的な表示機能は当たり前のことで、セールスポイントではないため、それが存在するかどうかは、実際に何が表示されるかよりも重要です。モジュール本体に注意を向けてください.
</Callout>

## よくある質問

<FAQList>

<FAQItem question="モジュールインジケーターはゲームプレイに影響を与えますか？">
いいえ、影響はありません。それはあなた自身の画面にテキストを描画するだけで、移動、戦闘、その他のゲームロジックには一切関与しないため、実際にプレイする上では全く影響しません。
</FAQItem>

<FAQItem question="チート対策はモジュールインジケーターを検出できますか？">
いいえ、そのようなことはありません。サーバーに何も送信しないため、サーバー側のチート対策が観測したり、最初からフラグを立てたりするものは何もありません。
</FAQItem>

<FAQItem question="モジュールインジケーターとアレイリストの違いは何ですか？">
どちらもアクティブなモジュールをリスト表示するという同じ役割を果たしますが、モジュールインジケーターは通常、アレイリストとの実装を共有せず、よりシンプルな、個別にコード化されたバージョンです。
</FAQItem>

<FAQItem question="クライアントがモジュールインジケーターと配列リストの両方を送ってくるのはなぜか？">
通常は、この2つが別々の機能として、異なるタイミングで構築された結果です。開発上の詳細なことで、画面に表示されるものに影響はありません。
</FAQItem>

<FAQItem question="モジュールインジケーターは他のプレイヤーに表示されますか？">
いいえ。これはあなたのクライアントでのみ表示され、サーバーに送信されることも、ゲーム内の他の誰にも表示されることもありません。
</FAQItem>

</FAQList>
