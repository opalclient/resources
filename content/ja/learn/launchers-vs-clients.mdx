<KeyTakeaways>

- ランチャーは、ログインし、バージョンを選択し、ゲームを開始するプログラムです。
- クライアントは、ロードされたMODを含めて、動作中のゲームそのものです。
- ランチャーはゲームを始める前に動作し、クライアントはゲームをプレイ中に動作します。
- MODはクライアント内にロードされます。ランチャーは、プロファイルを選択するだけです.

</KeyTakeaways>

ランチャーは、バージョンを選択し、ログインし、Minecraftを開始するプログラムです。クライアントは、開始されてから実行される実際のゲームで、ロードされたMODを含みます。ランチャーは玄関、クライアントはその部屋です。両方を使用しますが、それぞれ異なる役割を果たします。

## ランチャーの役割

ランチャーは、ゲームを始める前に開く小さなアプリです。セットアップ作業を行い、その後ゲームに引き継ぎます。

ランチャーは通常、以下の処理を行います。

- Microsoftアカウントでのサインイン。
- 実行するMinecraftのバージョンを選択。
- そのバージョンに対応するゲームファイルとJavaをダウンロード。
- プロファイルまたはインストールを選択（[MODローダー](/ja/learn/what-is-a-minecraft-mod-loader)でセットアップされたものも含む）。
- メモリやその他の起動オプションを設定し、ゲームを開始。

「プレイ」ボタンを押すと、ランチャーはゲームを開始するコマンドを構築し、その後、邪魔をしません。ゲームをプレイしている間は動作していません。

## クライアントの役割

クライアントは、ゲームそのものです。見たり操作したりするMinecraftの実行インスタンスです。「バニラクライアント」とは、MODが適用されていないゲームのことです。MODを適用したクライアントは、[MODローダー](/ja/learn/what-is-a-minecraft-mod-loader)のような[Fabric](https://fabricmc.net/)を通じてMODがロードされた同じゲームです.

「ユーティリティクライアント」や「カスタムクライアント」という言葉が出てくる際、それは動作中のゲームを改変したものであり、別のランチャーではないという意味です。改造は、ゲームがすでに起動した後に機能を追加します。

## 連携方法

ランチャーはクライアントを起動します。これこそが両者の関係性です.

<Steps>

<Step number={1} title="ランチャーを開いて、サインインしてください">ランチャーを開き、アカウントでサインインします</Step>

<Step number={2} title="バージョンかプロファイルを選んでください">バージョンか、改造されたプロファイルを選択して実行します</Step>

<Step number={3} title="ゲームを開始する">ランチャーが必要なものをダウンロードし、ゲームを開始します</Step>

<Step number={4} title="顧客が走る">クライアントが実行されます。モッドローダーのプロファイルを選択した場合、モッドはロードされます</Step>

</Steps>

モッドローダーは、その中間に位置します。ご自身のバージョンにFabric（https://fabricmc.net/）をインストールすると、起動プロファイルが追加されます。その後、ランチャーがそのプロファイルを通してゲームを開始し、ローダーがクライアント起動時にモッドを読み込みます。ランチャーは、どのようなモッドを導入しているかを知ったり、気にする必要はなく、選択したバージョンを起動するだけです.

<Callout variant="info" title="どこ Opal 合う">Opalは[Fabric](https://fabricmc.net/)のModなので、特別なランチャーは必要ありません。普段お使いのランチャーにFabricのプロファイルを作成し、他のModと同様にOpalを追加してください。セットアップについては[セットアップガイド](/ja/docs/setup)を参照するか、ロードされたら[スクリプティング入門](/ja/docs/scripting)をご覧ください</Callout>

## ランチャーとクライアントの比較

| | ランチャー | クライアント |
| :--- | :--- | :--- |
| 概要 | ゲームを開始するアプリ | 実際に動作しているゲーム |
| 実行タイミング | プレイ前 | プレイ中 |
| 役割 | ログイン、バージョン選択、ゲーム起動 | ワールドのレンダリング、MODの実行 |
| MOD | MOD付きプロファイルの選択 | MODを実際にロードして実行 |
| 例 | 公式のMinecraftランチャー | バニラ版またはMOD入りゲーム |

## サードパーティ製のランチャーはクライアントの代わりになるのか？

公式ランチャーよりも多くの機能を提供するランチャーもあります。MODパックの管理、複数のバージョン間の切り替え、または[MODローダー](/ja/learn/what-is-a-minecraft-mod-loader)を同梱することで、手動設定をスキップできます。また、いくつかの人気クライアントは独自のランチャーを同梱している場合もあります。

しかし、この区別は変わりません。ランチャーの役割は依然としてゲームを開始することであり、クライアントが実行するものです。MODをインストールしてくれるランチャーは、単に事前にセットアップ段階を完了させているだけです。MOD自体は、クライアントが実行されてから初めて重要になります。

</FAQItem>

<FAQItem question="modは実際どこで実行されるのですか？ランチャーですか、クライアントですか？">

</FAQItem>

<FAQItem question="カスタムクライアントはランチャーの一種ですか？">
いいえ。カスタムクライアントやユーティリティクライアントは、MODローダーを通して読み込まれる改造ゲームです。ゲームを起動するランチャーの後に動作します。
</FAQItem>
