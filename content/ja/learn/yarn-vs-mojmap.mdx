<KeyTakeaways>

- Yarnは、Fabric向けのコミュニティ命名のオープンソースマッピングで、メソッドパラメータをフルにサポートしています。
- Mojmapは、Mojangによる公式名称で、1.14.4からすべてのリリースで公開されています。
- どちらも安定したレイヤー上に構築されているため、読みやすい名称はMODを壊すことなく変更可能です。
- 多くのFabricプロジェクトは、Mojmapをベースに構築し、その上にYarnのリマッピングを適用しています.

</KeyTakeaways>

Yarnは、[Fabric](https://fabricmc.net/)向けの、コミュニティが命名したオープンソースのマップです。記述的な名前と完全なメソッドパラメータが同梱されています。Mojmapは、Mojangが公式に発行している命名規則で、1.14.4からすべてのリリースで公開されています。無料で、十分にドキュメント化された命名規則が欲しい場合はYarn、Mojang自身の命名規則と一致し、変化しない名前が欲しい場合はMojmapを選んでください。

## それぞれの違い

Yarnは、FabricMCによって維持されているマッピングセットです。ボランティアがMinecraftの難読化されたクラス、フィールド、メソッドに読みやすい名前を割り当て、それらの名前は安定した[Intermediary](/ja/learn/what-is-intermediary)レイヤーの上に配置されるため、コンパイルされたMODを壊すことなく名前を変更できます。

Mojmapは、Mojangが1.14.4以降、すべてのリリースとともに公開している公式のマッピングテーブルです。名前はMojang自身のソースコードから直接取得されているため、開発者が内部的に使用している名前と一致します。MCPとSeargeを、新しいプロジェクトのデフォルトとして置き換えたものです。

## どのように異なるか

この違いは、誰が名前を選び、ライセンスがそれらをどのように扱うかという点に帰着します。Yarnの名前はコミュニティによって選択され、自由に利用および共有できます。Mojmapの名前はMojangのものであり、ライセンスは歴史的にコンパイル時利用に関しては寛容でしたが、再配布に関してはより慎重でした。最新のローダーはそれをあなたの代わりに処理します.

| | Yarn | Mojmap |
| :--- | :--- | :--- |
| 名前を付ける人 | FabricMCコミュニティ | Mojang (公式) |
| 最初に利用可能 | Fabric時代 | 1.14.4 以降 |
| メソッドのパラメータ | 名前とドキュメント付き | 頻繁に欠落 |
| ライセンス | オープン、共有可能 | コンパイル時に許可 |
| 下の安定レイヤー | 中間層 | リリースごとの難読化マップ |
| 最適 | ドキュメントを必要とする Fabric Mods | Mojang ソースに一致するプロジェクト |

## どちらを選ぶか

- Fabric をベースに構築し、説明的な名前、名前付きメソッドパラメータ、インラインドキュメントを希望する場合は、Yarn を選択してください。Fabric エコシステムにおける長年のデフォルトです。
- Mojang 自身のソースに一致する名前を希望する場合、他のローダーを使用する人々とコードを共有する予定がある場合、または公式マッピングが一般的なベースラインとなる 1.21 以降をターゲットとする場合は、Mojmap を選択してください。
- 多くの Fabric プロジェクトは現在、Mojmap をベースにして Yarn のリマッピングを適用しているため、必ずしもどちらか一方を選択する必要はありません。

## 他のマッピングとの関連性

どちらも、より長い歴史の中で存在しています。Mojang がマッピングを公開する以前は、モディングシーンでは MCP と Searge を使用して難読化された名前を逆展開していました。1.14.4 以降、Mojmap は公式の命名を提供し、Yarn は [Intermediary](/ja/learn/what-is-intermediary) の上に構築されたオープンソースの Fabric 代替として成長しました。Intermediary レイヤーは、マッピングのリネームを越えて Mods の互換性を維持するものです.

## よくある質問

<FAQList>

<FAQItem question="FabricMODには、YarnとMojmapのどちらが良いですか？">
どちらもFabricで動作します。Yarnはエコシステムの標準として採用されており、名前付きパラメータとドキュメントが付属しています。一方、MojmapはMojang自身の命名規則に準拠しています。チームがどちらの命名規則とツールを好むかによって選択してください。
</FAQItem>

<FAQItem question="モジュマップの名前は、難読化された名前と同じですか？">
いいえ。難読化された名前は `a.a.a` のようになります。モジュマップは、それらを Mojang 社が内部的に使用する読みやすい名前に対応付けていますが、各リリースごとに公開されています。
</FAQItem>

<FAQItem question="YarnとMojmapを混用できますか？">
はい、ある意味では可能です。一部のFabric環境はMojmapをベースにしており、Yarnのリマッピングを適用しているため、両者は直接競合するのではなく、一つのツールチェーンで共存しています。
</FAQItem>

<FAQItem question="YarnやMojmapはMCPに取って代わりましたか？">
1.14.4 以降の新プロジェクトでは、どちらもMCPとSeargeを不要にしました。古い1.7から1.12のプロジェクトでは、MCPの命名規則が依然として使用されています。
</FAQItem>

</FAQList>
