<KeyTakeaways>

- ポストプロセッシングは、クライアントサイドのMinecraftモジュールであり、カラーグレーディングやぼかしなどのシングルスクリーン空間シェーダーパスを、レンダリングされたフレーム全体に適用します。
- ワールドとインターフェースがすでに描画された後に実行され、個々のブロックやエンティティを調整するのではなく、完成した画像を1つのレイヤーとして扱います。
- 位置、ヒットボックス、タイミング、サーバーが追跡する値など、一切に影響を与えないため、サーバーサイドのアンチチートには検出されません。
- Tenacityで示されているように文書化されており、フル外部シェーダーパックをインストールするための組み込みの代替手段です.

</KeyTakeaways>

ポストプロセッシングは、Minecraftのモジュールで、ゲーム画面全体にスクリーン空間のシェーダーパスを適用します。通常、フルシェーダーパックを導入しないと得られないような効果です。これを有効にすると、クライアント側で最終的なフレームの色を変更したり、ぼかしたり、カラーグレーディングを適用したりします。これはモニターに表示される直前の処理です。ゲームの値を一切変更することはありません。ブロックの位置は変わらず、ヒットボックスのサイズも変わりません。サーバーもこのモジュールの存在を知りません。これはゲームプレイではなく、表示に関するオプションであり、最初から最後まで一貫して動作します。

## 仕組み

Minecraftクライアントが描画するものの多くは、オブジェクトごとに処理されます。例えば、ブロックはテクスチャを取得し、エンティティはモデルを取得し、HUDは独自のパスで描画されます。ポストプロセッシングは異なります。ワールドとインターフェースがすでにレンダリングされた後で、コンポジットされたフレーム全体にシェーダーパスを適用します。これは、画面に表示される直前です。シーン内の個々の要素ではなく、完成した画像を単一の単位として扱うため、要素を一つずつ変更する代わりに、すべてを一度に変更したり、ぼかしたりすることができます.

モジュールで文書化されている効果は2つあります。1つはカラーグレーディングで、フレームの色とコントラストを調整して、シーンを暖かく、冷たく、またはフラットに見せるものであり、もう1つはブラーで、画像全体をソフトにします。どちらも同じパスで処理されるため、モジュールをオンにしても、画面上の各要素に対してレンダリングステップが追加されるわけではなく、結果全体に対する1回のパスのみとなります。また、それが、新しいライティングモデルを導入するよりも実行コストが低い理由でもあります。既存のピクセルを再着色しているだけで、世界の照明方法を再計算しているわけではないからです。

人々がこれを使う理由は、シェーダーパックをインストールするのと同じです。スクリーンショットやクリップをデフォルトのMinecraftとは異なるものに見せたいからです。カラーグレーディングは、リソースパックなしでシーンにムードを与えます。ブラーパスは、ごちゃごちゃした、または厳しい視界を柔らかくします。外部のローダーとパックの組み合わせではなく、組み込みのトグルであるため、ダウンロード、設定、アップデートの必要はありません。

## ポストエフェクト vs. フルシェーダーパック

| | ポストプロセッシング | フルシェーダーパック |
|---|---|---|
| インストール | クライアントに組み込み済み、1つのトグル | 別途ローダー（Iris、OptiFine）と、選択・設定するパックが必要 |
| 範囲 | 完了したフレームに対して1回のスクリーン空間パス | レンダリングパイプラインの多くを置き換えたり拡張したりする |
| 効果 | カラーグレーディング、ぼかし | パックによって異なるが、ライティング、影、反射など |
| セットアップ | 即時 | パックの選択、インストール、調整 |

どちらのアプローチもサーバー側では何も変更しません。違いは深さです。シェーダーパックは世界の光の当たり方を再構築するのに対し、ポストプロセッシングはゲームが既にレンダリングしたものを再着色またはソフトフォーカスします.

<Callout variant="tip" title="Opalにおけるこの機能の位置づけ">
Opalには「ポストプロセッシング」というモジュールは同梱されていません。しかし、クライアント側のビジュアル処理は、全く同じ原則に基づいて動作します。それは、ローカルでのみ適用される調整で、あなたのマシンから出ることなく、サーバーで追跡される値に影響を与えることはありません。シェーダーパックをインストールせずに、スクリーンショットや動画のために特定の見た目を追求したい場合に、この機能群を探してみてください。見た目に関するもので、リスクはゼロ、アンチチートが介入しようとしても介入できないものです。
</Callout>

## よくある質問

<FAQList>

<FAQItem question="Minecraftにおけるポストプロセシングはゲームプレイに影響を与えますか？">
いいえ、影響はありません。レンダリング後の最終的なフレームの見え方だけを変更するもので、移動、戦闘、ヒットボックス、またはサーバーが認識するその他の要素には影響しません。
</FAQItem>

<FAQItem question="ポストプロセッシングはBANされる可能性はありますか？">
いいえ。これは純粋に視覚的な、クライアントサイドのレンダリング処理であり、ネットワークやゲームプレイに影響を与えないため、検出されるリスクはありません。
</FAQItem>

<FAQItem question="どのMinecraftクライアントにポストプロセッシングモジュールがありますか？">
ポストプロセッシングは、Tenacityで確認されているようにドキュメント化されています。同様のスクリーン・スペース効果のパスは、他のクライアントでも異なる名前で、一般的な表示オプションとして登場します。
</FAQItem>

<FAQItem question="ポストプロセッシングにはどのような効果がありますか？">
ドキュメントに記載されている効果は、フレームの色とコントラストを変化させるカラーグレーディングと、画像全体をぼかすブラーの2つです。どちらも同じシェーダーパスで実行されます。
</FAQItem>

<FAQItem question="ポストプロセッシングはシェーダーパックと同じですか？">
ほぼ同じではありません。シェーダーパックは通常、レンダリングパイプライン全体を置き換えたり拡張したりする必要があり、インストールには別のローダーが必要です。一方、ポストプロセッシングはより限定的で、クライアント内で直接切り替えられる組み込みのエフェクトパスです。
</FAQItem>
