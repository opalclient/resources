<KeyTakeaways>

- JVM の引数は、ランチャーのプロファイルに記述されており、ゲーム内オプションにはありません。
- それらはゲームをホストする [Java ランタイム](/ja/learn/what-is-the-jvm) を設定するため、タイプミスはゲームの起動に影響を与え、ワールドには影響しません。
- 多くのプレイヤーが望むフラグは `-Xmx` で、ゲームが使用できるメモリ量を制限します。
- ほとんどの設定では、4～6ギガバイトで十分です。それ以上は、スタッターを悪化させる可能性があります.

</KeyTakeaways>

Minecraftのランチャーのプロファイル設定で、JVM引数を追加します。使用するプロファイルを開き、JVM引数の項目を見つけて、そこでスペース区切りのフラグリストを編集してください。これらのフラグは、ゲームをホストする[Java仮想マシン](/ja/learn/what-is-the-jvm)を設定するもので、ゲーム自体ではないため、タイプミスはワールドに影響を与えることなく、起動に影響します.

<Callout variant="info" title="どこ Opal 合う">Opalは[Fabric](https://fabricmc.net/)のModであり、動作するゲームと同じJVM引数を共有します。対象バージョンと正確な設定については、[セットアップガイド](/ja/docs/setup)を参照してください</Callout>

## JVM引数とは

JVM引数とは、ゲーム起動時にJavaランタイムに渡すフラグのことです。ランチャーはプレイボタンを押すたびに長いコマンドラインを構築し、JVM引数はその行の一部で、ランタイムを調整します。最も一般的なものは、ゲームが使用できるメモリ量を設定するものです。

これらのフラグはMinecraftの設定ではありません。これらはゲームがロードされる前からJVMによって読み込まれるため、ゲーム内オプションメニューではなく、ランチャーで設定します。

## 設定場所

ほとんどのランチャーは、各プロファイル内にJVM引数のフィールドを持っています。ラベルは異なる場合がありますが、場所は一貫しています。

| ランチャーの種類 | フィールドの場所 |
| :--- | :--- |
| 公式ランチャー | インストール、プロファイル編集、詳細オプション、JVM引数 |
| サードパーティ製ランチャー | プロファイルまたはインスタンス設定、多くの場合、Javaまたは詳細タブの下 |
| コマンドライン / スクリプト | 実行する`java`コマンド、`-jar`の前に |

既存のリストを置き換えるのではなく、編集してください。ランチャーは適切なデフォルト値を提供しており、それらを消去すると起動に失敗する可能性があります。

## 多くの人が望む引数：ヒープサイズ

多くのプレイヤーが求めるフラグは `-Xmx` で、JVM が使用できる最大ヒープサイズを制限します。`-Xmx4G` は最大 4 ギガバイトまで許可します。また、`-Xms` は初期ヒープサイズです。

```
-Xmx4G -Xms2G
```

この安全性を保つためのルールは 2 つあります。利用可能なメモリよりも多くのメモリを割り当てないでください。Java プロセスに余裕が必要であり、オペレーティングシステムが残りのメモリを使用する必要があるためです。より多くのメモリが常に良いとは限りません。非常に大きなヒープは、ガベージコレクションの一時停止を小さくするのではなく、悪化させる可能性があります。ほとんどの設定では、4 ～ 6 ギガバイトで十分です。

## よく見かける引数

- `-Xmx<サイズ>` は最大ヒープを設定します。例えば、`-Xmx4G` です。
- `-Xms<サイズ>` は初期ヒープを設定します。例えば、`-Xms2G` です。
- `-XX:+UseG1GC` は、一般的なデフォルトである G1 ガベージコレクタを選択します。
- `-Dプロパティ=値` は、ゲームまたは mod が起動時に読み取るシステムプロパティを設定します。

ヒープフラグよりも必要なものはほとんどありません。古いガイドからコピーペーストされた長い引数リストは、最新の Java ランタイムでは役に立つよりもむしろ害になることがよくあります。

## 何かうまくいかない場合

<Checklist>
<ChecklistItem>編集後、ゲームが起動しません。フラグにタイプミスがありました。デフォルトの引数を復元し、変更を一つずつ再追加してください.</ChecklistItem>
<ChecklistItem>`-Xmx` は無視されるか、エラーが発生します。単位または構文が誤っています。`-Xmx4G` のように、スペースなしで `G` または `M` を使用してください。`-Xmx 4 GB` のように記述しないでください.</ChecklistItem>
<ChecklistItem>スタッターがメモリ増加後に悪化した。ヒープサイズが大きすぎると、ガベージコレクションの一時停止が長引く可能性がある。`-Xmx` を4～6ギガバイト程度に下げる.</ChecklistItem>
</Checklist>

## よくある質問

<FAQList>

<FAQItem question="Minecraftにおける-Xmxとは何ですか？">
Javaランタイムが使用できる最大ヒープサイズを設定します。`-Xmx4G`と指定すると、ゲームは4ギガバイトに制限されます。多くの「RAMを増やす」ガイドの裏にあるフラグです。
</FAQItem>

<FAQItem question="MinecraftにどれくらいのRAMを割り当てるべきですか？">
多くのプレイヤーにとって、4～6ギガバイトあれば十分です。それ以上割り当てもほとんど役に立たず、ガベージコレクションの一時停止が長くなる可能性があるため、JVMにすべてのメモリを割り当てないでください。
</FAQItem>

<FAQItem question="MODs は JVM の引数を変更しますか？">
通常は変更しません。MODs は既存の Java プロセスの内部にロードされるため、すでに設定されているヒープを共有します。いくつかの MODs で特定のシステムプロパティがドキュメント化されていますが、ヒープフラグでほとんどの場合をカバーできます。
</FAQItem>

<FAQItem question="JVM引数はゲーム設定と同じですか？">
いいえ、違います。JVM引数はゲームが読み込まれる前にJavaランタイムを設定し、ランチャーに存在します。ゲーム設定は、ゲーム内オプションメニューにあります。
</FAQItem>

</FAQList>
