## ユーティリティクライアントとは

ユーティリティクライアントは、便利な機能をまとめて一度インストールするだけで利用できるMODです。その多くは、同じニーズに対応することを目指しています。より洗練されたインターフェース、パフォーマンスの向上、そしてバニラMinecraftでは提供されないちょっとした利便性です。

一般的な機能は以下の通りです。

- 設定可能なHUD：アーマーの状態、座標、FPSカウンター、ポーションの残り時間。
- フレームレートを向上させ、スタッターを軽減するパフォーマンスオプション。
- よく行う操作のための追加のキーバインドとクイックトグル。
- カスタムの照準やズームなどの、外観や視覚的な設定。

目的は利便性です。ユーティリティクライアントは、ゲームが許可していない能力を与えるものではありません。

## 動作原理

ユーティリティクライアントは、[MODローダー](/ja/learn/what-is-a-minecraft-mod-loader)を通してロードされます。Fabric側では、通常のFabric MODとして動作します。Minecraftのバージョンに対応したローダーをインストールし、クライアントのJARファイルをmodsフォルダにドロップするだけで、起動時にロードされます。そこから、既存のゲームの上にその機能と設定画面を追加します。

ローダー上に存在するため、ユーティリティクライアントは他のMODと同様のインストールパスに従い、互換性のあるMODと並行して動作できます。また、多くのクライアントは、[Fabric API](/ja/learn/what-is-the-fabric-api)のようなローダーの共有APIに依存し、ゲームにクリーンに統合されます。

## ユーティリティクライアントとチートクライアントの違い

その境界線は、クライアントが不正なアドバンテージを与えるかどうかです.

| | ユーティリティクライアント | チートクライアント |
| :--- | :--- | :--- |
| 目的 | 利便性と生活の質の向上 | ゲームプレイの有利性 |
| 例となる機能 | HUD、FPSブースト、キーバインド | エイムアシスト、リーチ、フライト |
| サーバーのルール | 通常許可される | ほとんど常に禁止される |
| 検出リスク | 低い | 高い |

ユーティリティクライアントは許可されるように設計されています。一方、チートクライアントはルールを破るために作られ、不正行為とみなされます。クライアントには両方の種類の機能が混在している場合があるため、アドオンツールを使用する前にサーバーのルールを確認するのが安全です。

## なぜ人々がそれを使用するのか

魅力は、多数の個別のMODを検索して設定することなく、より良く、スムーズなゲームプレイを得られることです。1つのインストールで、HUD、パフォーマンスオプション、キーバインドがすべて単一の設定メニューにまとまります。競技プレイヤーにとって、クリーンなHUDと安定したフレームレートは、ルール違反ではなく、情報と快適さにつながる、許可された有利な点です。

</FAQItem>
