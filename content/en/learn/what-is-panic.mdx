<KeyTakeaways>

- Panic is a hotkey that instantly disables every active module in a Minecraft utility client.
- It's a workflow feature, not a gameplay one: it doesn't touch combat, movement, or rendering.
- It gives no advantage by itself, so it's invisible to anticheat alone.
- It exists for the moment a player suspects a moderator or screen-share review is starting.

</KeyTakeaways>

Panic is a single-key emergency shutoff built into some Minecraft utility clients. Press it once and every module that's currently running gets switched off in the same instant, instead of you hunting through a menu or mashing individual hotkeys. It's less a module in the traditional sense and more a kill switch for all the others.

## How it works

Panic binds to its own key, separate from the toggle each module already has. Trigger it and the client walks the full list of active modules and flips every one of them off in a single pass. In Fan, it's documented as a player-category module for exactly this reason: it's about the player's state, not the world's.

There's no confirmation prompt and no partial disable. You don't get to keep two modules running and kill the rest. It's binary by design, because the whole point is speed. A menu with checkboxes is safer to reason about, but it costs seconds you might not have.

## Is Panic itself bannable

No. Flipping modules off changes nothing a server can measure. Panic doesn't touch combat, movement, or rendering on its own, so there's nothing for an anticheat to score against and nothing a human observer can catch mid-press. It's closer to a convenience toggle than to anything resembling a cheat.

What it actually does is defensive: it helps conceal that other modules were running a moment ago, not win anything by itself. A player with nothing else active has no reason to touch it, since there'd be nothing to shut off. That's the tell. A client's panic key says a lot more about the modules sitting next to it in the list than it does about Panic.

Opal doesn't ship a module called Panic. If a one-key kill switch is what you're comparing clients on, that's worth checking directly in each client's own module list rather than assuming every utility client ships the same convenience features.
</Callout>
