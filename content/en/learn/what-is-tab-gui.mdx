Tab GUI is a keyboard-navigable menu built into Minecraft utility clients for browsing and toggling modules without touching the mouse. Instead of dragging panels around the screen, you cycle through categories like combat, movement, and visual mods with key presses, then select an entry to flip it on, off, or into its settings. It's the other standard menu system alongside Click GUI, and the two solve the same problem, getting to a module fast, with opposite inputs.

## How it works

A Tab GUI trades panels for a list. Press the menu's keybind and it drops onto the screen as a compact, scrollable set of entries grouped by category: combat, movement, player, visual, whatever the client organizes into. Key presses move the selection between categories and then between modules inside one, and another press toggles the highlighted module on or off or drills into its settings if it has any. Close it with the same keybind that opened it and the screen is clean again.

At no point does a cursor appear. That's the whole pitch: everything a Click GUI does with a mouse, done here with the keys your hand is already resting on.

## Tab GUI vs Click GUI

Both menus point at the same underlying module list. What changes is how you get there.

| Aspect | Tab GUI | Click GUI |
|---|---|---|
| Input | Keyboard | Mouse |
| Screen footprint | Compact list, opens and closes with one keypress | Floating panels you position and resize |
| Best for | Fast toggles without leaving WASD | Browsing visually, dragging sliders and dropdowns |

A Tab GUI rewards players who reconfigure often mid-match, since jumping straight to a category by keyboard beats hunting through nested panels with a cursor. A Click GUI rewards players who'd rather see everything laid out and drag values into place by feel. Fan is one client documented shipping both side by side, letting you use whichever fits your hands better instead of forcing one style on you.

## FAQ

<FAQList>

<FAQItem question="Is Tab GUI a cheat?">
No. It's a navigation menu for browsing and toggling a client's modules and settings, comparable to a settings screen. It doesn't alter gameplay or send anything to the server on its own.
</FAQItem>

<FAQItem question="How is Tab GUI different from Click GUI?">
Tab GUI is keyboard-driven: you cycle through categories and entries with key presses. Click GUI is mouse-driven: you drag and click floating panels. Both usually expose the same underlying modules, just through different input methods.
</FAQItem>

<FAQItem question="Which clients ship a Tab GUI?">
Fan is one documented client that ships a Tab GUI alongside a Click GUI, letting the player pick whichever navigation style fits them. Opal ships one too, described in its own catalog as a keyboard-navigable menu for the client's features.
</FAQItem>

<FAQItem question="Does using Tab GUI affect detection risk?">
No. It only renders and toggles local settings, so it has no bearing on anything a server-side anticheat checks. Risk comes from the modules you toggle, not the menu you toggle them with.
</FAQItem>

<FAQItem question="Can a client have both a Tab GUI and a Click GUI?">
Yes. Several clients ship both menu systems side by side instead of forcing a single navigation style, so you can switch depending on whether your hand is on the mouse or the keyboard.
</FAQItem>
