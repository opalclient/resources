<KeyTakeaways>

- New Chunks is an exploit module that tells brand new terrain apart from land the server already generated and sent before.
- It reads packet and lighting cues from ordinary chunk loading, no exploring, digging, or map-checking required.
- On anarchy servers like 2b2t, that border marks the edge of explored land and points toward where bases are hiding.
- It never sends the server anything abnormal, which is why fair-play rules catch it more often than anticheats do.

</KeyTakeaways>

New Chunks is an exploit module that tells the difference between terrain the server just generated for the first time and terrain that already existed and is only being reloaded. It makes that call by reading packet and lighting cues sent during ordinary chunk loading, the same data every client already receives whether the module is running or not. No player action is required. On a world that's been picked over for years around spawn, that single distinction is the fastest way to find the actual frontier, and whatever someone tried to bury past it.

## How New Chunks reads the world

The module watches the data streaming in as chunks pop into render distance. Freshly generated terrain and terrain that already existed send subtly different packet and lighting signatures as they load, and that's the whole trick: New Chunks reads a difference that was already there, it just puts a label on it. There's nothing for the player to interpret by eye. The module runs passively in the background and flags the distinction the moment a nearby chunk loads, so a visible line between old world and new world tracks along as you travel.

That line is worth chasing for two reasons. Land explored for years around spawn on a server like 2b2t has already been picked over, so knowing which chunks are new tells you where the frontier actually starts. It also works in reverse: a base built away from spawn sits inside chunks that were generated long ago, so sweeping unfamiliar terrain for the old-versus-new border narrows down where a hidden build is likely sitting.

## Why anticheats can't just flag it

New Chunks doesn't lie to the server about anything. It reads packet and lighting data the server already sends to every connected client as part of normal chunk loading, then draws a conclusion from it entirely on the client side. There's no forged packet, no altered movement, nothing to compare against a baseline the way an anticheat compares reach or attack timing on a combat cheat.

That puts it in a different bucket than something like an aimbot or a scaffold. It's an information exploit: the advantage comes from reading data you were always going to receive, just interpreting it differently than the server intended, not from sending anything false. Most servers that ban tools like this fall back on general fair-play rules rather than a packet-anomaly flag, because there usually isn't a specific anomaly to point to.
