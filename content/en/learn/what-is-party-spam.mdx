Party Spam is a module that spams a target player with repeated party invite requests on any server that has a party system, Hypixel being the clearest example. It doesn't touch your aim or your movement, it just calls the same invite command a normal player would type, and calls it over and over, faster than any human could manage. Some people run it purely to be irritating. Others chain it into other behavior that happens to key off party invites. Either way it reads less like a cheat and more like a very committed macro.

## How Party Spam actually works

Every server with a party system ties invites to one simple action: type `/party invite <player>`, or click a menu button that fires the same command, and the target gets a single invite. Party Spam automates that action and puts it on a loop, firing the identical command at the same target again and again, as fast as the server's command handler will let it.

That's the whole trick. There's no packet forgery and no exploit of server-side physics, because there's no physics involved to begin with. It's the exact request the server already processes for a legitimate invite; Party Spam just refuses to stop sending it. On Hypixel and servers like it, the result is a chat window that fills up with invite after invite until the target dismisses them, mutes the sender, or the server's own rate limit steps in.

## Is Party Spam safe to use

Mechanically, yes. Party Spam doesn't send anything the server wouldn't accept from a normal player clicking the invite button unusually fast. There's no position spoofing and nothing outside the ordinary chat and command channel, so there's nothing for a combat or movement anticheat to flag.

The actual risk sits with the server's rules, not its anticheat. Most servers cap how many chat messages or commands you can fire per second, and a fair number have an explicit rule against spamming other players, invites included. A moderator can still mute or ban you for annoying someone with invites even though the client itself did nothing mechanically unusual. That's a conduct problem, not a detection problem, and it's worth knowing the difference before you aim it at a stranger.
