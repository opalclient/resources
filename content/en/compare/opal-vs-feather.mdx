## Fabric mod vs standalone: what each client is actually built on

Feather ships standalone: no Fabric, no Forge, it does not inject through a mod loader at all. That is a real architectural choice, not a footnote. Standalone clients are not tied to how fast a mod loader catches up to a new Minecraft release, and that is a genuine advantage worth crediting.

Opal takes the opposite path. It ships as a Fabric mod (Fabric Loader 0.19.3, Fabric API 0.150.0+26.2) wrapped inside its own launcher, so you never touch Fabric yourself, but under the hood it is riding the Fabric ecosystem: mixins, an access widener, the whole toolchain. That gets Opal onto Minecraft 26.2 on Java 25 quickly, because it is building on an actively maintained loader instead of reimplementing the client from the ground up. The trade-off runs both ways: Opal updates fast because Fabric does the heavy lifting, and it is tied to Fabric's own release cadence to do it.

## What $8.99 buys with Opal

$8.99, with a $24.99 list price used as the anchor, gets you Opal: 70 modules split across combat, movement, visual, world, and utility, run through a launcher so there is no separate Fabric install to manage. The UI is custom, built on NanoVG and a Blaze3D-based effects engine, with a GraalJS/Yoga runtime driving panels like the dynamic island overlay. The loud set, flight, scaffold, aggressive combat, is tuned for BedWars/SkyWars-style minigame combat, while a quieter set is built to hold up on most servers. On top of that, Opal ships an embedded GraalJS scripting engine, so if none of the 70 built-in modules do exactly what you want, you can write one.
