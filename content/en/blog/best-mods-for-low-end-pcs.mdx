On a weak PC or laptop, the goal is fewer things to draw and less memory to manage. [Sodium](https://modrinth.com/mod/sodium) does the heavy lifting on rendering, a couple of memory and culling mods finish the job, and [Dynamic FPS](https://modrinth.com/mod/dynamic-fps) stops the game cooking your laptop when you tab out. All free on [Fabric](https://fabricmc.net). One note for Opal users: Lithium is not compatible with Opal, so build around Sodium.

<KeyTakeaways>

- Sodium is the biggest win on weak hardware. Install it first.
- FerriteCore lowers RAM pressure; Entity Culling skips hidden mobs.
- Dynamic FPS stops the game wasting power in the background.
- Lithium is not compatible with Opal. Use Sodium if you run Opal.

</KeyTakeaways>

## Sodium

The biggest win on weak hardware. The rendering rewrite is where most of your frames come back, and it helps integrated GPUs the most. Install [Fabric API](https://modrinth.com/mod/fabric-api) first.

## FerriteCore

Lowers RAM use so the game pauses less for garbage collection. Important when you only have a few gigabytes to spare.

## Entity Culling

Skips rendering entities you cannot see. On a low-end GPU, not drawing hidden mobs is an easy gain.

## Dynamic FPS

Drops the frame rate when the window is in the background, so the game stops working hard while you are tabbed out. Kind to laptops.

## Can a low-end PC run a client?

Yes, if the base is solid. Opal is a Fabric mod for Minecraft 1.21.10 and installs into the same `mods` folder as Sodium. It is a full client with 200+ features, so it asks for some resources, which is why the headroom from Sodium matters most on weak hardware. Get the performance stack in, keep render distance low, leave Lithium out, and Opal can run on modest machines.
