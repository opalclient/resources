A config in Opal is a saved snapshot of all your module settings: which modules are on, their values, keybinds, and parameters. Think of it as a loadout you save, share, and swap between. Opal's config browser turns that into a community library, so you can be competitive on a new server in seconds instead of tuning from scratch.

<KeyTakeaways>

- A config is a full loadout: modules, values, keybinds, parameters.
- Browse, search, and sort community configs from the dashboard.
- Save your own, then publish it to the public browser in a click.
- Use scripting to auto-load the right config per server.

</KeyTakeaways>

## Browsing public configs

The config browser on the Opal dashboard is where you find configs the community has shared. Open Configs, browse the public list, search by name, and sort by Newest, Most Downloaded, or Most Liked.

<Callout variant="tip" title="What a good config looks like">
A descriptive name like "Competitive Bedwars Ghost" or "Blatant Speed Bridge", solid download and like counts, and a recent upload date. Newer is more likely to work with current bypasses.
</Callout>

## Downloading configs

Found one you like? Open it, read the details, and download. It saves to your private collection and loads in-game.

To load it, use the config command:

```
/load config-name
```

Or open the in-game Config Browser module and pick from your saved configs.

## Creating your own

<Steps>

<Step number={1} title="Tune your setup">
Set sensitivity, ranges, delays, and visuals exactly how you want them.
</Step>

<Step number={2} title="Save it locally">
Save the current setup as a named config. That creates a private config only you can see.
</Step>

<Step number={3} title="Publish it (optional)">
On the dashboard, open Configs, find your saved config, click Publish, and give it a clear name. Published configs show up in the public browser.
</Step>

</Steps>

## Config strategies

### Server-specific

- Server Ghost: subtle aim assist, reach 3.1, velocity 85%.
- Server Blatant: full bypass suite for casual games.
- Practice PvP: ghost features only, tuned for duels.

### By playstyle

- Aggressive: higher reach, stronger velocity reduction, faster attacks.
- Defensive: velocity and movement first, minimal aim assist.
- Balanced: moderate across the board.

### Quick-switch

Use Opal's [scripting engine](/en/blog/introducing-opal-scripting-engine) to auto-load a config based on the server you join, so the right setup is always live without flipping it by hand.

## Why config sharing matters

A built-in config platform is rare. With most clients you are trading JSON files in Discord or guessing settings off a video. Opal's system means new users can get competitive right away with a proven config, experienced users can share what works, and everyone benefits from community-tested settings. It is part of what makes Opal a platform rather than just a client.
