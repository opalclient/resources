# Overlay Proxy

Custom Dynamic Island HUD elements, registered and driven entirely from a script.

---

## Overlay Proxy
**Global Binding:** `overlay`

Allows scripts to register and manage custom "Dynamic Island" HUD elements.

<Method
    name="createIsland"
    signature="String createIsland(Object config)"
    description="Creates a new island and returns its unique ID. Config must include width, height, priority, and a render callback."
    params={[
        { name: "config", type: "Object", desc: "{ width, height, priority, render(posX, posY, width, height, progress) }" }
    ]}
    returns={{ type: "String", desc: "Unique Island ID." }}
/>

```javascript
const island = overlay.createIsland({
    width: 160,
    height: 28,
    priority: 10,
    render: (posX, posY, width, height, progress) => {
        // Build colors with renderer.color(...): a raw 0xAARRGGBB literal
        // above 2^31 narrows to the wrong int and loses the alpha channel.
        renderer.roundedRect(posX, posY, width, height, 4, renderer.color(26, 26, 26));
        renderer.text("productsans-bold", "Custom HUD",
            posX + 10, posY + 14, 8, renderer.color(255, 255, 255));
    }
});
overlay.showIsland(island);
```

<Method name="showIsland" signature="void showIsland(String islandId)"

<Callout type="info">
    **Cleanup is automatic and not something you call.** The client destroys every island a script created when its module disables or the script unloads, so you never have to tidy up yourself. `overlay.cleanup()` is not part of the script API: it is a host-side hook, and calling it from a script throws. To clear islands early, call `destroyIsland(id)` for each one you own.
</Callout>
