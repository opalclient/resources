# Client Proxy

General client interaction: module lookups, chat messages, screen metrics, and theme colors.

---

## Client Proxy
**Global Binding:** `client`

Provides access to high-level client functions, including module management, chat interaction, and system metrics.

### Chat & Messaging

<Method
    name="print"
    signature="void print(Object o)"
    description="Prints a message to the in-game chat for the local player only. Calls toString() on the object."
    params={[
        { name: "o", type: "Object", desc: "The object or message to print." }
    ]}
/>

<Method
    name="success"
    signature="void success(String message)"
    description="Prints a success-styled (green) message to the local chat."
    params={[
        { name: "message", type: "String", desc: "The success message text." }
    ]}
/>

<Method
    name="error"
    signature="void error(String message)"
    description="Prints an error-styled (red) message to the local chat."
    params={[
        { name: "message", type: "String", desc: "The error message text." }
    ]}
/>

<Callout type="info">
    `print`, `success`, and `error` are **local only**: they render in your own chat and send nothing to the server. To actually talk to the server, use `sendChat` / `runCommand` below.
</Callout>

<Method
    name="sendChat"
    signature="void sendChat(String message)"
    description="Sends a chat message to the server as though the local player typed it."
    params={[
        { name: "message", type: "String", desc: "The message to send." }
    ]}
/>

<Method
    name="runCommand"
    signature="void runCommand(String command)"
    description="Runs a command as the local player. Pass the command without its leading slash."
    params={[
        { name: "command", type: "String", desc: "The command to run, with no leading slash (e.g. 'spawn')." }
    ]}
/>

```javascript
client.print("Only I can see this");     // local
client.sendChat("gg");                   // sent to the server
client.runCommand("spawn");              // no leading slash
```

### Module Access

<Method
    name="isModuleEnabled"
    signature="boolean isModuleEnabled(String id)"
    description="Checks if a specific module is currently enabled."
    params={[
        { name: "id", type: "String", desc: "The module name." }
    ]}
    returns={{ type: "boolean", desc: "True if enabled, false otherwise." }}
/>

<Method
    name="setModuleEnabled"
    signature="void setModuleEnabled(String id, boolean enabled)"
    description="Toggles a module on or off."
    params={[
        { name: "id", type: "String", desc: "The module name." },
        { name: "enabled", type: "boolean", desc: "True to enable, false to disable." }
    ]}
/>

<Callout type="info">
    `client.isModuleEnabled` / `setModuleEnabled` cover the common single-module case. For metadata lookups, category listings, and bulk operations, use the [Modules Proxy](modules) instead.
</Callout>

<Callout type="warning">
    There is no `client.getModule(id)`. It returned a raw module the sandbox exposes nothing on, so every call against the returned value failed. Toggle modules by id with `client.setModuleEnabled(id, enabled)` or the `modules` proxy (`modules.toggle`, `modules.setEnabled`) instead.
</Callout>
