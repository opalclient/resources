# Types

Bound Java and wrapper types available to every script. The class globals are
injected; you do not need to import them. The rest are wrapper types that proxy
methods hand back.

<Callout type="warning">
    **Read every value with getters, never properties.** The scripting sandbox
    grants member access only to members its wrappers explicitly export, and it
    does **no bean-property mapping**. `vec.x` is `undefined`, not a number, and
    it does not throw: it silently poisons your arithmetic with `NaN`. Always
    write `vec.getX()`.
</Callout>

<Callout type="info">
    Every type below is a script-safe wrapper with real, readable getters. There
    are no raw Minecraft types left in the scripting API surface, except for a
    few deliberately **opaque tokens** (`mc.getWorld()`, `HitResult`) that you
    only ever hand straight back to another proxy method.
</Callout>

## BlockPos

An integer block position. Bound as `BlockPos`; backed by the
`ScriptBlockPos` wrapper around `net.minecraft.core.BlockPos`. Construct with
`new BlockPos(x, y, z)`. Returned by proxies such as
`player.getBlockPosition()`.

```javascript
const pos = player.getBlockPosition();
const below = new BlockPos(pos.getX(), pos.getY() - 1, pos.getZ());
```

## Vec2f

A yaw/pitch rotation pair. Bound as `Vec2f`; backed by the `ScriptVec2f`
wrapper around `net.minecraft.world.phys.Vec2`. Construct with
`new Vec2f(yaw, pitch)`. Returned by rotation proxy methods such as
`rotation.getRotation()` and `rotation.getRotationFromPosition(pos)`.

```javascript
const rot = rotation.getRotation();
client.print("Yaw: " + rot.getYaw() + " Pitch: " + rot.getPitch());
```

## Direction

A cardinal or vertical facing direction. Backed by the `ScriptDirection`
wrapper around `net.minecraft.core.Direction`. Scripts do not construct this
directly. Obtain one from `world.getAdjacentDirections(pos)`.

```javascript
const dirs = world.getAdjacentDirections(pos);
if (!dirs.isEmpty()) {
    const side = dirs.get(0);
    const neighbor = pos.offset(side.getOpposite());
}
```

## RaytracedRotation

A validated rotation paired with the hit result needed for block or entity
interaction. Backed by the `ScriptRaytracedRotation` wrapper. Returned by
`rotation.getRotationFromRaycastedBlock()` and
`rotation.getRotationFromRaycastedEntity()`. May be `null` when no valid
rotation reaches the target.

```javascript
const raytrace = rotation.getRotationFromRaycastedBlock(
    neighbor, side.getOpposite(), rotation.getRotation(), player.getEyePosition()
);
if (raytrace === null) return;

rotation.set(raytrace.getYaw(), raytrace.getPitch());
mc.interactionManager.interactBlock(MAIN_HAND, raytrace.getHitResult());
```

<Method name="getYaw" signature="float getYaw()" description="Returns the yaw angle in degrees to aim at the target." returns={{ type: "float", desc: "Yaw in degrees." }} />
<Method name="getPitch" signature="float getPitch()" description="Returns the pitch angle in degrees to aim at the target." returns={{ type: "float", desc: "Pitch in degrees." }} />
<Method name="getHitResult" signature="HitResult getHitResult()" description="Returns the hit result to pass straight to mc.interactionManager.interactBlock(). An opaque token: it exposes no readable members of its own." returns={{ type: "HitResult", desc: "Opaque hit result." }} />

<Callout type="info">
    `getHitResult()` returns an **opaque token**. Hand it directly to

</Callout>

## Entity

Any entity a script can see: what `world.getEntities()`,
`world.getLivingEntitiesInRange()`, `mc.getPlayer()`, and `event.getTarget()`
hand back. Backed by the `ScriptEntity` wrapper. Scripts do not construct one.

<Callout type="info">
    Reads that need a living entity return the `-1` sentinel on one that is not
    (`getHealth()`, `getMaxHealth()`, `getAbsorption()`, `getArmor()`, and
    `getDistance()`). Check `isLiving()`, or test for `-1`, before trusting them.
</Callout>

```javascript
const targets = world.getLivingEntitiesInRange(16);
for (let i = 0; i < targets.size(); i++) {
    const entity = targets.get(i);
    client.print(entity.getName() + ": " + entity.getHealth() + " HP");
}
```

## Effect

One active status (potion) effect, from `player.getEffect(name)`,
`player.getEffects()`, or the `Entity` equivalents. Backed by the `ScriptEffect`
wrapper. Scripts do not construct one.

<Callout type="warning">
    **`getAmplifier()` is 0-based; `getLevel()` is 1-based.** Speed II is
    amplifier `1` and level `2`. `getAmplifier()` matches vanilla's internal
    value and is what movement math expects; `getLevel()` is the number players
    read on the screen. Picking the wrong one is an off-by-one in every
    calculation that follows.
</Callout>

<Callout type="info">
    `getDurationSeconds()` returns **`-1` when the effect is infinite**. Check
    `isInfinite()` before formatting a countdown, or the HUD reads "-1s".
</Callout>

```javascript
const speed = player.getEffect("speed");
if (speed !== null) {
    // Speed II -> getLevel() is 2, getAmplifier() is 1.
    const remaining = speed.isInfinite() ? "∞" : speed.getDurationSeconds() + "s";
    client.print(speed.getName() + " " + speed.getLevel() + " (" + remaining + ")");
}
```

## Box2D

`ScriptBox2D` wrapper. Scripts do not construct one. The rectangle is laid out
as `(x, y, width, height)`, and exposes two equivalent spellings of the same

<Callout type="info">

## List

Every collection the scripting API returns is a `List` wrapper (`ScriptList`),
never a Java array and never a JavaScript array.

<Callout type="warning">
    **`.length`, `[i]` indexing, `for...of`, and `Array.from()` do not work on a
    `List`.** They do not throw either: they silently evaluate to `undefined` or
    an empty array, so a script that uses them appears to run and quietly does
    nothing. Walk a list with `size()` and `get(i)`.
</Callout>

```javascript
const list = world.getLivingEntitiesInRange(64);
for (let i = 0; i < list.size(); i++) {
    const entity = list.get(i);
}
```

## MathHelper

`MathHelper` is bound as a global, but **nothing on it is callable from a
script**: it is a raw Minecraft type with no exported members, so every call
against it is denied by the sandbox.

<Callout type="warning">
    Treat `MathHelper` as unavailable. Use JavaScript's built-in `Math` instead,
    which covers `abs`, `floor`, `ceil`, `round`, `min`, `max`, `pow`, `sqrt`,
    `hypot`, `random`, and the trigonometric functions. For interpolation use
    `esp.lerp(start, end, tickDelta)`. The Minecraft-specific helpers scripts
    need are exposed on the proxy globals themselves.
</Callout>

## Color

The standard `java.awt.Color`, bound as `Color`. Its two constructors and
`getRGB()` are explicitly allow-listed by the sandbox, so they genuinely work.
Use it as an alternative to `renderer.color(r, g, b, a)` for building packed
ARGB integers.

```javascript
const c = new Color(255, 80, 0, 200);
renderer.rect(10, 10, 40, 40, c.getRGB());
```

## Hand Constants

`MAIN_HAND` and `OFF_HAND` are bound `net.minecraft.world.InteractionHand`
constants. Pass them to interaction calls such as `player.swingHand(hand)` and
`mc.interactionManager.interactBlock(hand, hitResult)`.

| Constant | Description |
|---|---|
| `MAIN_HAND` | The player's main hand. |
| `OFF_HAND` | The player's off hand. |

```javascript
player.swingHand(MAIN_HAND);
```
