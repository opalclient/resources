# Events

A script module registers behaviour by attaching JavaScript handlers to named events with `module.on(...)`. This page lists every event a script module can subscribe to, the payload object each handler receives, and which events can be cancelled.

---

## Registering Handlers

Inside a `registerModule` callback, call `module.on(eventName, handler)` for each event you want to react to. Events that carry data pass a payload object as the handler's only argument; the rest pass nothing your handler can read, so write those handlers with no parameter.

<Method
    name="on"
    signature="void on(String event, Function handler)"
    description="Registers a JavaScript callback for the named event. Data-carrying events pass a payload object as the only argument; the rest pass nothing readable."
    params={[
        { name: "event", type: "String", desc: "The event name (e.g. 'preGameTick'). See the tables below." },
        { name: "handler", type: "Function", desc: "The JS function to run when the event fires." }
    ]}
/>

<Callout type="info">
    There is one handler slot per event name per module. Calling `on` again with the same event name replaces the previous handler, it does not stack. Register a given event once.
</Callout>

```javascript
script.registerModule({ name: "Example", description: "Event demo" }, (module) => {
    module.on("enable", () => {
        client.print("Module enabled!");
    });

    module.on("preGameTick", () => {
        if (mc.getPlayer() === null || mc.getWorld() === null) return;
        // logic runs 20 times per second
    });

    module.on("disable", () => {
        client.print("Module disabled!");
    });
});
```

<Callout type="warning">
    **Every payload is read through `getX()` getters, and only the getters listed on this page exist.** There are no bare record accessors (`event.hand()`, `event.tickDelta()`) and no readable properties: the scripting sandbox exposes only what each payload explicitly declares. Anything else either throws `Unknown identifier` or evaluates to `undefined`.
</Callout>

<Callout type="warning">
    The events that **carry no readable data at all** are `preGameTick`, `postGameTick`, `renderScreen`, `renderWorld`, `renderBloom`, `joinWorld`, `itemUse`, `serverDisconnect`, and `resolutionChange`. Write their handlers with no parameter. For the partial tick inside a render event, call `client.getTickDelta()`.
</Callout>

## Lifecycle Events

Fired when the module itself is toggled. These are the only events whose handler receives no argument.

| Event | Payload | Fires when |
| :--- | :--- | :--- |
| `enable` | _(none)_ | The module is toggled on. Error reporting is reset here (see Error Handling). |
| `disable` | _(none)_ | The module is toggled off. Any owned Dynamic Islands are cleaned up. |

```javascript
module.on("enable", () => notification.info("Example", "Started"));
module.on("disable", () => notification.info("Example", "Stopped"));
```

## Game Tick Events

Fired around the 20 TPS client tick. Both pass an event object, but neither object carries any data. They are timing hooks for game logic.

| Event | Payload | Fires when |
| :--- | :--- | :--- |
| `preGameTick` | `PreGameTickEvent` | At the start of a tick, before vanilla tick logic runs. Ideal for logic updates. |
| `postGameTick` | `PostGameTickEvent` | At the end of a tick, after vanilla tick logic has run. Symmetric with `preGameTick`. |

```javascript
module.on("preGameTick", (event) => {
    if (mc.getPlayer() === null || mc.getWorld() === null) return;
    if (player.getHealth() < 5) {
        notification.warn("Low HP", "Health below 5!");
    }
});

module.on("postGameTick", (event) => {
    // end-of-tick cleanup
});
```

## Render Events

Fired every frame. Draw with the `renderer` proxy here.

| Event | Payload | Fires when |
| :--- | :--- | :--- |
| `renderScreen` | _(no readable data)_ | During the 2D HUD pass. Use for screen-space overlays. |
| `renderWorld` | _(no readable data)_ | During the 3D world pass. Use for world-space drawing (`esp.*` projection). |
| `renderBloom` | _(no readable data)_ | During the HUD bloom pass. Shapes drawn here feed the glow/bloom effect rather than showing directly. |

<Callout type="warning">
    **Render event payloads expose nothing.** `drawContext()`, `canvas()`, `mouseX()`, `mouseY()`, `matrixStack()`, and `tickDelta()` all **throw** if you call them: the render events declare no script-readable members. Write these handlers with no parameter.
</Callout>

<Callout type="info">
    Everything a render handler needs is on the globals instead:

    - **Partial tick** — `client.getTickDelta()`, not `event.tickDelta()`.
    - **Draw context** — implicit; the `renderer` proxy targets the open pass automatically.
    - **Screen size** — `client.getScaledWidth()` / `client.getScaledHeight()`.
    - **World projection** — the `esp` proxy, which handles the camera transform itself.
</Callout>

<Callout type="warning">
    Colors are 32-bit ARGB ints, but JavaScript has only one number type: the IEEE-754 `double`. A literal like `0xFF1a1a1a` is `4278190080`, which is larger than `2^31`; narrowing that double to a Java `int` truncates to the wrong value. Build colors with `renderer.color(r, g, b)` or `renderer.color(r, g, b, a)` instead of raw `0xAARRGGBB` literals.
</Callout>

```javascript
module.on("renderScreen", () => {
    renderer.roundedRect(10, 10, 120, 24, 4, renderer.color(0, 0, 0, 180));
    renderer.text("productsans-bold", "FPS: " + client.getFPS(), 18, 24, 8, renderer.color(255, 255, 255));
});

module.on("renderWorld", () => {
    const delta = client.getTickDelta();   // not event.tickDelta(), which throws
    // use esp.* for 3D-to-2D projection here
});
```

## Movement Events

Fired around the player's per-tick movement and the movement packet sent to the server. `preMove` and `preMovementPacket` are **cancellable**; the `pre`-packet variant also lets you rewrite the server-visible position, rotation, and flags to spoof movement.

| Event | Payload | Cancellable | Fires when |
| :--- | :--- | :--- | :--- |
| `preMove` | `PreMoveEvent` | Yes | Before the player's movement is applied for the tick. |
| `postMove` | `PostMoveEvent` | No | After the player's movement has been applied. |
| `preMovementPacket` | `PreMovementPacketEvent` | Yes | Before the movement packet is sent (position/rotation update). |
| `postMovementPacket` | `PostMovementPacketEvent` | No | After the movement packet has been sent. |

### Move payload (`preMove` / `postMove`)

<Method name="getInputX" signature="double getInputX()" description="The X component of the directional movement input for the step." returns={{ type: "double", desc: "Input X." }} />

<Method name="getInputY" signature="double getInputY()" description="The Y component of the directional movement input for the step." returns={{ type: "double", desc: "Input Y." }} />

<Method name="getInputZ" signature="double getInputZ()" description="The Z component of the directional movement input for the step." returns={{ type: "double", desc: "Input Z." }} />

<Callout type="info">
    The movement input is read as three separate components, not as a vector: there is no `getMovementInput()`. `postMove` exposes the same four getters and cannot be cancelled.
</Callout>

<Method name="setZ" signature="void setZ(double z)" description="Overrides the Z coordinate to report." params={[{ name: "z", type: "double", desc: "New Z." }]} />
<Method name="getYaw" signature="double getYaw()" description="The yaw rotation reported to the server." returns={{ type: "double", desc: "Yaw." }} />

<Method name="setYaw" signature="void setYaw(double yaw)" description="Overrides the yaw rotation (server-side rotation spoofing)." params={[{ name: "yaw", type: "double", desc: "New yaw." }]} />
<Method name="setPitch" signature="void setPitch(double pitch)" description="Overrides the pitch rotation (server-side rotation spoofing)." params={[{ name: "pitch", type: "double", desc: "New pitch." }]} />
<Method name="isOnGround" signature="boolean isOnGround()" description="Whether the player is reported as on the ground." returns={{ type: "boolean", desc: "On-ground flag." }} />
<Method name="setOnGround" signature="void setOnGround(boolean onGround)" description="Overrides the on-ground flag." params={[{ name: "onGround", type: "boolean", desc: "New on-ground flag." }]} />
<Method name="isSprinting" signature="boolean isSprinting()" description="Whether the player is reported as sprinting." returns={{ type: "boolean", desc: "Sprint flag." }} />
<Method name="setSprinting" signature="void setSprinting(boolean sprinting)" description="Overrides the sprinting flag." params={[{ name: "sprinting", type: "boolean", desc: "New sprint flag." }]} />
<Method name="isHorizontalCollision" signature="boolean isHorizontalCollision()" description="Whether the player collided horizontally this tick." returns={{ type: "boolean", desc: "Collision flag." }} />
<Method name="setHorizontalCollision" signature="void setHorizontalCollision(boolean horizontalCollision)" description="Overrides the horizontal-collision flag." params={[{ name: "horizontalCollision", type: "boolean", desc: "New collision flag." }]} />
<Method name="isForceInput" signature="boolean isForceInput()" description="Whether the packet is sent even when no movement occurred." returns={{ type: "boolean", desc: "Force flag." }} />
<Method name="setForceInput" signature="void setForceInput(boolean forceInput)" description="Forces the packet to be sent even when no movement occurred." params={[{ name: "forceInput", type: "boolean", desc: "New force flag." }]} />

<Callout type="info">
    `postMovementPacket` exposes the read-only subset of the same getters (`getX`/`getY`/`getZ`, `getYaw`/`getPitch`, `isOnGround`, `isSprinting`) describing what was actually sent. It has no setters and cannot be cancelled.
</Callout>

```javascript
// Server-side rotation spoof: report a custom yaw/pitch
module.on("preMovementPacket", (event) => {
    event.setYaw(targetYaw);
    event.setPitch(targetPitch);
});

// Speed modification before movement is applied
module.on("preMove", (event) => {
    if (movement.isMoving()) {
        movement.setSpeed(0.5);
    }
});
```

## Packet Events

Fired for raw network packets. All four are **cancellable**: cancelling drops the packet so vanilla never sends/handles it. The `instantaneous` pair runs synchronously on the network thread; the plain pair runs on the main thread.

| Event | Payload | Cancellable | Fires when |
| :--- | :--- | :--- | :--- |
| `sendPacket` | `SendPacketEvent` | Yes | A packet is about to be sent to the server. |
| `receivePacket` | `ReceivePacketEvent` | Yes | A packet is received from the server, before it is handled. |
| `instantaneousSendPacket` | `InstantaneousSendPacketEvent` | Yes | An outbound packet is sent immediately on the network thread. |
| `instantaneousReceivePacket` | `InstantaneousReceivePacketEvent` | Yes | An inbound packet is received on the network thread, before main-thread queueing. |

<Method name="getType" signature="String getType()" description="Returns the packet's type name, for identifying which packet this is." returns={{ type: "String", desc: "The packet type name." }} />

<Callout type="warning">
    There is no `getPacket()`. A raw Minecraft packet exposes nothing a script could read, so the payload gives you the **type name** and the ability to cancel. Match on `getType()` to identify a packet, then `cancel()` to drop it.
</Callout>

```javascript
module.on("sendPacket", (event) => {
    const type = event.getType();       // outbound packet type name
});

module.on("receivePacket", (event) => {
    if (event.getType().includes("Explode")) {
        event.cancel();                 // drop it (see Cancelling Events)
    }
});
```

## Player Interaction Events

Fired around the player's combat and item interactions. Only `jump` is cancellable.

| Event | Payload | Cancellable | Fires when |
| :--- | :--- | :--- | :--- |
| `attack` | `AttackEvent` | No | The player attacks an entity, before the interaction is processed. |
| `swing` | `SwingEvent` | No | The player swings an arm, before the swing is sent. |
| `itemUse` | _(no readable data)_ | No | The player uses (right-clicks) the held item. |
| `jump` | `JumpEvent` | Yes | Before the jump impulse is applied. |

<Method name="getTarget" signature="Entity getTarget()" description="attack: the entity being attacked, as an Entity wrapper." returns={{ type: "Entity", desc: "The targeted entity." }} />

<Method name="getTargetName" signature="String getTargetName()" description="attack: the target's display name. A shortcut for getTarget().getName()." returns={{ type: "String", desc: "Target display name." }} />

<Method name="getTargetId" signature="int getTargetId()" description="attack: the target's numeric network ID." returns={{ type: "int", desc: "Target entity ID." }} />

<Method name="getTargetHealth" signature="double getTargetHealth()" description="attack: the target's current health, or -1.0 if it is not a living entity." returns={{ type: "double", desc: "Target health, or -1.0." }} />

<Method name="getTargetMaxHealth" signature="double getTargetMaxHealth()" description="attack: the target's maximum health, or -1.0 if it is not a living entity." returns={{ type: "double", desc: "Target maximum health, or -1.0." }} />

<Method name="getTargetDistance" signature="double getTargetDistance()" description="attack: the distance from the player to the target." returns={{ type: "double", desc: "Distance in blocks." }} />

<Method name="isMainHand" signature="boolean isMainHand()" description="swing: whether the swing came from the main hand rather than the off hand." returns={{ type: "boolean", desc: "True for a main-hand swing." }} />

<Method name="isSprinting" signature="boolean isSprinting()" description="jump: whether the player is sprinting while jumping." returns={{ type: "boolean", desc: "Sprint state." }} />

<Method name="setSprinting" signature="void setSprinting(boolean sprinting)" description="jump: override whether the jump is treated as a sprint jump, changing the forward boost applied." params={[{ name: "sprinting", type: "boolean", desc: "True to apply the sprint-jump boost." }]} />

```javascript
module.on("attack", (event) => {
    const target = event.getTarget();       // an Entity wrapper
    client.print("Hit " + target.getName() + " at " + event.getTargetDistance());

    // The convenience getters read the same target without the hop.
    if (event.getTargetHealth() !== -1.0) {
        client.print("HP: " + event.getTargetHealth() + "/" + event.getTargetMaxHealth());
    }
});

module.on("swing", (event) => {
    if (event.isMainHand()) { /* main-hand swing */ }
});

module.on("jump", (event) => {
    if (event.isSprinting()) {
        event.setSprinting(false);          // drop the sprint-jump boost
    }
    // event.cancel() suppresses the jump entirely
});
```

## World & Connection Events

Fired around world loading, block changes, and server connection lifecycle. Only `serverConnect` is cancellable.

| Event | Payload | Cancellable | Fires when |
| :--- | :--- | :--- | :--- |
| `joinWorld` | _(no readable data)_ | No | The local player joins a world, after the client world is initialised. |
| `blockUpdate` | `BlockUpdateEvent` | No | A loaded block changes state. Carries position plus old/new block names. |
| `serverConnect` | `ServerConnectEvent` | Yes | Before connecting to a multiplayer server. Cancel to abort the connection. |
| `serverDisconnect` | _(no readable data)_ | No | The client disconnects from a server. |

<Method name="getX" signature="int getX()" description="blockUpdate: the X coordinate where the block change occurred." returns={{ type: "int", desc: "Block X." }} />

<Method name="getY" signature="int getY()" description="blockUpdate: the Y coordinate where the block change occurred." returns={{ type: "int", desc: "Block Y." }} />

<Method name="getZ" signature="int getZ()" description="blockUpdate: the Z coordinate where the block change occurred." returns={{ type: "int", desc: "Block Z." }} />

<Method name="getOldBlock" signature="String getOldBlock()" description="blockUpdate: the name of the block before the update." returns={{ type: "String", desc: "Previous block name." }} />

<Method name="getNewBlock" signature="String getNewBlock()" description="blockUpdate: the name of the block after the update." returns={{ type: "String", desc: "New block name." }} />

<Method name="getHost" signature="String getHost()" description="serverConnect: the hostname being connected to." returns={{ type: "String", desc: "Server hostname." }} />

<Method name="getPort" signature="int getPort()" description="serverConnect: the port being connected to." returns={{ type: "int", desc: "Server port." }} />

<Method name="getAddress" signature="String getAddress()" description="serverConnect: the full address being connected to." returns={{ type: "String", desc: "Target server address." }} />

<Callout type="info">
    `blockUpdate` reports the position as three plain coordinates and the blocks as **names**, not as a `BlockPos` and raw block states. Build a `BlockPos` from the coordinates if you need one: `new BlockPos(event.getX(), event.getY(), event.getZ())`.
</Callout>

```javascript
module.on("joinWorld", () => {
    client.print("Joined a world!");
});

module.on("blockUpdate", (event) => {
    const pos = new BlockPos(event.getX(), event.getY(), event.getZ());
    const before = event.getOldBlock();     // a block name, e.g. "Air"
    const after = event.getNewBlock();
});

module.on("serverConnect", (event) => {
    client.print("Connecting to " + event.getHost() + ":" + event.getPort());
    // event.cancel() aborts the connection (see Cancelling Events)
});
```

## Chat Events

| Event | Payload | Cancellable | Fires when |
| :--- | :--- | :--- | :--- |
| `chatReceived` | `ChatReceivedEvent` | Yes | A chat message is received from the server, before it is shown. Cancel to suppress it. |

<Method name="getMessage" signature="String getMessage()" description="The received chat message as a plain string." returns={{ type: "String", desc: "Message text." }} />

<Method name="isOverlay" signature="boolean isOverlay()" description="Whether the message is an action-bar overlay message rather than a chat-line message." returns={{ type: "boolean", desc: "True for action-bar messages." }} />

<Method name="setOverlay" signature="void setOverlay(boolean overlay)" description="Reroutes the message to (true) or away from (false) the action bar." params={[{ name: "overlay", type: "boolean", desc: "True to render as an action-bar message." }]} />

<Callout type="info">
    `getMessage()` returns a **plain string**, already flattened. There is no `getText()` and no `.getString()` hop.
</Callout>

```javascript
module.on("chatReceived", (event) => {
    if (event.isOverlay()) return;               // skip action-bar messages
    const raw = event.getMessage();
    if (raw.includes("spam")) {
        event.cancel();                          // suppress the message
    }
});
```

## Input Events

Raw keyboard and mouse input, plus the screen-resize hook. None are cancellable.

| Event | Payload | Fires when |
| :--- | :--- | :--- |
| `keyPress` | `KeyPressEvent` | A keyboard key is pressed. Carries the key code. |
| `mousePress` | `MousePressEvent` | A mouse button is pressed. Carries the button code. |
| `resolutionChange` | _(no readable data)_ | The GUI is resized (the framebuffer resolution changed). |

Both `keyPress` and `mousePress` expose the code through the same accessor:

<Method name="getCode" signature="int getCode()" description="Returns the code of the key (keyPress) or mouse button (mousePress) that triggered the event. Compare it against the keys global." returns={{ type: "int", desc: "The key or mouse-button code." }} />

```javascript
module.on("keyPress", (event) => {
    if (event.getCode() === keys.F7) { /* ... */ }
});

module.on("mousePress", (event) => {
    if (event.getCode() === keys.MOUSE_1) { /* right click */ }
});

module.on("resolutionChange", () => {
    // recompute layout that depends on screen size
});
```

## Cancelling Events

Cancellable events extend the client's cancellable base, letting a handler suppress the vanilla behaviour the event precedes.

| Cancellable event | Cancelling effect |
| :--- | :--- |
| `chatReceived` | Suppresses the message. |
| `jump` | Suppresses the jump. |
| `serverConnect` | Aborts the connection. |
| `preMove` | Suppresses the vanilla movement step for the tick. |
| `preMovementPacket` | Suppresses the movement packet. |
| `sendPacket` | Prevents the outbound packet from being sent. |
| `receivePacket` | Drops the inbound packet before vanilla handles it. |
| `instantaneousSendPacket` | Prevents the outbound packet from being sent. |
| `instantaneousReceivePacket` | Drops the inbound packet. |

<Method name="cancel" signature="void cancel()" description="Marks the event cancelled. Takes no argument, and the flag cannot be cleared once set." />

<Method name="isCancelled" signature="boolean isCancelled()" description="Returns whether the event has already been cancelled by another handler." returns={{ type: "boolean", desc: "True once cancel() has been called." }} />

<Callout type="warning">
    The method is **`cancel()`**, not `setCancelled()`. There is no `setCancelled` on any payload: calling it throws `Unknown identifier`.
</Callout>

<Callout type="warning">
    Calling `cancel()` on a non-cancellable event (for example `attack`, `swing`, `postMove`, `blockUpdate`, or any render event) throws, since the method does not exist on those payloads. Only call it on the events listed above.
</Callout>

```javascript
// Drop a specific outbound packet
module.on("sendPacket", (event) => {
    if (someCondition) {
        event.cancel();              // packet is never sent
    }
});
```

## Number & Color Notes

<Callout type="info">
    JavaScript has a single number type: the 64-bit `double`. Java accessors that return `int`, `float`, or `long` arrive as ordinary JS numbers, and JS numbers you pass back are narrowed to the Java parameter type. Two consequences worth remembering:

    - **Colors:** build ARGB ints with `renderer.color(r, g, b[, a])`. A raw `0xAARRGGBB` literal above `2^31` (any color with alpha ≥ 0x80) is a positive double that truncates to the wrong `int` when narrowed.
    - **Codes:** `getCode()` returns an `int` key or button code; compare it against the `keys` constants directly (`event.getCode() === keys.F7`).
</Callout>

## Error Handling

If a callback throws, the client prints the **first** error for that event name to chat (with internal class paths sanitized) and then suppresses further errors from that same event until the module is re-enabled. The handler is **not** removed: it keeps running, so a transient per-tick error does not silently disable your script.

To reset error reporting after a fix, toggle the module off and on (the `enable` event clears the suppressed-error set), or reload the script.

## Null Safety

Always check `mc.getPlayer()` and `mc.getWorld()` before touching game state, since they are `null` in menus and loading screens:

```javascript
module.on("preGameTick", () => {
    if (mc.getPlayer() === null || mc.getWorld() === null) return;
    // safe to use the player / world / inventory proxies here
});
```

<Callout type="warning">
    **Call the getters.** `mc.player` and `mc.world` read as `undefined`, not `null`, so `if (mc.player === null) return;` is always false: the guard never fires and the handler runs on unloaded state anyway. Only `mc.getPlayer()` / `mc.getWorld()` work.
</Callout>
