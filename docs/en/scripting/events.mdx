# Event Reference

Events are registered using `module.on(eventName, callback)`. The callback receives the Java event object as its argument (except lifecycle events which receive no argument).

## Lifecycle Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `enable` | None | Fired when the module is toggled on. |
| `disable` | None | Fired when the module is toggled off. |

```javascript
module.on("enable", () => {
    client.print("Module enabled!");
});

module.on("disable", () => {
    client.print("Module disabled!");
});
```

## Game Loop Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `preGameTick` | `PreGameTickEvent` | Fired at the start of a game tick (20 TPS). Ideal for logic updates. |
| `renderScreen` | `RenderScreenEvent` | Fired every frame while the 2D HUD is rendering. Use `renderer` here. |
| `renderWorld` | `RenderWorldEvent` | Fired every frame while the 3D world is rendering. |

```javascript
// Game logic — runs 20 times per second
module.on("preGameTick", (event) => {
    if (mc.player === null) return;
    
    const health = player.getHealth();
    if (health < 5) {
        notification.warn("Low HP", "Health below 5!");
    }
});

// 2D HUD rendering — runs every frame
module.on("renderScreen", (event) => {
    renderer.roundedRect(10, 10, 120, 24, 4, renderer.color(0, 0, 0, 180));
    renderer.text("productsans-bold", "FPS: " + client.getFPS(), 18, 24, 8, 0xFFFFFFFF);
});

// 3D world rendering — runs every frame
module.on("renderWorld", (event) => {
    // Use esp.* for 3D-to-2D projection here
});
```

## Movement Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `preMovementPacket` | `PreMovementPacketEvent` | Fired before the client sends the movement packet (motion update). |
| `postMovementPacket` | `PostMovementPacketEvent` | Fired after the movement packet is sent. |
| `preMove` | `PreMoveEvent` | Fired before player movement calculations are applied. |
| `postMove` | `PostMoveEvent` | Fired after player movement is calculated. |

```javascript
// Rotation spoofing — set server-side rotation before packet
module.on("preMovementPacket", (event) => {
    rotation.set(targetYaw, targetPitch);
});

// Speed modification — adjust motion before it's applied
module.on("preMove", (event) => {
    if (movement.isMoving()) {
        movement.setSpeed(0.5);
    }
});
```

## Packet Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `sendPacket` | `SendPacketEvent` | Fired when a packet is about to be sent to the server. |
| `receivePacket` | `ReceivePacketEvent` | Fired when a packet is received from the server. |
| `instantaneousSendPacket` | `InstantaneousSendPacketEvent` | Fired when an immediate packet is sent (no queue). |
| `instantaneousReceivePacket` | `InstantaneousReceivePacketEvent` | Fired when an immediate packet is received (no queue). |

```javascript
module.on("sendPacket", (event) => {
    // event.getPacket() returns the raw Minecraft packet object
});

module.on("receivePacket", (event) => {
    // event.getPacket() returns the incoming packet
});
```

## World & Lifecycle Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `postGameTick` | None | Fired at the end of a game tick (20 TPS). Symmetric with `preGameTick`. |
| `joinWorld` | `JoinWorldEvent` | Fired when the local player joins a world, after the client world is initialised. |
| `blockUpdate` | `BlockUpdateEvent` | Fired when a loaded block changes state. Carries the position and old/new states. |
| `serverConnect` | `ServerConnectEvent` | **Cancellable.** Fired before connecting to a multiplayer server. |
| `serverDisconnect` | `ServerDisconnectEvent` | Fired when the client disconnects from a server. |

```javascript
// Logic that runs after vanilla tick processing
module.on("postGameTick", () => {
    // end-of-tick cleanup
});

// React to the player loading into a world
module.on("joinWorld", (event) => {
    client.print("Joined a world!");
});

// Watch for block changes near the player
module.on("blockUpdate", (event) => {
    const pos = event.getPos();
    const oldState = event.getOldState();
    const newState = event.getNewState();
});

// Block a connection to a specific server
module.on("serverConnect", (event) => {
    const address = event.getServerAddress();
    // event.setCancelled() aborts the connection (see Cancellable events)
});

module.on("serverDisconnect", (event) => {
    client.print("Disconnected.");
});
```

## Chat Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `chatReceived` | `ChatReceivedEvent` | **Cancellable.** Fired when a chat message is received, before it is shown. |

```javascript
module.on("chatReceived", (event) => {
    const text = event.getText();        // Component
    if (event.isOverlay()) return;       // action-bar message
    // event.setOverlay(true) reroutes it to the action bar
    // event.setCancelled() suppresses the message (see Cancellable events)
});
```

## Input Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `keyPress` | `KeyPressEvent` | Fired when a keyboard key is pressed. Carries the GLFW key code. |
| `mousePress` | `MousePressEvent` | Fired when a mouse button is pressed. Carries the GLFW button code. |
| `resolutionChange` | `ResolutionChangeEvent` | Fired when the GUI is resized (the "screen resize" hook). |

Both `keyPress` and `mousePress` expose the raw GLFW code via `getInteractionCode()`.

```javascript
module.on("keyPress", (event) => {
    const keyCode = event.getInteractionCode();   // GLFW key code
});

module.on("mousePress", (event) => {
    const button = event.getInteractionCode();    // GLFW mouse button code
});

module.on("resolutionChange", () => {
    // recompute layout that depends on screen size
});
```

## Interaction Events

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `attack` | `AttackEvent` | Fired when the player attacks an entity, before the interaction is processed. |
| `swing` | `SwingEvent` | Fired when the player swings an arm, before the swing is sent. |
| `itemUse` | `ItemUseEvent` | Fired when the player uses (right-clicks) the held item. |
| `jump` | `JumpEvent` | **Cancellable.** Fired before the jump impulse is applied. |

```javascript
module.on("attack", (event) => {
    const target = event.getTarget();    // Entity
});

// SwingEvent is a record — use hand() not getHand()
module.on("swing", (event) => {
    const hand = event.hand();           // InteractionHand
});

module.on("itemUse", (event) => {
    // no arguments
});

module.on("jump", (event) => {
    if (event.isSprinting()) {
        event.setSprinting(false);       // drop the sprint-jump boost
    }
    // event.setCancelled() suppresses the jump (see Cancellable events)
});
```

## Bloom Rendering (Advanced)

| Event Name | Callback Args | Description |
| :--- | :--- | :--- |
| `renderBloom` | `RenderBloomEvent` | Advanced. Shapes drawn here feed the HUD glow/bloom pass. |

`renderBloom` is a record event. Draw into the bloom-target canvas it carries; anything drawn here is fed through the glow/bloom pass rather than shown directly.

```javascript
// SwingEvent/RenderBloomEvent are records — use field() accessors, no get prefix
module.on("renderBloom", (event) => {
    const ctx = event.drawContext();     // GuiGraphicsExtractor
    const delta = event.tickDelta();     // float
    const canvas = event.canvas();       // RenderCanvas
    // draw glowing shapes into the bloom target here
});
```

## Cancellable Events

Some events extend the client's cancellable base, letting a handler suppress the vanilla behaviour: `chatReceived` (suppress the message), `jump` (suppress the jump), and `serverConnect` (abort the connection).

Cancel with `event.setCancelled()` — there is no argument and the flag cannot be cleared once set. Read the current state with `event.isCancelled()`.

```javascript
// Hide messages containing a phrase
module.on("chatReceived", (event) => {
    const raw = event.getText().getString();
    if (raw.includes("spam")) {
        event.setCancelled();            // message is suppressed
    }
});
```

## Record-Type Events

A few events are Java **records**. Their data is read through bare field accessors — `event.hand()`, not `event.getHand()`:

| Event | Accessors |
| :--- | :--- |
| `swing` | `event.hand()` |
| `renderWorld` | `event.tickDelta()` |
| `renderBloom` | `event.drawContext()`, `event.tickDelta()`, `event.canvas()` |

All other events documented here use `getField()` getters (for example `event.getTarget()`, `event.getPos()`).

## Error Handling

If a callback throws an error, the error is printed to chat and the handler is **removed** to prevent spam. Fix the error and `.script reload`.

## Null Safety

Always check `mc.player` and `mc.world` before accessing game state — they're null in menus and loading screens:

```javascript
module.on("preGameTick", (event) => {
    if (mc.player === null || mc.world === null) return;
    // Safe to use player/world proxies here
});
```
