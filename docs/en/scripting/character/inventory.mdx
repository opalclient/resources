# Inventory Proxy

Slot switching (including silent/spoof variants), item searching, and stack inspection for the local player's inventory.

---

## Inventory Proxy
**Global Binding:** `inventory`

Wraps the internal `SlotHelper` system used by Scaffold and other modules for server-side slot manipulation, plus convenience helpers for searching the hotbar and full inventory and inspecting individual stacks.

<Callout type="info">
    Hotbar slots are indexed `0-8`. The full inventory uses slots `0-35` (hotbar `0-8`, main inventory `9-35`). Search and count helpers match an item's display name as a case-insensitive substring.
</Callout>

<Callout type="warning">
    All methods read the local player without a null guard. Always confirm the player exists before calling (`if (mc.getPlayer() === null) return;`) to avoid errors when not in a world. Note the call: `mc.player` reads as `undefined`, so a `mc.player === null` guard never fires.
</Callout>

    name="getSelectedSlot"

```javascript
module.on("preGameTick", (event) => {
    if (mc.getPlayer() === null) return;

    const blockSlot = inventory.findBlock();
    if (blockSlot !== -1) {
        inventory.setSlotSilent(blockSlot);
        client.print("Switched to blocks in slot " + blockSlot);
    }
});
```

/>

<Method
    name="findItemInInventory"
    signature="int findItemInInventory(String itemName)"
    description="Searches the full inventory (slots 0-35) for an item whose display name contains the given substring (case-insensitive)."
    params={[
        { name: "itemName", type: "String", desc: "The search string." }
    ]}
    returns={{ type: "int", desc: "The inventory slot index (0-35), or -1 if not found." }}
/>

<Method
    name="findItemById"
    signature="int findItemById(String id)"
    description="Searches the hotbar (slots 0-8) for an item by its stable registry id and returns the slot index. Accepts 'diamond' or 'minecraft:diamond' (the default namespace is minecraft)."
    params={[
        { name: "id", type: "String", desc: "The item's registry id, bare or namespaced." }
    ]}
    returns={{ type: "int", desc: "The hotbar slot index (0-8), or -1 if not found." }}
/>

<Callout type="info">
    `findItemById` / `countItemById` key on the **registry id** (`minecraft:diamond`), which is stable and locale-independent, so prefer them for logic. `findItem` / `countItem` match the **display name** as a case-insensitive substring: convenient, but locale-dependent and changed by an anvil rename. The id here is the same one `ItemStack.getId()` returns, so the two always agree.
</Callout>

```javascript
if (mc.getPlayer() === null) return;

const swordSlot = inventory.findItem("sword");
if (swordSlot !== -1) {
    inventory.setSlot(swordSlot);
}

const totemSlot = inventory.findItemInInventory("totem");
client.print("Totem at inventory slot " + totemSlot);
```

</Callout>

    name="countBlocks"
    signature="int countBlocks()"
    description="Counts the total number of placeable blocks across the hotbar (slots 0-8) and the off hand, using the Scaffold block validation."
    returns={{ type: "int", desc: "The total block count." }}
/>

```javascript
module.on("preGameTick", (event) => {
    if (mc.getPlayer() === null) return;

    const blocks = inventory.countBlocks();
    if (blocks < 16) {
        notification.warn("Low Blocks", "Only " + blocks + " blocks left", 2000);
    }
});
```

<Callout type="info">
    Numbers in scripts are always JavaScript doubles. Slot indices and counts are narrowed back to `int` on the Java side, so pass whole-number values. When working with ItemStack colors or any ARGB value elsewhere, build colors with `renderer.color(r, g, b[, a])` rather than raw `0xAARRGGBB` literals: a JS double above 2^31 truncates incorrectly when narrowed to an `int`.
</Callout>
