# Player Proxy

Local player state, position and rotation, health, and combat/entity utilities pulled from the client's own KillAura and Scaffold helpers.

---

## Player Proxy
**Global Binding:** `player`

Wraps the local player so scripts can read position, rotation, movement state, health, and combat context, and trigger basic actions like swinging or using the held item.

<Callout type="warning">
    Every method here reads from `mc.player`, which is `null` on the main menu, during world loads, and between disconnects. Calling a `player` method while it is `null` throws into your script and stops the handler. Guard every handler before touching `player`:
</Callout>

```javascript
if (mc.player === null) return;
```

<Callout type="info">
    JavaScript has a single number type (double), so every `int`, `float`, and `double` return on this page arrives as a plain JS number. The entity helpers fail soft: `getDistanceToEntity()` returns `-1.0` and `getClosestPoint()` returns `null` when handed a non-living entity, so test for those sentinels before using the result.
</Callout>

<Callout type="info">
    `getEyePosition()`, `getPosition()`, and `getVelocity()` return raw `Vec3d` values. Their instance methods are intermediary-named at runtime and are **not** callable by readable name — pass them into proxy methods (such as `rotation.getRotationFromPosition` or `esp.projectVec`), or use `getBlockPosition()` when you need readable `getX()`/`getY()`/`getZ()`. See the Types reference.
</Callout>

```javascript
module.on("preGameTick", (event) => {
    if (mc.player === null) return;

    const pos = player.getBlockPosition();
    client.print("Standing at Y=" + pos.getY());
    client.print("Yaw " + player.getYaw() + " / Pitch " + player.getPitch());
});
```

```javascript
// Aim at and attack the nearest reachable living entity.
module.on("preGameTick", (event) => {
    if (mc.player === null || mc.world === null) return;

    const targets = world.getLivingEntitiesInRange(player.getEntityInteractionRange());
    if (targets.isEmpty()) return;

    const target = targets.get(0);
    if (player.getDistanceToEntity(target) < 0) return; // non-living entity

    const aim = player.getClosestPoint(target);
    if (aim === null) return;

    const rot = rotation.getRotationFromPosition(aim);
    rotation.set(rot.getYaw(), rot.getPitch());

    mc.interactionManager.attackEntity(target);
    player.swingHand(MAIN_HAND);
});
```

```javascript
// Scaffold-style edge check: is there ground under the next step?
if (mc.player !== null && player.isBoxEmptyBelow(-0.6)) {
    client.print("No ground below — would fall");
}
```

<Method
    name="swingHand"

    description="Plays the hand swing animation for the given hand — call it after attacking or placing to trigger the visual swing."
    params={[
        { name: "hand", type: "InteractionHand", desc: "MAIN_HAND or OFF_HAND." }
    ]}
/>

<Method
    name="useItem"
    signature="void useItem(InteractionHand hand)"
    description="Uses (right-clicks) the item in the given hand and plays the swing animation."
    params={[
        { name: "hand", type: "InteractionHand", desc: "MAIN_HAND or OFF_HAND." }
    ]}
/>

<Callout type="info">
    `MAIN_HAND` and `OFF_HAND` are global `InteractionHand` constants injected into every script — see the Types reference. Both `swingHand` and `useItem` already null-check internally, but guarding the whole handler with `if (mc.player === null) return;` keeps your logic consistent.
</Callout>

```javascript
module.on("preGameTick", (event) => {
    if (mc.player === null) return;

    // Right-click the main-hand item (eat, throw a pearl, place, ...).
    player.useItem(MAIN_HAND);
});
```
