# Player Proxy

Local player state, position and rotation, health, and combat/entity utilities pulled from the client's own KillAura and Scaffold helpers.

---

## Player Proxy
**Global Binding:** `player`

Wraps the local player so scripts can read position, rotation, movement state, health, and combat context, and trigger basic actions like swinging or using the held item.

<Callout type="warning">
    Every method here reads the local player, which is `null` on the main menu, during world loads, and between disconnects. Calling a `player` method while it is `null` throws into your script and stops the handler. Guard every handler before touching `player`:
</Callout>

```javascript
if (mc.getPlayer() === null) return;
```

<Callout type="warning">
    Note the **call**: `mc.getPlayer()`. Read as a property, `mc.player` is `undefined`, not `null`, so `if (mc.player === null) return;` is always false and the guard silently never fires.
</Callout>

<Callout type="info">
    JavaScript has a single number type (double), so every `int`, `float`, and `double` return on this page arrives as a plain JS number. The entity helpers fail soft: `getDistanceToEntity()` returns `-1.0` and `getClosestPoint()` returns `null` when handed a non-living entity, so test for those sentinels before using the result.
</Callout>

## Status Effects

Read the player's active potion effects. Each returns an [`Effect`](../world/types#effect) wrapper. Effect names accept either the bare name (`"speed"`) or the namespaced form (`"minecraft:speed"`).

<Callout type="warning">
    **`getAmplifier()` counts from 0; `getLevel()` counts from 1.** Speed II is amplifier `1` and level `2`. Use `getAmplifier()` for vanilla movement math, and `getLevel()` for anything a player reads. Mixing them up is an off-by-one through every calculation downstream.
</Callout>

<Callout type="info">
    `getDurationSeconds()` returns **`-1` for an infinite effect**, not a large number. Call `isInfinite()` before formatting a countdown.
</Callout>

<Method
    name="hasEffect"
    signature="boolean hasEffect(String name)"
    description="Returns whether the player currently has the named status effect."
    params={[
        { name: "name", type: "String", desc: "Effect name, with or without the minecraft: namespace." }
    ]}
    returns={{ type: "boolean", desc: "True if the effect is active." }}
/>

<Method
    name="getEffect"
    signature="Effect getEffect(String name)"
    description="Returns the named active status effect, or null if the player does not have it."
    params={[
        { name: "name", type: "String", desc: "Effect name, with or without the minecraft: namespace." }
    ]}
    returns={{ type: "Effect | null", desc: "The active effect, or null." }}
/>

<Method
    name="getEffects"
    signature="List&lt;Effect&gt; getEffects()"
    description="Returns every status effect currently active on the player. A List wrapper, not a JS array: walk it with size() and get(i)."
    returns={{ type: "List<Effect>", desc: "Active effects." }}
/>

```javascript
module.on("preGameTick", () => {
    if (mc.getPlayer() === null) return;

    // Vanilla speed math wants the 0-based amplifier.
    if (player.hasEffect("speed")) {
        const speed = player.getEffect("speed");
        const boost = 1.0 + 0.2 * (speed.getAmplifier() + 1);
        client.print("Speed multiplier: " + boost.toFixed(2));
    }

    // Anything the player reads wants the 1-based level.
    const effects = player.getEffects();
    for (let i = 0; i < effects.size(); i++) {
        const effect = effects.get(i);
        const time = effect.isInfinite() ? "∞" : effect.getDurationSeconds() + "s";
        client.print(effect.getName() + " " + effect.getLevel() + " — " + time);
    }
});
```

```javascript
// Aim at and attack the nearest reachable living entity.
module.on("preGameTick", (event) => {
    if (mc.getPlayer() === null || mc.getWorld() === null) return;

    const targets = world.getLivingEntitiesInRange(player.getEntityInteractionRange());
    if (targets.isEmpty()) return;

    const target = targets.get(0);
    if (player.getDistanceToEntity(target) < 0) return; // non-living entity

    const aim = player.getClosestPoint(target);
    if (aim === null) return;

    const rot = rotation.getRotationFromPosition(aim);
    rotation.set(rot.getYaw(), rot.getPitch());

    mc.interactionManager.attackEntity(target);
    player.swingHand(MAIN_HAND);
});
```

```javascript
// Scaffold-style edge check: is there ground under the next step?
if (mc.getPlayer() !== null && player.isBoxEmptyBelow(-0.6)) {
    client.print("No ground below — would fall");
}
```

    description="Uses (right-clicks) the item in the given hand and plays the swing animation."
    params={[

    ]}
/>

<Callout type="info">
    `MAIN_HAND` and `OFF_HAND` are global `InteractionHand` constants injected into every script. See the Types reference. Both `swingHand` and `useItem` already null-check internally, but guarding the whole handler with `if (mc.getPlayer() === null) return;` keeps your logic consistent.
</Callout>

```javascript
module.on("preGameTick", (event) => {
    if (mc.getPlayer() === null) return;

    // Right-click the main-hand item (eat, throw a pearl, place, ...).
    player.useItem(MAIN_HAND);
});
```
