# Movement Proxy

Read and manipulate the local player's movement: query speed, set velocity, and resolve movement direction from input.

---

## Movement Proxy
**Global Binding:** `movement`

Provides scripts with access to movement utilities including speed manipulation, direction calculation, and movement state queries.

<Callout type="info">
    JavaScript has only one number type (a 64-bit double). The Java methods below distinguish between `float` and `double` parameters, but from a script you just pass plain numbers. The engine narrows them as needed. Angles are always in degrees unless the method name ends in `Radians`.
</Callout>

```javascript
module.on("preGameTick", (event) => {
    if (movement.isMoving()) {
        client.print("Speed: " + movement.getSpeed().toFixed(2));
        movement.setSpeed(0.5);
    }
});
```

### Speed manipulation

<Method
    name="yawPos"
    signature="List&lt;double&gt; yawPos(float yaw, double value)"
    description="Computes the X and Z offsets for a given yaw direction and distance. Returns a two-element List wrapper, not a JS array: read it with get(0) and get(1)."
    params={[
        { name: "yaw", type: "float", desc: "The direction angle in degrees." },
        { name: "value", type: "double", desc: "The distance to project." }
    ]}
    returns={{ type: "List<double>", desc: "A two-element list: get(0) is deltaX, get(1) is deltaZ." }}
/>

<Callout type="warning">
    `yawPos` returns a [`List`](../world/types#list), **not a JS array**. `offset[0]` evaluates to `undefined` without throwing, so the arithmetic that follows quietly becomes `NaN`. Read it with `get(0)` / `get(1)`.
</Callout>

```javascript
// Project 1 block forward along the player's movement yaw.
const offset = movement.yawPos(movement.getMoveYaw(), 1.0);
client.print("dX=" + offset.get(0).toFixed(3) + " dZ=" + offset.get(1).toFixed(3));
```

    name="setSpeed"
    signature="void setSpeed(double speed, float yaw)"
    description="Sets the local player's horizontal speed along a specific yaw direction."
    params={[

    ]}

<Callout type="warning">
    `setSpeed(speed, strafePercentage)` and `setSpeed(speed, yaw)` are different overloads that both take two numbers. Because JavaScript has a single number type, the engine cannot tell a strafe blend (`0.0-1.0`) apart from a yaw angle (degrees) purely by value; both resolve to the same call site. If you need an explicit yaw, prefer building the offset with `yawPos(...)` and applying it, or pass a value that is unambiguous for your intent. Test both behaviors in-game before shipping a script.
</Callout>

### Direction

<Method
    name="getMoveYaw"
    signature="float getMoveYaw(double fromX, double fromZ, double toX, double toZ)"
    description="Calculates the movement yaw from one horizontal world position to another. Takes plain numbers, and the pairs are world X and Z (not X and Y)."
    params={[
        { name: "fromX", type: "double", desc: "The starting world X coordinate." },
        { name: "fromZ", type: "double", desc: "The starting world Z coordinate." },
        { name: "toX", type: "double", desc: "The target world X coordinate." },
        { name: "toZ", type: "double", desc: "The target world Z coordinate." }
    ]}
    returns={{ type: "float", desc: "The yaw angle in degrees." }}
/>

<Callout type="info">
    This overload takes four plain numbers, and both pairs are **world X and Z** — the horizontal plane. Passing a Y coordinate gives a meaningless yaw.
</Callout>

```javascript
// Yaw from the player toward a target block's column.
const pos = player.getPosition();
const yaw = movement.getMoveYaw(pos.getX(), pos.getZ(), 100.0, -240.0);
```

### State queries

<Method
    name="isMoving"
    signature="boolean isMoving()"
    description="Returns whether the local player is currently providing movement input (any of WASD or equivalent keys are pressed)."
    returns={{ type: "boolean", desc: "True if the player has active movement input." }}
/>

<Callout type="info">
    `isMoving()` reflects input keys, not actual velocity. A player held still against a wall while pressing W still counts as moving; a player sliding on ice after releasing the keys does not.
</Callout>
