<SetupCard icon="settings" title="Setup Instructions" description="From zero to in-game in four steps.">
<Steps>

<Step number={1} title="Create your account">
Register on the [dashboard](/dash), then activate your license by buying one or redeeming a key. While you're there, link your integrations under Settings → Connections:

- **Discord** for community features and support
- **Spotify** for in-game music (you'll create a Spotify app on the connections page)
</Step>

<Step number={2} title="Download the launcher">
Grab the **Opal Launcher** for your platform from the [Downloads](/dash) page. It installs Fabric, manages your mods, and keeps everything up to date, so there's nothing to set up by hand.
</Step>

<Step number={3} title="Sign in and configure">
Open the launcher and sign in with your Opal account. Everything lives here: your Minecraft path, Java version, memory, mod list, and packs. Have a poke around and set it up how you like. Fabric API is the one mod you can't skip.
</Step>

<Step number={4} title="Launch">
Hit **Launch**. The launcher handles auth, updates, and loading for you. It'll ask you to sign in to your Opal account on first launch, and after that you're straight in.
</Step>

</Steps>
</SetupCard>

<SetupCard icon="settings" title="Installing by platform" description="Grab the launcher for your OS, then follow the notes for that platform.">

### Windows (x64)

1. Download the launcher for Windows from the [Downloads](/dash) page and run it.
2. If Windows SmartScreen warns you, click **More info → Run anyway** — fresh builds aren't reputation-flagged yet.
3. The WebView2 runtime ships with Windows 10 and 11. If the window opens blank, install the [WebView2 runtime](https://developer.microsoft.com/microsoft-edge/webview2/) and relaunch.

### macOS (Apple Silicon or Intel)

1. Download the build for your chip — **arm64** for Apple Silicon (M-series), **x64** for Intel. Check under Apple menu → About This Mac.
2. macOS quarantines apps that don't come from the App Store, so clear the quarantine flag and mark the file executable before the first launch:

   ```bash
   xattr -dr com.apple.quarantine ~/Downloads/launcher-macos-arm64
   chmod +x ~/Downloads/launcher-macos-arm64
   ```

   Swap `arm64` for `x64` on an Intel Mac. You can also right-click the file → **Open** → **Open** to clear Gatekeeper for that file once.
3. If it's still blocked, allow it under **System Settings → Privacy & Security → Open Anyway**, then launch again.

<Note>
macOS builds aren't Apple Developer-ID signed yet, so the quarantine step above is required for now. Once the builds are notarized this step goes away.
</Note>

### Linux (x64)

1. Download and unpack the Linux build, then make it executable: `chmod +x launcher-linux-x64`.
2. Install the runtime pieces the launcher needs — package names vary by distro:
   - **WebKitGTK** — `libwebkit2gtk-4.1-0` (Debian/Ubuntu), `webkit2gtk4.1` (Fedora), `webkit2gtk-4.1` (Arch)
   - **A keyring** — `gnome-keyring` or your desktop's equivalent, running before you launch
   - **A dialog service** — `zenity` or `kdialog`
3. Run `./launcher-linux-x64`. If it crashes referencing `wtf.opal` or spams the log, you're missing the keyring or dialog service above — see **Linux-Specific Issues** below.

<SetupCard icon="alert" title="Common Errors" description="Codes you might run into and how to clear them.">
<ErrorList>
<ErrorItem code="G-26" title="HWID Mismatch">
Your hardware ID doesn't match the one on file. Reset it from the [dashboard](/dash) under account settings (once every 30 days), or open a support ticket if you need an early reset.
</ErrorItem>

<ErrorItem code="0x31" title="Debugging Software Detected">
A traffic inspection tool such as HTTP Debugger is running. Close or uninstall it, then launch again.
</ErrorItem>
</ErrorList>
</SetupCard>

<SetupCard icon="alert" title="Linux-Specific Issues" description="A couple of extra things for Linux users.">
<ErrorList>
<ErrorItem code="Chat Spam" title="No Dialog Service">
Repeated spam in the logs, or nothing after Minecraft-style log output, usually means you're missing a dialog popup service. Install your distro's equivalent, such as zenity or kdialog.
</ErrorItem>

<ErrorItem code="Crash (wtf.opal.*)" title="Missing Keyring">
A crash referencing `wtf.opal` means no keyring service is running. Install `gnome-keyring` (or your desktop's equivalent) and make sure it's started before you launch.
</ErrorItem>
</ErrorList>
</SetupCard>

<SetupCard icon="faq" title="FAQ" description="Quick answers to the usual questions.">
<FAQList>
<FAQItem question="Can I use Opal on older-version servers?">
Yes. Install [ViaFabricPlus](https://modrinth.com/mod/viafabricplus) and you can join 1.8 and other older servers while running Minecraft 1.21.10.
</FAQItem>

<FAQItem question="Do I need the Opal Launcher?">
Yes. The Opal Launcher is the only supported way to run Opal. It sets up Fabric, keeps the client updated, and manages your mods, so there's nothing to configure yourself.
</FAQItem>

<FAQItem question="A mod is crashing my game. How do I find it?">
Binary-search it. Remove half your mods and test. Still crashing? Remove half of what's left. Works again? Add half back and repeat. Leave Fabric API and Opal in place since they're required, and you'll narrow it down fast.
</FAQItem>

<FAQItem question="I can't connect from my region.">
Some networks block the connection. A VPN or proxy usually sorts it out. If it keeps happening, open a support ticket and we'll help you get on.
</FAQItem>

<FAQItem question="How do I reset my HWID?">
Dashboard → Settings → Account. Resets are available once every 30 days. Need one sooner? Open a support ticket on Discord.
</FAQItem>
</FAQList>
